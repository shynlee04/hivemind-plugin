# Acceptance Test Patterns

## API Tests

- Request/response shape validation
- Status code coverage (200, 400, 401, 403, 404, 500)
- Header validation
- Rate limit behavior

## UI Tests

- Element presence/absence
- Interaction flow (click → navigate → assert)
- Visual regression (screenshot diff)
- Accessibility (axe-core scan)

## Algorithm Tests

- Known input/output pairs
- Boundary conditions (empty, single, max)
- Randomized property-based tests
- Performance benchmarks
