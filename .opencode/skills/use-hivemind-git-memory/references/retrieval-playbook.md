# Retrieval Playbook

## Resume
- start with recent commits on the active branch
- pull forward only work relevant to the current objective

## Trace
- use the target commit or file history
- explain what changed and why it likely changed

## Retrieve
- scan a bounded history window
- collect decisions and rationale into one continuity result

## Anchor
- propose the fields that should be reflected in the next commit or follow-up note
- do not mutate git from the skill itself

## Confidence Rule
- label evidence as `confirmed` when present in commit/message/diff history
- label as `inferred` when reconstructed from partial evidence
