# Linked Knowledge Expectations

| Domain | Relevant Pack Skills | Router Should Assume | Router Should Refuse / Block | Output Artifact |
| --- | --- | --- | --- | --- |
| `context` | `context-intelligence-entry`, `context-entry-verify` | `context-intelligence-entry` is diagnostic and `context-entry-verify` is hard proof | unsupported auto-run claims; using diagnostics as proof; overlapping entry claims | investigation report, verification handoff |
| `delegation` | `hivemind-delegation-protocol` | delegation needs bounded scope, explicit mode, and a return contract | vague handoff, recursive delegation, packets that depend on deleted legacy skills | partition plan, verification handoff, refactor stage report |
| `git-memory` | `git-continuity-memory` | continuity is background evidence for recovery, not generic git ops; history quality must be classified first | phantom resume routes; stable-recovery claims without commit evidence or continuity results | continuity manifest, knowledge synthesis |
| `codemap` | `hivemind-codemap` | whole-repo mapping should generate explicit seam and concern slices before refactor; deep or exhaustive scans should be stateful and batch-based | shallow file listing without structural synthesis; exhaustive scans without state or accounting | scan plan, seam inventory, codemap synthesis, optional repomix extraction report |
| `system-debug` | `hivemind-system-debug` | reproducibility and containment are required before repair claims | fix-first behavior; evidence-free debugging; skipping rollback posture | debug stage report, refactor readiness |
| Stabilization | router templates + `context-entry-verify` | stabilization is a late-stage reintegration pass, not entry routing | calling the work complete without proof and follow-up guidance | stabilization report, follow-up guidance |
