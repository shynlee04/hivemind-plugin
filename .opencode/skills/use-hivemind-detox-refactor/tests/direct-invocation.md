# Direct Invocation

## Scenario
- The project has conflicting entry routers, missing downstream targets, and unreliable git-memory routing.
- The user asks for strategic detox rather than a single-file fix.

## Expected Behavior
1. Classify the work as `use-hivemind-detox-refactor`.
2. Select the smallest local family first instead of depending on deleted sibling skills.
3. Enter the explicit stage workflow rather than jumping straight to edits.
4. Produce a stage-appropriate report output and, when needed, a partition plan.
5. Make linked-knowledge expectations and blocked routes explicit.
6. Block phantom or deprecated routes explicitly.

## Scenario 2
- The project has a governance recovery problem: root `AGENTS.md` guidance conflicts with late-stage detox findings, and builder-oriented skills are trying to reclaim entry routing.

## Expected Behavior
1. Keep governance recovery as a late-stage reintegration concern rather than the default entry route.
2. Route builder overreach through Bundle D instead of trusting builder surfaces by default.
3. Require explicit stabilization and follow-up outputs before calling the router flow complete.
