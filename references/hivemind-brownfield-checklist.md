# Brownfield Integrity Checklist

## Before Code Changes
- [ ] Framework mode identified (GSD / Spec-kit / BMAD / none)
- [ ] Framework conflicts resolved or explicitly overridden
- [ ] Baseline anchors persisted (`project-stack`, `framework-context`, `brownfield-risks`)
- [ ] Memory baseline saved to `project-intel`
- [ ] Trajectory/tactic/action mapped

## During Refactor
- [ ] Drift checked at major checkpoints
- [ ] Evidence captured for each quality claim
- [ ] Long-haul changes delegated with export cycle discipline

## Before Completion
- [ ] Re-run analyze/recommend to confirm risk reduction
- [ ] Verify docs and runbook remain accurate
