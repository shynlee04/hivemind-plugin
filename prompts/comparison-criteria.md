# Comparison Criteria

Use this prompt when comparing technologies, patterns, tools, or implementation approaches.

## Comparison Method

1. List all candidates and confirm each candidate is viable for the stated context.
2. Define 5-8 criteria with explicit weights that total 100.
3. Tie each criterion to business or engineering goals (cost, speed, reliability, security, maintainability).
4. Score each candidate on a 1-5 scale per criterion with evidence-backed justification.
5. Separate current-state evidence from future assumptions.
6. Compute weighted totals and rank candidates.
7. Run sensitivity analysis by adjusting key weights (+/-20 percent) and note rank changes.
8. Document key tradeoffs, migration concerns, and lock-in risks.

## Output Requirements

- Show criteria, weights, and scoring rubric before final scores.
- Include a scoring matrix with weighted totals.
- Report the winner, runner-up, and conditions where recommendation would change.
- End with adoption recommendation and risk mitigation actions.
