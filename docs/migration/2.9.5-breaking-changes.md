# 2.9.5 Breaking Changes & Migration Guide

## Overview
Version 2.9.5 is a **hard cutover** to SDK-native architecture. There are no compatibility shims — all deprecated paths are removed.

## Removed Modules

| Module | Replacement |
|--------|-------------|
| `src/core/session/` (5 files) | SDK hooks (`chat.message`, `permission.ask`) |
| `src/shared/event-bus.ts` | SDK hook system — no custom pub/sub |
| `src/hooks/session-lifecycle.ts` | SDK `chat.message` + `start-work` hooks |
| `src/hooks/messages-transform.ts` | `experimental.chat.messages.transform` hook bridge |
| `src/schemas/` | Zod schemas in `src/tools/*/tools.ts` |
| `src/lib/hooks/` | Direct SDK hook registration |

## Renamed Exports

| Old | New |
|-----|-----|
| `emitGovernanceToast()` | `showGovernanceToast(category, message)` |
| Inline tools in plugin entry | `createHivemind*Tool()` in `src/tools/` |

## New APIs

| API | Module | Purpose |
|-----|--------|---------|
| `getEffectivePaths(root)` | `src/shared/paths.ts` | Canonical path authority |
| `showGovernanceToast(cat, msg)` | `src/hooks/soft-governance.ts` | SDK-bridged toast |
| `agentToolCatalog` | `src/tools/index.ts` | Full tool surface catalog |
| `createRuntimeSurfaceRegistry()` | `src/plugin/surface-registry.ts` | Runtime surface catalog |

## New Governance Guards

7 new guard scripts added to `npm run lint:boundary` (11 total). See `docs/architecture/sdk-native-architecture.md` for the full list.

## Migration Steps

1. **Remove old imports** — Any import from `src/core/session/`, `src/schemas/`, `src/shared/event-bus` will fail
2. **Update toast calls** — `emitGovernanceToast({id, message, variant})` → `showGovernanceToast(category, message)`
3. **Use path authority** — Replace ad-hoc path construction with `getEffectivePaths(root)`
4. **Tool access** — Use `createHivemind*Tool(root)` factories from `src/tools/` barrel
5. **Run gates** — `npx tsc --noEmit && npm run lint:boundary && npx tsx --test tests/*.test.ts`
