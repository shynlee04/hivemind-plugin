## Vision

Create an interactive planning artifact system that enables agents to collaboratively build structured workflows using JSON-validated contracts with progressive patch application.

## Context

### Reference Implementations Analyzed

1. **Octto** (`octto.xml`) - Branch-based brainstorming with WebSocket Q&A
   - Dual-store architecture: SessionStore (ephemeral) + StateStore (persistent)
   - Branch-based parallel exploration with scoped subagents
   - Probe/Bootstrapper/Orchestrator pattern for adaptive continuation
   - Session-level locking for concurrency safety

2. **json-render** (`json-render.xml`) - Schema-Catalog-Registry pattern
   - `defineSchema()` → `SchemaBuilder DSL` → `createCatalog()` type-safe composition
   - SpecStream (RFC 6902 JSON Patches) for progressive building
   - Mixed stream parsing: Text + JSONL patch differentiation
   - Zod integration with `s.zod()` and type inference

3. **OpenCode SDK** (`opencode-api-sdk.xml`) - Plugin surface
   - `tool.schema` is Zod re-export for type-safe arguments
   - `context` provides sessionID, agent, directory, worktree, abort, metadata(), ask()
   - 17+ hooks available for lifecycle integration

### Current Codebase Issues (from analysis)

1. **God-Function**: `loadRuntimeBindingsSnapshot` called from 11 files, triggering 7+ I/O ops per call
2. **Cross-Feature Import Spaghetti**: Heavy upward/lateral imports in `features/`
3. **Multiple Writers**: No locking or coordination between writers to same state files
4. **Barrel Re-export**: `src/index.ts` exposes entire surface area with no boundary enforcement

## Architecture Decision: JSON over YAML/MD/XML

| Format | Advantage | Disadvantage |
|--------|-----------|--------------|
| **JSON** | Native `JSON.parse/stringify`, Zod validation, RFC 6902 patches, streamable | Not human-friendly for editing |
| YAML | Human-readable | Needs `yaml` dep, no streaming, ambiguous types |
| Markdown | Agent-readable | Not programmatically parseable without custom parser |
| XML | Structured | Verbose, no native TS tooling |

- Use the existing plugin `event` hook for `session.created` and `session.compacted` side effects.
- Keep `chat.message` as reset/coordination only if needed; it must not become a runtime context injector.
- Keep `experimental.chat.messages.transform` as the sole AI-visible runtime packet injector.
- Keep `experimental.session.compacting` as the compaction-preservation seam.

## Key Patterns from References

### From Octto: Session Lock Pattern

```typescript
type SessionLock = <T>(sessionId: string, operation: () => Promise<T>) => Promise<T>

function createSessionLock(): SessionLock {
  const queues = new Map<string, Promise<unknown>>()
  
  return <T>(sessionId: string, operation: () => Promise<T>): Promise<T> => {
    const currentQueue = queues.get(sessionId) ?? Promise.resolve()
    const newQueue = currentQueue.then(operation, operation)
    queues.set(sessionId, newQueue)
    return newQueue as Promise<T>
  }
}
```

### From json-render: SpecStream Pattern

```typescript
const compiler = createSpecStreamCompiler<WorkContract>()

for await (const chunk of llmStream) {
  const { result, newPatches } = compiler.push(chunk)
  if (newPatches.length > 0) {
    // Real-time state updates with validation
    dispatcher.emit('contract:predicate', result)
  }
}
```

### From OpenCode SDK: Tool Pattern

```typescript
export default tool({
  description: 'Create or update agent work contract',
  args: {
    action: tool.schema.enum(['create', 'update', 'export']),
    contractId: tool.schema.string().optional(),
    chainActions: tool.schema.object({...}).optional(),
  },
  async execute(args, context) {
    // context.sessionID, context.agent, context.directory, context.worktree
    return JSON.stringify(await contractStore.execute(args))
  }
})
```

## Success Criteria

1. **Containment**: All code in `src/features/agent-work-contract/` with relative imports
2. **JSON Schema**: Zod-validated contract schema with RFC 6902 patch support
3. **Single Snapshot**: One `TurnSnapshotLoader` per turn, not scattered reads
4. **SDK-First**: Use `client.session`, `client.tui`, `client.app.log()` for runtime operations
5. **Lifecycle correctness**: `event` owns startup side effects while `experimental.chat.messages.transform` and `experimental.session.compacting` own AI-visible context
