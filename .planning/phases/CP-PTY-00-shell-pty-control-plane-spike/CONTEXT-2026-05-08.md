# CP-PTY-00 Context — Shell / PTY Control-Plane Spike

## User Intent

The user identified a cross-cutting utility cluster around interactive bash, background command execution, PTY handling, child-session delegation, lifecycle loops, command streaming, and sidecar/tmux-style visibility. The requested action is to add the phase skeleton, preserve BOOT momentum, and prepare research/spec/requirements artifacts so GSD automation can continue from a correct dependency map.

## Decision Boundary

This phase is a planning and architecture spike. It does not implement runtime behavior. It prepares the contract for later implementation after BOOT recovery and E2E proof are stable.

## Current Codebase Surfaces

| Surface | Current role | Risk |
|---|---|---|
| `run-background-command` | Agent-facing background command tool with run/output/input/list/terminate actions. | High-risk shell surface; needs bounded output and permission clarity. |
| PTY manager | Bun PTY integration with graceful fallback. | Runtime-specific capability; must not claim restart resumability. |
| Command delegation handler | Bridges command sessions into delegation lifecycle. | Cross-cuts delegation records and process lifecycle. |
| Delegation manager | Owns SDK delegation, command dispatch, queueing, status, recovery. | At module size threshold; avoid ad hoc growth. |
| SDK delegation handler | WaiterModel child-session completion path. | Must stay separate from command-process semantics. |
| Plugin hooks | Event observers and tool guards. | Must enforce CQRS: hooks observe/guard, tools mutate. |

## External Reference Inputs

- `opencode-pty`: useful PTY tool split and lifecycle patterns.
- `oh-my-openagent`: useful background-session, child-session, concurrency, and tmux observability patterns.
- `just-bash`: useful for deterministic shell simulation and tests, not a PTY replacement.
- OpenCode plugin/server docs: relevant tool, hook, session, command, shell, and TUI surfaces.

## Required Output

CP-PTY-00 must leave a precise implementation contract for CP-PTY-01 with no ambiguity about permission gates, process lifecycle, output bounding, recovery semantics, and integration with BOOT/MCM/f-04 dependencies.
