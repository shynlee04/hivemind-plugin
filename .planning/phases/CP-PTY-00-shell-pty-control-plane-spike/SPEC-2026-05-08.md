# CP-PTY-00 Spec — Background Shell / PTY Control-Plane Spike

## Scope

Define the future runtime contract for background command execution, interactive PTY sessions, command-process delegation, SDK child-session delegation boundaries, permission gates, lifecycle mapping, output streaming, and optional read-only sidecar/tmux projections.

## Out Of Scope

- Runtime implementation in `src/**`.
- Test creation in `tests/**`.
- Mutation of `.opencode/**` primitives.
- Mutation of `.hivemind/**` runtime state.
- Sidecar UI implementation.

## Control-Plane Model

| Lane | Description | Resumability | Future owner |
|---|---|---|---|
| SDK child-session delegation | OpenCode child sessions dispatched through SDK APIs. | Resumable by persisted delegation/session metadata where OpenCode session remains available. | `DelegationManager` + SDK handler |
| PTY command delegation | Interactive process managed through PTY adapter. | Not resumable after parent process restart; may be controllable while process is alive. | Command delegation handler + PTY adapter |
| Headless command delegation | Non-PTY child process fallback. | Not resumable after parent process restart. | Command delegation handler |
| Projection lane | Read-only output/progress projection for sidecar or tmux. | Projection-only; no canonical mutation. | Future sidecar/read-model module |

## Future Tool Contract

The future implementation SHOULD preserve a narrow control surface:

- `start`: create command session with cwd, command, args, timeout, permission context, and capability mode.
- `read`: return bounded output by offset/limit or cursor.
- `write`: send input only to interactive-capable sessions.
- `list`: list command sessions visible to the current root/parent session.
- `terminate`: stop command session and mark terminal state.

## Permission Contract

- Validate cwd against project and approved external workspace policy before spawn.
- Reject unsupported interactive input for headless sessions.
- Treat ambiguous plugin permission states conservatively.
- Preserve OpenCode-native permissions where available; do not duplicate policy without a traceable reason.

## Lifecycle Contract

| Event | Required future behavior |
|---|---|
| command started | Create delegation/status record with lane, terminal kind, parent session, and visibility. |
| output emitted | Append to bounded buffer or projection store without flooding context. |
| input sent | Record action metadata without storing secrets. |
| command exits | Mark terminal status and release queue slot. |
| parent aborts/deletes | Cleanup or mark non-resumable. |
| runtime restarts | Mark PTY/headless sessions `non-resumable-after-restart`; do not pretend recovery happened. |

## CP-PTY-01 Entry Gate

CP-PTY-01 may begin only when one of these is true:

- BOOT-07 E2E proof is complete, or
- the user explicitly authorizes runtime mutation before BOOT-07 and accepts the higher integration risk.
