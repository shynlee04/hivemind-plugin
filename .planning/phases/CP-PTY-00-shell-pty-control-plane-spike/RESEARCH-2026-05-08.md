# CP-PTY-00 Research — Shell / PTY Control Plane

## Executive Finding

The correct path is a small architecture spike now and an implementation phase after BOOT-07. The shell/PTY surface is real and cross-cutting, but too risky to fold into BOOT-02.

## Adopt / Adapt / Reject

| Source | Adopt | Adapt | Reject |
|---|---|---|---|
| OpenCode plugin/server docs | Tool registration, hooks, session APIs, async prompt/command/shell surfaces. | Verify exact installed SDK signatures before implementation. | Do not assume docs signatures match installed package without tests. |
| `opencode-pty` | Tool split: spawn/write/read/list/kill; bounded buffers; cleanup patterns. | Hivemind should preserve its DelegationManager and CQRS boundaries. | Do not copy implementation wholesale or rely on PTY as universal capability. |
| `oh-my-openagent` | Background child-session manager, concurrency, notification-first task model, tmux optionality. | Apply as Hivemind control-plane contracts, not OMO folder structure. | Do not make tmux required or let background tasks bypass evidence gates. |
| `just-bash` | Deterministic shell simulation for tests and safe sandboxed execution. | Use only where command simulation is enough. | Do not use as interactive PTY/TUI/dev-server substrate. |

## Standard Stack

- OpenCode plugin `tool(...)` for agent-facing control-plane tools.
- OpenCode hooks for observation, guard decisions, and TUI affordances only.
- Existing Hivemind `DelegationManager` for dispatch/status ownership.
- Existing `CommandDelegationHandler` for command-process lifecycle.
- Optional `bun-pty` adapter for interactive PTY support.
- Node `child_process` fallback for headless non-PTY commands.

## Architecture Patterns

- Separate SDK child-session delegation from command-process delegation.
- Keep tool split explicit: start/read/write/list/terminate.
- Treat PTY/tmux as optional adapters behind capability detection.
- Store bounded outputs and expose offset/limit reads.
- Use notification-first completion; prevent agent polling loops.
- Persist honest restart semantics: PTY/headless processes are non-resumable after parent runtime restart.

## Common Pitfalls

- Claiming PTY support on Node-only hosts when `bun-pty` is unavailable.
- Allowing unbounded dev-server output into model context.
- Treating plugin tool `ask` permissions as enforceable when OpenCode does not expose a plugin permission prompt path.
- Mixing command-process status with SDK child-session status.
- Letting sidecar/tmux projections mutate canonical state.

## Implementation Readiness

CP-PTY-00 is docs/spec ready. CP-PTY-01 implementation remains blocked until BOOT-07 or explicit user authorization provides a stable runtime baseline.
