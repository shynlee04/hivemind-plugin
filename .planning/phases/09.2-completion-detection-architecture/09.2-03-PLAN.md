<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| child session status → observer | Untrusted or stale SDK/session data crosses into completion state decisions. |
| continuity store → parent auto-loop | Persisted lifecycle state is used to decide whether the parent keeps waiting or synthesizes results. |
| auto-loop retry prompt → LLM | Parent retry prompts consume context budget and can amplify malformed delegation summaries if not bounded. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09.2-08 | Tampering | lifecycle-background-observer.ts | mitigate | Only count assistant/tool evidence from SDK message fetches, require start-gate + two idle polls, and treat malformed parts as zero evidence rather than completion. |
| T-09.2-09 | Denial of Service | create-session-hooks.ts parent auto-loop | mitigate | Enforce bounded parent polling with iteration caps, concise prompts, and compaction-visible state so retries do not consume context indefinitely. |
| T-09.2-10 | Repudiation | continuity-backed parent coordination | mitigate | Patch lifecycle truth before parent synthesis and rely on continuity snapshots for terminal-state auditability rather than transient notification success. |
| T-09.2-11 | Information Disclosure | parent synthesis/wait prompts | accept | Prompts expose only internal session IDs/status counts already present in harness continuity; no external secret boundary is crossed. |
</threat_model>
