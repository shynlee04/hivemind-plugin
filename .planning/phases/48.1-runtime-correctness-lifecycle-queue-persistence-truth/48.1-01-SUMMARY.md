# Phase 48.1 Plan 01: Lifecycle Completion Contract Summary

Corrected the lifecycle/completion contract so `session.idle` is routed to `CompletionDetector.feed(eventType, sessionID)` and queued work cannot jump directly to completed without runtime evidence.

## What Changed

- Fixed `HarnessLifecycleManager.handleEvent()` to feed `"session.idle"` as the event type and the real session ID as the second argument.
- Removed `queued -> completed` from lifecycle transitions, aligning lifecycle state with `TaskStatus` transition truth.
- Added regressions proving reversed `CompletionDetector.feed()` arguments do not work and non-terminal events do not resolve watchers.
