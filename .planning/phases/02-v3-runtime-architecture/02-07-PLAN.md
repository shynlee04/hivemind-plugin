1. Add types to `src/lib/types.ts` (RUN-3e AC#1):
   ```typescript
   export type GovernanceRule = {
     id: string;
     condition: {
       tool?: string;           // Match specific tool name (or "*" for all)
       agent?: string;          // Match specific agent (or "*" for all)
       sessionType?: string;    // Match session type (or "*" for all)
     };
     action: "warn" | "escalate" | "block";
     scope: "session" | "root" | "global";
     description?: string;
   };
   export type RuleEvaluation = {
     ruleId: string;
     matched: boolean;
     sessionID: string;
     toolName: string;
     timestamp: number;
   };
   export type ViolationRecord = {
     id: string;
     sessionID: string;
     toolName: string;
     ruleId: string;
     action: "warn" | "escalate" | "block";
     timestamp: number;
     details: string;
   };
   ```

2. Create `src/lib/governance-engine.ts`:
   - `GovernanceEngine` class:
     - `rules: GovernanceRule[]` — in-memory rule store
     - `violations: ViolationRecord[]` — in-memory violation log
     - `loadRules(path?: string): GovernanceRule[]` — reads `.hivemind/state/governance.json`, returns rules
     - `addRule(rule: GovernanceRule): void` — add rule at runtime (RUN-3e AC#4)
     - `removeRule(ruleId: string): boolean` — remove rule at runtime
     - `listRules(): GovernanceRule[]` — list all rules
     - `evaluateRules(sessionID: string, toolName: string, agent?: string): RuleEvaluation[]` — evaluate all rules matching context
     - `recordViolation(sessionID: string, toolName: string, ruleId: string, action: string, details: string): ViolationRecord` — append to violations, write to `.hivemind/state/violations.json`
     - `getActionForRule(rule: GovernanceRule, evaluation: RuleEvaluation): "warn" | "escalate" | "block" | null` — determine action (null if rule didn't match)
   - Soft policy evaluation (D-04): "warn" → `addWarning()`, "escalate" → log + notify parent (via continuity), "block" → throw error (rare)
   - Rule matching: condition.tool matches toolName (glob or exact), condition.agent matches agent, condition.sessionType matches session context
   - All errors prefixed `[Harness]`

3. Tests in `tests/lib/governance-engine.test.ts`:
   - Rule matches → action executed (warn adds warning, block throws)
   - Rule doesn't match → no action
   - Add/remove rule at runtime → rule list updated
   - Violation logged with correct structure
   - Soft policy: warn/escalate preferred, block only when configured

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Governance config file → Engine | Malicious rules could block all tool execution |
| Runtime rule addition → Engine | Unauthorized rule addition could change behavior |
| Violation log → Disk | Unbounded growth of violations file |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-22 | Tampering | GovernanceEngine.loadRules | mitigate | Validate all rules on load: required fields (id, condition, action, scope); action must be "warn"|"escalate"|"block"; reject rules with missing fields |
| T-02-23 | Denial of Service | Block rule proliferation | mitigate | Max 100 rules in store; adding beyond limit throws `[Harness] Governance rule limit exceeded` |
| T-02-24 | Repudiation | Violation log integrity | mitigate | Append-only writes to violations.json; each violation has unique ID (UUID or timestamp-based); no overwrite of existing entries |
| T-02-25 | Elevation of Privilege | Runtime rule addition | mitigate | addRule() validates rule shape before adding; no execution of rule conditions (declarative only, not code) |
</threat_model>
