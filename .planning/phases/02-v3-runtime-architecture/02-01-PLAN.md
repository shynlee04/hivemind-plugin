     defaultTimeoutMs: number;

   - Add `config: ConcurrencyConfig` as constructor parameter (with sensible defaults: defaultLimit=3, defaultTimeoutMs=30000)

   - Timeout rejects: acquire with 50ms timeout on key with limit=1 and one active holder that never releases, verify rejection with error message
   - Per-key limits: configure key "exclusive" with limit=1, verify only 1 active at a time
