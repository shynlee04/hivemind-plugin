   ```typescript
   export type TaskClassification = {
     category?: DelegationCategory;
     matchedTriggers: string[];
     confidence: number;  // 0-1, based on trigger match ratio
   };
   export type RouteResult = DelegationRouteResolution;  // reuse existing type
   ```

2. Create `src/lib/specialist-router.ts`:
   - `SpecialistRouter` class with constructor(presets: AgentPreset[], generalistAgent: SpecialistAgent = "builder")
   - `classifyTask(description: string, requestedCategory?: DelegationCategory): TaskClassification`:
     - Tokenize description to lowercase words
     - For each preset, count matching triggers in description
     - Category-based routing: if requestedCategory provided, map to preset by domain match
     - Confidence = matchedTriggers.length / max(1, total triggers checked)
   - `route(description: string, requestedCategory?: DelegationCategory, requestedAgent?: SpecialistAgent): RouteResult`:
     - If requestedAgent is valid (VALID_AGENTS), use it directly (agentSource: "explicit")
     - Else classify task → find best-matching preset by trigger count
     - If no preset matches (confidence < 0.1), fall back to generalist (builder) (D-07)
     - Set temperature from preset, model from category mapping
     - Populate warnings if confidence is low
   - `getGeneralistAgent(): SpecialistAgent` — returns configured generalist

3. Wire into `src/plugin.ts` delegate-task tool:
   - Replace existing inline agent/category routing logic with `SpecialistRouter.route()`
   - Pass result to lifecycleManager as `DelegationRouteResolution`
   - Record specialist assignment in delegation packet metadata (Plan 04)
