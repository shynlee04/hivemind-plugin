<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| delegated task description → execution classifier | Untrusted task text influences execution-family choice |
| runtime policy → owned-process spawn | Policy and task metadata decide whether a system process is launched |
| background command request → OS process | Potentially dangerous command/cwd values cross into child process execution |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Spoofing | `src/lib/execution-mode.ts` | mitigate | Record classifier rationale and chosen execution path in returned metadata for parent verification |
| T-02-05 | Elevation of Privilege | `src/lib/background-manager.ts` | mitigate | Enforce command allowlist + project-root/workspace cwd policy before spawn |
| T-02-06 | Denial of Service | `src/tools/delegate-task.ts` | mitigate | Reuse queue limits/timeouts from Plan 01 and reject unavailable tmux requests with deterministic fallback instead of hanging |
| T-02-07 | Repudiation | `src/lib/lifecycle-manager.ts` | mitigate | Persist execution-family/submode + failure context before cleanup so parents can audit what happened |
</threat_model>
