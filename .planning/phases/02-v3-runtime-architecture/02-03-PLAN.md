1. Add types to `src/lib/types.ts` (D-02, D-13):
   ```typescript
   export type ExecutionMode = "builtin-subsession" | "builtin-process";
   export type AgentResult = {
     sessionID: string;
     status: "completed" | "failed" | "timeout";
     output?: string;
     error?: string;
     durationMs: number;
     executionMode: ExecutionMode;
   };
   export type BackgroundAgentOptions = {
     sessionID: string;
     description: string;
     agent: SpecialistAgent;
     promptText: string;
     queueKey: string;
     timeoutMs?: number;
   };
   ```

2. Create `src/lib/background-agent.ts`:
   - `BackgroundAgentRunner` class takes OpenCodeClient + DelegationConcurrencyQueue
   - `async execute(opts: BackgroundAgentOptions): Promise<AgentResult>`:
     a. Acquire queue lane (use concurrency.ts from Plan 01)
     b. Create child session via `client.session.create()` with parentID
     c. Send prompt via `client.session.prompt()` with agent config
     d. Watch completion via `CompletionDetector.watch()`
     e. On completion: capture result, release queue, return AgentResult
     f. On error: capture error, release queue, write error context to `.hivemind/delegation/` (D-14 prep)
   - `getAgentStatus(sessionID: string): AgentResult | undefined` — query current status
   - Execution mode: use "builtin-subsession" (D-13, tmux unavailable per D-02). Mode detection stubbed for future "builtin-process" addition.
   - All errors prefixed `[Harness]`

3. Tests in `tests/lib/background-agent.test.ts`:
   - Mock client + queue → verify session create → prompt → completion flow
   - Error case: session creation fails → error result with context
   - Timeout case: completion detector times out → timeout result
   - Status query: getAgentStatus returns current result

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Parent session → Background agent | Parent could pass malicious prompt text or invalid agent names |
| Background agent → Delegation directory | Failed agent writes to .hivemind/delegation/ — could overwrite existing files |
| Queue → Background agent | Queue exhaustion could prevent background agent from starting |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Injection | BackgroundAgentRunner.execute | mitigate | Validate agent name against VALID_AGENTS; sanitize promptText via existing buildPromptText helper; Zod validation via tool.schema |
| T-02-08 | Tampering | Error context write to .hivemind/delegation/ | mitigate | Use sessionID-unique filename (`${sessionID}.json`); use `writeFileSync` with `{ flag: 'wx' }` to prevent overwrites |
| T-02-09 | Denial of Service | Queue acquisition | mitigate | Timeout on acquire (Plan 01); background agent fails gracefully if queue unavailable after timeout |
| T-02-10 | Spoofing | Parent session query | mitigate | Only parent session (via parentSessionID in continuity) can query child status; verify lineage via session-api.ts walkParentChain |
</threat_model>
