## Warnings

### WR-01: Persisted delegation status is accepted without validation

**File:** `src/lib/delegation-persistence.ts:47-90`

**Issue:** `normalizePersistedDelegation()` validates that `record.status` is a string but then casts arbitrary content to `Delegation["status"]`. A corrupted or manually edited `delegations.json` record with a status outside `dispatched | running | completed | error | timeout` can enter memory and be exposed through `delegation-status`, while recovery and terminal logic only understand the known status set.

**Fix:** Validate persisted status against the `DelegationStatus` union before returning a normalized record.

```typescript
const VALID_DELEGATION_STATUSES = new Set<Delegation["status"]>([
  "dispatched",
  "running",
  "completed",
  "error",
  "timeout",
])

if (!VALID_DELEGATION_STATUSES.has(record.status as Delegation["status"])) {
  return null
}
```
