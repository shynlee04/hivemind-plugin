    export type SplitDirection = "h" | "v"

      direction: SplitDirection

    - Method: `computeSplitSequence(root: PaneTreeNode): SplitCommand[]`

      - For each non-root node, emit a SplitCommand targeting its parent
      - Direction: emit "h" for depth-1 nodes (level 1, direct children of root); emit "v" for depth-2+ nodes

      - When the timer fires: invoke `onComputed(computeSplitSequence(storedRoot))`
      - Implementation uses `setTimeout` / `clearTimeout` — do NOT add an external dependency
      - Expose a `cancel()` method that clears any pending timer (useful for tests and shutdown)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Co-pilot tool → fork sendKeys/listPanes | Untrusted agent prompt text crosses into tmux CLI argv via execFile |
| PaneTreeNode → tmux split-window | User-defined tree shape crosses into split sequence generator |
| Debounce timer → in-process state | Trailing-edge timer mutates planner state; no cross-process impact |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-01 | Tampering | sendKeys argv construction | mitigate | P42 quoteShellArg wraps all argv; literal flag (-l) prevents tmux from interpreting special keys; await only confirms acceptance not execution |

| T-43-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages; uses node:child_process, node:fs, node:path (built-ins only); bun test for verification |
