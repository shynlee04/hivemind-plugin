    ```

      onSessionCreated: (event: import("./observers.js").EnrichedSessionEvent) => Promise<void>
      respawnIfKnown: (sessionId: string) => Promise<{ paneId: string } | null>
      getMainPaneId: () => string | undefined

      // held by the fork SessionManager. The tool (Task 2) accesses them through
      // a small adapter stored alongside the manager. See "Adapter" below.
    }
    ```
    The structural type intentionally matches the P42 fork `SessionManager` interface so any compatible implementation can be injected without depending on the fork package directly.

    **Adapter (returned alongside the manager):**
    The tool needs both the manager (for `onSessionCreated` / `respawnIfKnown`) and direct access to `sendKeys` / `listPanes` / `PaneGridPlanner`. Add:
    ```
    export type ForkSessionManagerAdapter = ForkSessionManager & {
      sendKeys: (paneId: string, text: string, literal?: boolean) => Promise<void>
      listPanes: (mainPaneId?: string) => Promise<import("@hivemind/opencode-tmux").PaneState[]>
      createPaneGridPlanner: () => import("@hivemind/opencode-tmux").PaneGridPlanner
    }
    ```
    Note: this DOES import from `@hivemind/opencode-tmux` for the TYPE only (PaneState, PaneGridPlanner). At runtime, the imported type is erased — no cross-package coupling at runtime. The fork package is consumed at runtime through the adapter instance, never via a direct Hivemind-side import statement.

    **Singleton state (module-private):**
    ```
    let adapter: ForkSessionManagerAdapter | null = null
    ```

    **Functions (exported):**
    - `setForkSessionManager(a: ForkSessionManagerAdapter | null): void` — replaces the current adapter; passing `null` clears it (used on shutdown and during HMR-style hot reloads)
    - `getForkSessionManager(): ForkSessionManagerAdapter | null` — returns the current adapter or null if not yet wired

    **Design notes:**
    - Per the runtime-injection pattern (per ASSUMPTIONS.md), Hivemind NEVER imports the fork package at runtime from a stable import path. The adapter is opaque to the rest of Hivemind; only the tool (Task 2) and the plugin wiring (Task 3) touch it.
    - Module-private state means the bridge is process-local — HMR could leak state across reloads; for v1 this is acceptable since the plugin always re-sets the adapter at startup.
    - Do NOT add a logger or telemetry in v1 — the tool will report its own status.
