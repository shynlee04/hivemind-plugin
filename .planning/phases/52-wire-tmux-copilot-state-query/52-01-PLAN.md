### Task 2: Create `src/tools/tmux-state-query.ts`

**Type:** execute

<read_first>
- src/tools/tmux-copilot.ts (tool pattern, Zod schema, permission gate, renderToolResult)
- src/features/tmux/types.ts (SessionManagerAdapter, getSessionManagerAdapter)
- src/shared/tool-helpers.js (renderToolResult)
- src/features/tmux/session-manager.ts (TrackedSession, sessions map)
</read_first>

<action>

Create a new read-only tool `src/tools/tmux-state-query.ts` that exposes session metadata for the observability layer:

1. **Schema:** Define a discriminated union with 3 actions:
   - `list-sessions` — returns all tracked session metadata
   - `get-session` — returns metadata for one session by `sessionId`
   - `get-summary` — returns aggregate counts (active, spawning, total)

2. **Permission gate:** Same orchestrator-tier check as `tmux-copilot.ts` — import `ORCHESTRATOR_AGENT_NAMES` or duplicate the pattern

3. **Result type** (`TmuxStateQueryResult`):
   ```typescript
   | { error: { kind: "invalid-input"; issues: z.ZodIssue[] } }
   | { error: { kind: "permission-denied"; agent: string } }
   | { available: false; reason: "tmux-not-wired" }
   | { sessions: SessionSummary[] }
   | { session: SessionSummary | null }
   | { summary: { total: number; active: number; spawning: number } }
   ```

4. **Implementation:** 
   - Bridge check via `getSessionManagerAdapter()` — return `{available: false, reason: "tmux-not-wired"}` when null
   - For `list-sessions` / `get-session`, expose metadata from the SessionManager's internal tracking
   - For `get-summary`, compute counts from the available tracking
   - This tool is READ-ONLY — no mutation of any state

5. **Export:** `export const tmuxStateQueryTool: ReturnType<typeof tool>`

6. **Registration:** Export both `tmuxStateQueryTool` and a `tmuxStateQueryToolName` constant

</action>

<acceptance_criteria>
- `src/tools/tmux-state-query.ts` exists with 3 actions (list-sessions, get-session, get-summary)
- Tool uses same permission gate pattern as tmux-copilot.ts
- Tool returns `{available: false, reason: "tmux-not-wired"}` when adapter is null
- tsc exits 0
</acceptance_criteria>

1. Add event subscription types:
   ```typescript
   export type TmuxEventType = "session.created" | "session-state-changed" | "pane-captured"
   
   export interface SessionStateChangedEvent {
     type: "session-state-changed";
     sessionId: string;
     previousState: string;
     currentState: string;
     timestamp: number;
   }
   
   export interface PaneCapturedEvent {
     type: "pane-captured";
     sessionId: string;
     paneId: string;
     contentLength: number;
     timestamp: number;
   }
   ```

2. Add listener registry pattern to `createTmuxEventObserver`:

   - Private storage for registered callbacks (arrays)
   - Return an extended observer object with both the original `(input: { event?: unknown }) => Promise<void>` function AND the new registration methods

3. Update `ForkSessionManager` interface to accept the expanded observer shape if needed

4. The existing `session.created` forwarding to ForkSessionManager must remain unchanged

</action>
