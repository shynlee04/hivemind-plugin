## Changes Made

### Task 1: Message history fallback (REQ-23.2-02, Bug #2 MEDIUM)

Modified `handleSessionCompacted()` to add message history lookup when event payload has no summary:

```typescript
const eventSummary = this.findCompactionText(event)
if (eventSummary) {
  // Event payload has summary text — use it directly
  compactContext = `**compact_summary:**\n\n${eventSummary}\n`
} else {
  // Event payload has no summary (metadata-only event) — read message history
  compactContext = await this.resolveCompactionFromMessages(sessionID)
}
```

Added new helper method `resolveCompactionFromMessages()`:
- Calls `getSessionMessages(this.client, sessionID)` to fetch message history
- Finds last assistant message (typically contains compact summary)
- Uses `extractTextFromSdkMessage()` to extract text content
- Falls back to last message of any role if no assistant message
- Returns fallback message "**Compaction occurred — summary unavailable.**" if all attempts fail

**Why it works:** The `session.compacted` event from OpenCode SDK contains only metadata (sessionID, timestamp), not actual compact summary text. By reading message history, we get the readable summary that was generated during compaction.
