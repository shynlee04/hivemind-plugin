## Changes Made

### Task 1: getByParent() accessor (REQ-23.2-03, Bug #3 MEDIUM)

Added `getByParent(parentSessionID: string)` method to PendingDispatchRegistry class. The method:
- Iterates the `byParent` Map (parentSessionID → Set<callID>)
- Resolves callIDs to entries using `dispatches.get("call:${callID}")`
- Checks stale threshold (STALE_THRESHOLD_MS = 30s)
- Returns first non-stale match, or undefined

**Why it works:** When `session.created` fires before `updateWithChildID()` has mapped callID to child session ID, `get(sessionID)` returns undefined. The parent-based fallback finds the pending dispatch via the byParent index.
