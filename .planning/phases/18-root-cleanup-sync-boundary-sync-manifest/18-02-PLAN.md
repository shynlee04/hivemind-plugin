**Extract the storeCache singleton from continuity/index.ts into a dedicated store-cache.ts module with explicit get/set/reset API.**

Purpose: Fix test isolation fragility caused by module-level singleton state. Currently masked by `vi.resetModules()` boilerplate. The new `resetStoreCache()` API provides a clean way to reset cache between tests.

- `tests/lib/continuity.test.ts` — updated to use resetStoreCache() where it replaces vi.resetModules()
</objective>

    ### store-cache.ts (new)
    
    ```typescript
    import type { ContinuityStoreFile } from "../../shared/types.js"
    ```
