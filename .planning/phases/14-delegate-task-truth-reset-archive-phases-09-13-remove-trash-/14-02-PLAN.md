STEP 2 — Create `src/lib/delegation-manager.ts` with these exact exports:

The class `DelegationManager` must have:
1. **Constructor** — takes `client: OpenCodeClient` (from `@opencode-ai/sdk`). Store as `this.client`.
2. **In-memory Maps:**
   - `delegations: Map<string, Delegation>` — delegationId → Delegation
   - `delegationsBySession: Map<string, string>` — childSessionId → delegationId
   - `timeoutTimers: Map<string, NodeJS.Timeout>` — delegationId → timer
   - `completionCallbacks: Map<string, { resolve: (result: DelegationResult) => void; reject: (error: Error) => void }>` — delegationId → Promise callbacks
3. **`async delegateSync(params: { parentSessionId: string; agent: string; prompt: string; title?: string; timeoutMs?: number }): Promise<DelegationResult>`**
   - Validate agent is in VALID_AGENTS (throw `[Harness] Invalid agent: ${agent}` if not)
   - Create child session via `this.client.session.create({ body: { title: params.title ?? 'Delegation: ' + params.agent, parentID: params.parentSessionId } })`
   - Register Delegation in Maps with status `'running'`
   - Schedule timeout with `setTimeout(() => this.handleTimeout(delegation.id), timeoutMs)`
   - Send prompt via `this.client.session.prompt({ path: { id: child.id }, body: { parts: [{ type: 'text', text: params.prompt }], agent: params.agent } })`
   - Return Promise that resolves when handleSessionIdle fires for this child session
4. **`handleSessionIdle(sessionId: string): void`**
   - Look up delegationId from delegationsBySession Map
   - If not found, return (not a delegation session)
   - If delegation.status !== 'running', return (already handled)
   - Clear timeout timer
   - Call `this.finalizeDelegation(delegationId)` asynchronously
5. **`handleSessionDeleted(sessionId: string): void`**
   - Look up delegationId from delegationsBySession Map
   - If found, clear timer, update status to 'error', clean Maps
6. **`private async finalizeDelegation(delegationId: string): Promise<void>`**
   - Get messages from child session via `this.client.session.messages()`
   - Extract assistant text: filter messages where `role === 'assistant'`, flatMap parts where `type === 'text'`, join with newline
   - Update delegation status to 'completed', set result and completedAt
   - Resolve completion callback if waiting
   - Clean Maps (but keep delegation in delegations Map for status queries)
7. **`private async handleTimeout(delegationId: string): Promise<void>`**
   - Update delegation status to 'timeout'
   - Abort child session via `this.client.session.abort()`
   - Reject completion callback with timeout error
   - Clean Maps

CRITICAL design rules per RESEARCH.md pitfalls:
- NEVER mark delegation as completed before session.idle fires (Pitfall 1: phantom completion)
- ALWAYS abort child session on timeout (Pitfall 4: zombie sessions)
- ALWAYS clear timeout timer when handling completion (prevent double-handling)
- Agent validation against VALID_AGENTS whitelist (security: T-14-02)
- Max module size: 500 LOC. If approaching limit, extract finalizeDelegation into a helper.

Import from the project's own modules:
- `import { VALID_AGENTS, type Delegation, type DelegationResult, type DelegationStatus, DEFAULT_DELEGATION_TIMEOUT_MS } from "./types.js"`
- `import type { Client as OpenCodeClient } from "@opencode-ai/sdk"`
- DO NOT import from deleted modules (governance-engine, injection-engine, etc.)

   - Schedule timeout
   - Send prompt
   - Return `{ delegationId: delegation.id }` immediately (do NOT await completion)
   - Completion will be handled asynchronously by handleSessionIdle → finalizeDelegation → notifyParent

2. **`private async persistDelegation(delegation: Delegation): Promise<void>`**
   - Write delegations Map values to a JSON file at the harness state directory
   - Use pattern: `const stateDir = path.join(process.cwd(), '.opencode', 'state', 'opencode-harness')`

3. **`async recoverPending(): Promise<void>`**
