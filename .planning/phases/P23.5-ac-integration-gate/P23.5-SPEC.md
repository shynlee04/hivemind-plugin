# P23.5 — A→C Cross-Cluster Integration Gate — SPEC

**Created:** 2026-05-29
**Source research:** `../../research/P23-5-ac-integration-research-2026-05-29.md`
**Domain:** Agent→Command Workflow integration audit

## Overview

Phase 23.5 closes 3 integration gaps between the Agent cluster (`.opencode/agents/`) and the Commands cluster (`.opencode/commands/` + `.opencode/command/`), with the Workflow cluster (`.opencode/workflows/`) serving as the bridge layer.

**Constraint:** Do NOT modify agent files. Only fix command files and workflow documentation.

---

## Requirements

### REQ-01: 5 Orphan Agents Receive Command Routing

**Problem:** 5 agents exist as files but have ZERO command routing AND ZERO workflow dispatch — they are undiscoverable dead code.

| Agent | Purpose |
|-------|---------|
| `hm-architect` | Technical architecture design and ADR production |
| `hm-ecologist` | Feature dependency and ecosystem mapping |
| `hm-intel-updater` | Codebase intelligence maintenance (`.planning/intel/`) |
| `hm-l0-orchestrator` | Front-facing L0 strategist for hm-* lineage |
| `hm-shipper` | Release coordination and changelog management |

**Acceptance:** Each agent has a corresponding command file in both `.opencode/commands/` and `.opencode/command/` directories, with YAML `agent:` field routing to the correct agent file.

### REQ-02: hm-specifier Receives Accurate Command Routing

**Problem:** `hm-plan-phase` command description claims routing through `hm-specifier` but the YAML `agent:` field routes to `hm-planner` only. No command dispatches `hm-specifier`.

**Acceptance (A):** The `hm-plan-phase` description is corrected to remove the misleading `hm-specifier` claim — aligning documentation with actual YAML routing.

**Acceptance (B):** A dedicated `hm-specifier` command exists in both `commands/` and `command/` directories, routing to the `hm-specifier` agent, so it CAN be dispatched when needed.

### REQ-03: hm-synthesizer Workflow Documentation Aligned with Reality

**Problem:** `hm-synthesize` command `<context>` block claims "Routed Agent: hm-synthesizer" but YAML `agent:` field routes to `hm-orchestrator`. The `hm-synthesize` workflow's Agent Routing Table lists `hm-synthesizer` but no actual `subagent_type` spawn exists.

**Acceptance (A):** The `hm-synthesize` command's `<context>` block is updated to remove the misleading "Routed Agent: hm-synthesizer" claim, aligning with actual YAML routing to `hm-orchestrator`.

**Acceptance (B):** The `hm-synthesize` workflow's Agent Routing Table is updated to reflect the actual routing path (routed via `hm-orchestrator`).

### REQ-04: Cross-Directory Sync Maintained

**Problem:** `.opencode/commands/` and `.opencode/command/` must remain synchronized per project rules.

**Acceptance:** All command files created or modified in `.opencode/commands/` have identical mirror copies in `.opencode/command/`.

### REQ-05: No Regressions

**Acceptance:** `npm run typecheck` passes. `npm test` passes with no regressions from baseline.

## Verification Commands

```bash
# 1. Verify all 6 new command→agent mappings resolve
for agent in hm-architect hm-ecologist hm-intel-updater hm-l0-orchestrator hm-shipper hm-specifier; do
  if [ -f ".opencode/agents/$agent.md" ]; then echo "✓ Agent $agent exists"; else echo "✗ Agent $agent MISSING"; fi
  if [ -f ".opencode/commands/$agent.md" ]; then echo "✓ commands/$agent.md"; else echo "✗ commands/$agent.md MISSING"; fi
  if [ -f ".opencode/command/$agent.md" ]; then echo "✓ command/$agent.md"; else echo "✗ command/$agent.md MISSING"; fi
done

# 2. Verify no broken command→agent links
for agent in $(grep -rh '^agent:' .opencode/commands/hm-*.md | sort -u | sed 's/agent: //'); do
  if [ -f ".opencode/agents/$agent.md" ]; then echo "✓ $agent"; else echo "✗ $agent MISSING"; fi
done

# 3. Verify commands/ and command/ directories are in sync
diff <(ls .opencode/commands/hm-*.md | xargs -n1 basename | sort) \
     <(ls .opencode/command/hm-*.md | xargs -n1 basename | sort)

# 4. Verify the 6 new command files exist in BOTH directories
for cmd in hm-architect hm-ecologist hm-intel-updater hm-l0-orchestrate hm-shipper hm-specifier; do
  diff ".opencode/commands/${cmd}.md" ".opencode/command/${cmd}.md" > /dev/null 2>&1 \
    && echo "✓ ${cmd}: synced" \
    || echo "✗ ${cmd}: DRIFT"
done

# 5. Verify hm-plan-phase description no longer mentions hm-specifier
! grep -q "hm-specifier" .opencode/commands/hm-plan-phase.md && echo "✓ hm-plan-phase description clean"
! grep -q "hm-specifier" .opencode/command/hm-plan-phase.md && echo "✓ hm-plan-phase (command/) description clean"

# 6. Verify hm-synthesize context no longer claims "Routed Agent: hm-synthesizer"
! grep -q "Routed Agent: hm-synthesizer" .opencode/commands/hm-synthesize.md && echo "✓ hm-synthesize context clean"
! grep -q "Routed Agent: hm-synthesizer" .opencode/command/hm-synthesize.md && echo "✓ hm-synthesize (command/) context clean"

# 7. Verify hm-synthesize workflow table updated
! grep -q "hm-synthesizer" .opencode/workflows/hm-synthesize.md && echo "✓ workflow table clean" \
  || grep "hm-synthesizer" .opencode/workflows/hm-synthesize.md | head -5

# 8. Regression baseline
npm run typecheck && echo "✓ typecheck clean"
npm test 2>&1 | tail -5
```
