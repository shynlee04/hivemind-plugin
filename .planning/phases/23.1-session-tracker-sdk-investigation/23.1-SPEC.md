# Phase 23.1: Session-Tracker SDK Dispatch Investigation — Specification

**Created:** 2026-05-24
**Ambiguity score:** 0.08 (gate: ≤ 0.20)
**Requirements:** 3 locked

## Goal

Fix race condition in `sessionTracker.initialize()` where fire-and-forget `void` call causes `onChildSessionCreated` callbacks from delegate-task dispatch to be silently dropped before critical dependencies are assigned — permanently resolving U4/U5 bugs that failed 4 rounds of live UAT.

## Background

Phase 23 notification fixes have been live-tested 4 rounds. U4 (session-tracker cannot find delegate-task child sessions) and U5 (session-tracker hierarchy broken — childCount:0, depth:0) failed ALL 4 rounds.

Investigation report at `.planning/phases/23.1-session-tracker-sdk-investigation/23.1-SESSION-TRACKER-INVESTIGATION-2026-05-24.md` (316 lines) confirmed the root cause:

- `plugin.ts:328` calls `void sessionTracker.initialize()` — fire-and-forget
- `initialize()` is `async` — it constructs all critical deps (`eventCapture`, `pendingRegistry`, `childWriter`, `hierarchyIndex`) INSIDE the async method via `constructDependencies()` + `Object.assign(this, deps)`
- Any `onChildSessionCreated` callback fired before `Object.assign` completes hits `handleSessionEvent()` guard at `index.ts:140` (`if (this.eventCapture)`) — short-circuits, silently drops the event
- The timing window is deterministic and fires every time a `delegate-task` dispatch happens in the first seconds after plugin load

## Requirements

1. **Primary fix: Sync dependency construction before delegation wiring**: Move sync `constructDependencies()` (or a sync subset of critical deps) before `setupDelegationModules()` in `plugin.ts` — ensures `eventCapture` and `pendingRegistry` are available before `onChildSessionCreated` can fire.
   - Current: `plugin.ts:280` — `sessionTracker = new SessionTracker(...)`, then `plugin.ts:282` `setupDelegationModules()` captures ref, then `plugin.ts:328` `void sessionTracker.initialize()` — deps assigned ASYNC inside initialize
   - Target: Critical deps (`eventCapture`, `pendingRegistry`, `childWriter`, `hierarchyIndex`) are available SYNCHRONOUSLY before `setupDelegationModules()` runs
   - Acceptance: Calling `onChildSessionCreated(sessionID, parentID)` immediately after `new SessionTracker()` writes child `.json` instead of silently dropping

2. **Defensive logging for skipped events**: Add warning log when `handleSessionEvent` skips due to uninitialized deps at `index.ts:140`.
   - Current: Silent skip — no diagnostic signal
   - Target: `client.app?.log` warning with `[Harness] Session tracker: handleSessionEvent skipped — deps not initialized (sessionID, eventType)`
   - Acceptance: Log appears when event arrives before init complete

3. **Cleanup fix**: Remove `.then(() => sessionTracker.cleanup())` chain from plugin.ts — keeps retry queue alive for plugin lifetime.
   - Current: `void sessionTracker.initialize().then(() => sessionTracker.cleanup())` — cleanup kills retry queue prematurely
   - Target: `void sessionTracker.initialize()` — no auto-cleanup, retry queue lives for plugin lifetime
   - Acceptance: Retry queue does not die after initialize completes
