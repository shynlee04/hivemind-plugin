## Goal

Fix 6 critical flaws in `execute-slash-command` tool: add return envelope consistency, Zod schema validation, commandSource tracking, execution tracking, delegation-aware context, and typed error classes.

---

## Background

`execute-slash-command.ts` is the CQRS write-side tool for dispatching OpenCode slash commands. It supports 3 dispatch paths: (1) synthetic parent prompt (subtask:false + agent), (2) subtask delegation (subtask:true + agent), (3) TUI pipeline (no overrides).

Current implementation has 6 critical flaws identified in deep-analysis-tools-2026-05-21.md:
1. **No return envelope consistency** — returns raw `{ output, metadata }` instead of `renderToolResult()`
2. **No Zod schema** — relies solely on loose `tool.schema` validation
3. **No commandSource tracking** — cannot distinguish user-appended vs agent-appended commands
4. **No execution tracking** — no status tracking for dispatched commands
5. **No delegation-aware context** — `parentSessionID` not attached to delegation packets
6. **No typed errors** — uses generic errors with string type classification

These flaws cause agent hallucination (cannot tell command origin), unreliable execution status, and broken delegation context.

---

## Requirements

1. **Return envelope consistency**: All tool results use OpenCode SDK's `ToolResponse` shape: `{ output: string; metadata?: Record<string, unknown>; error?: boolean }`.
    - Current: Returns raw `{ output, metadata }` object — inconsistent shape (some paths have `error: true`, some don't; some have `metadata`, some don't)
    - Target: All paths return `{ output, metadata, error }` — consistent shape across all return paths
    - Note: `renderToolResult()` does NOT exist in codebase. PATTERNS.md and earlier SPEC drafts incorrectly referenced it. All 23 other tools in `src/tools/` return raw `{ output, metadata }` shape directly.
    - Acceptance: Every return statement in execute-slash-command.ts returns `{ output, metadata, error }` shape consistently

2. **Zod schema validation**: Add `ExecuteSlashCommandSchema` with validation for command name, arguments, agent, model, subtask, commandSource, trackExecution, parentSessionID.
   - Current: No Zod schema, only loose `tool.schema` validation
   - Target: Import `const ExecuteSlashCommandSchema = z.object({...})` from `schema-kernel/commands.schema.ts`
   - Acceptance: Schema validates command format (lowercase alphanumeric with hyphens), arguments length (max 10000), agent/model format

3. **CommandSource tracking**: Add `commandSource` field ('user' | 'agent' | 'system') to track command origin and attach to metadata and delegation packets.
   - Current: No commandSource field anywhere
   - Target: Default 'user' if not specified; agent-appended commands set 'agent'; system-initiated set 'system'
   - Acceptance: Metadata includes `commandSource: 'user' | 'agent' | 'system'`; delegation packet includes `commandSource` field

4. **Execution tracking**: Add `trackExecution` field (boolean, default true) and track via sessionTracker.pendingRegistry.
    - Current: No execution tracking mechanism
    - Target: When `trackExecution: true`, write execution record to `sessionTracker.pendingRegistry` with parentSessionID, subagentType, tool name
    - Acceptance: Metadata includes `trackExecution: boolean`; execution artifacts created via pendingRegistry when enabled
    - Note: File artifacts (.hivemind/commands/*.json) deferred — pendingRegistry sufficient for execution tracking

5. **Delegation-aware context**: Attach `parentSessionID` to all delegation packets (subtask mode) and read it in sub session for context restoration.
   - Current: No parentSessionID tracking in delegation packets
   - Target: When `parentSessionID` provided, include in delegation packet; sub session reads it on initialization
   - Acceptance: Delegation packet includes `parentSessionID` field; sub session context includes `parentSessionID`

6. **Typed error classes**: Replace generic errors with typed error classes: `CommandNotFoundError`, `AgentNotFoundError`, `DelegationTimeoutError`, `InvalidCommandError`, `DelegationContextError`.
   - Current: Generic errors with `errorType: "not_found" | "bad_request" | "internal"` string
   - Target: Export typed error classes from `src/shared/errors/commands.ts`
   - Acceptance: 5 error classes exported from dedicated file; each extends `Error` with name property

7. **Move session tools**: Relocate `session-context.ts`, `session-hierarchy.ts`, `session-tracker.ts` from `src/tools/hivemind/` to `src/tools/session/`.
    - Current: 3 session tools mislocated in `hivemind/` directory
    - Target: Move files, update imports in `plugin.ts`, update barrel exports
    - Acceptance: All 3 files exist in `src/tools/session/`; `plugin.ts` imports updated; no broken imports

8. **Unified tool import**: Change all tools to use `@opencode-ai/plugin/tool` narrow import instead of `@opencode-ai/plugin`.
    - Current: 5 tools (4 config + execute-slash-command) use wide import
    - Target: All 24 tools use narrow `/tool` import
    - Acceptance: `grep -r "@opencode-ai/plugin" src/tools/` returns 0 matches (except deep imports like features)

> **Deferred to P24.3.3** (namespace routing — no pain signal at ~90 commands, hardcoded enum cannot cover end-user variety):
> - REQ-9: Namespace frontmatter field (`gsd | hf | test | core`)
> - REQ-10: `resolveCommandNamespace()` read-side method on command-engine
> - REQ-11: Contract validation for gsd-* commands
> - REQ-12: Module extraction (`resolve-command.ts`, `dispatch-command.ts`)

9. **Metadata enrichment**: Add `agent`, `model`, `mode`, `commandSource`, `trackExecution`, `parentSessionID` to success metadata envelope.
   - Current: Metadata only has `command`, `agent`, `model` in some paths
   - Target: All success paths include complete metadata shape from requirement #1
   - Acceptance: All success metadata has consistent shape with all required fields

10. **Error metadata enhancement**: Add `field`, `cause` to error metadata for better debugging.
    - Current: Error metadata has `errorType`, `command`, `message`
    - Target: Error metadata includes `field?: string`, `cause?: unknown`
    - Acceptance: Error metadata includes `field` and `cause` fields

11. **Schema module creation**: Create `src/schema-kernel/commands.schema.ts` with `ExecuteSlashCommandSchema` and related types.
    - Current: No dedicated schema file for commands
    - Target: New file exports schema and types
    - Acceptance: File exists at correct path; exports `ExecuteSlashCommandSchema`

12. **Error module creation**: Create `src/shared/errors/commands.ts` with 5 typed error classes.
    - Current: No dedicated error file for commands
    - Target: New file exports all error classes
    - Acceptance: File exists at correct path; all 5 error classes exported

## Boundaries

**In scope:**
- Fix 6 critical flaws in `execute-slash-command.ts` (inline refactor, no module extraction)
- Add Zod schema with validation rules
- Add typed error classes (5 classes)
- Add commandSource tracking (field + metadata + artifact)
- Add execution tracking (field + status + artifact)
- Add delegation-aware context (parentSessionID in packets)
- Move 3 session tools from `hivemind/` to `session/`
- Update all imports to use narrow `/tool` import
- Create dedicated schema and error modules
- Update 15+ unit tests to cover new functionality

**Deferred to P24.3.3 (namespace routing):**
- Namespace routing via frontmatter `namespace:` field — deferred; no hardcoded enum values can cover end-user command variety
- Module extraction (`resolve-command.ts`, `dispatch-command.ts`) — deferred; inline refactor sufficient for ~350 LOC tool
- Contract validation for gsd-* commands — deferred; no pain signal at ~90 commands
- `resolveCommandNamespace()` read-side method — deferred; orthogonal to 6 critical flaws

**Out of scope:**
- Natural language intent parsing (P24.3.3)
- Fuzzy discovery and caching (P24.3.3)
- Dynamic agent/subtask resolver (P24.3.3)
- Full agent switch + restore (P24.3.3)
- Full sub session context implementation (P24.3.3)
- Command auto-append via natural language (P24.3.3)
- Shell command integration with slash commands (P24.3.3)
- Workflow file parsing in slash commands (P24.3.3)
