    **Zod schema as single source of truth — passed directly to `tool()` via OpenCode SDK Zod support.**

    ```typescript
    import { z } from "zod"

    export const ExecuteSlashCommandSchema = z.object({
      command: z.string().min(1).max(200).regex(/^[a-z][a-z0-9-]*$/).describe("Command name (lowercase alphanumeric + hyphens)"),
      arguments: z.string().max(10000).optional().default(""),
      agent: z.string().regex(/^[a-z][a-z0-9-]*$/).optional().describe("Agent override name"),
      model: z.string().regex(/^[a-zA-Z0-9]+\/[a-zA-Z0-9._-]+$/).optional().describe("Model override (provider/modelID format)"),
      subtask: z.boolean().optional().default(false),
      commandSource: z.enum(['user', 'agent', 'system']).optional().default('user'),
      trackExecution: z.boolean().optional().default(true),
      parentSessionID: z.string().optional(),
    })

    export type ExecuteSlashCommandInput = z.infer<typeof ExecuteSlashCommandSchema>
    ```

    Follow the pattern from `src/schema-kernel/command-engine.schema.ts` per PATTERNS.md section "Zod schema pattern":
    - Use `z.object()`, `.describe()`, `.optional().default()` patterns
    - Export both the schema (with `typeof`) and the inferred TypeScript type
    - Place in `src/schema-kernel/commands.schema.ts` (not `command-engine.schema.ts` — separate concern, per SPEC REQ-11)
    - Use `verbatimModuleSyntax` compatible exports: `import type` for type-only imports

    Per Code Example from RESEARCH.md section "Namespace Frontmatter Schema Extension":
    - command name validation: `z.string().min(1).max(200)` — spec says lowercase alphanumeric + hyphens
    - arguments length: `z.string().max(10000)` — per SPEC acceptance criteria
    - agent name format: regex `^[a-z][a-z0-9-]*$` for lowercase alphanumeric with hyphens
    - model format: regex `^[a-zA-Z0-9]+\/[a-zA-Z0-9._-]+$` for provider/model pattern
