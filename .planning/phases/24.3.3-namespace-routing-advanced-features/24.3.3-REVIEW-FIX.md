## Fix Summary

### CR-01: Fire-and-forget dispatch returns `{ success: true }` before prompt executes

**File:** `src/tools/session/dispatch-command.ts`

**What was changed:**
- Removed the `setTimeout(fn, 50)` deferral pattern that returned `{ success: true }` immediately
- Now awaits `client.session.prompt(request)` directly
- Returns `{ success: true }` on completion, or `{ success: false, error: true, output: message }` on failure
- Removed the `DEFERRED_SUBTASK_DISPATCH_DELAY_MS` constant
- All 3 call sites in `execute-slash-command.ts` now check the dispatch result and return error envelopes on failure

**Evidence:**
- `dispatchCommand` now correctly propagates errors instead of swallowing them in `.catch()`
- New test `returns error envelope when prompt dispatch fails` verifies error propagation
- Removed fake timers from test (no longer needed)

### CR-02: Two-stage agent selection pipeline not wired into execution flow

**Files:**
- `src/tools/session/execute-slash-command.ts`
- `src/tools/session/semantic-agent-selector.ts`

**What was changed:**
- Added `selectAgent` import from `./semantic-agent-selector.js`
- Added `getAppAgents` import from `../../shared/app-api.js`
- After command resolution succeeds, if no agent is explicitly provided (from args or command frontmatter), calls `selectAgent()` with the command text and available agents
- Safety guard: only calls `selectAgent()` when `client.app?.agents` is available (prevents breakage for mocks without agent registry)

**Evidence:**
- SPEC R5 and R7 two-stage routing is now implemented: `resolveCommand()` ‚Üí `selectAgent()` after command resolution
- The `selectAgent` module is no longer dead code in the execution flow

### CR-03: Pervasive `any` types on all new/modified modules

**Files fixed:**
- `src/tools/session/dispatch-command.ts` ‚Äî `client: any` ‚Üí `client: OpenCodeClient`, `a: any` ‚Üí typed map callback, `parts: any[]` ‚Üí `Array<Record<string, unknown>>`
- `src/tools/session/validate-command.ts` ‚Äî `args: any` ‚Üí `args: Record<string, unknown>`
- `src/tools/session/semantic-agent-selector.ts` ‚Äî `agents: any[]` ‚Üí `Array<{ name: string; id?: string; description?: string }>`, dead string-check code path removed from normalization
- `src/tools/session/execute-slash-command.ts` ‚Äî All `as any` casts replaced with `as ToolResult`, `args: any` ‚Üí `args: ExecuteInput` (typed interface), `validated: any` ‚Üí `validated: Record<string, unknown>`, `sessionTracker?: any` ‚Üí `sessionTracker?: unknown`, `(err as any).name` ‚Üí `getErrorName(err)` helper

**Type definitions added:**
