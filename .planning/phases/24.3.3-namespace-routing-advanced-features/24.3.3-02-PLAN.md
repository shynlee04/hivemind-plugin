  <action>
    **src/command-engine/cache-layer.ts:**
    - `class CommandCache`:
      - `constructor(options?: { ttl?: number; maxEntries?: number })` — defaults: TTL 60000ms, max 100 entries
      - `get(commandName: string): CommandCacheEntry | undefined` — checks existence + TTL
      - `set(commandName: string, entry: CommandCacheEntry): void` — stores with current timestamp
      - `invalidate(commandName?: string): void` — invalidate specific or all (when undefined)
      - `clear(): void` — full invalidation
      - Internal Map<string, CommandCacheEntry> storage
      - LRU eviction when exceeding maxEntries

    - `fuzzyFindCommand(partialName: string, options?: { namespace?: string; cache?: CommandCache }): CommandConfig[]`
      - Scan `.opencode/commands/` directory for .md files
      - Match partial name against filename (case-insensitive substring match)
      - If namespace provided, filter by namespace field
      - Return matches sorted by relevance: exact prefix match > substring match > distant match
      - Target <50ms TTK by using cache on repeated scans
      - When cache hit: return cached results directly
      - When cache miss: scan filesystem, populate cache, return results
      - Limit results to options.limit or 10 default

    **Integration into resolve-command.ts:**
    - `resolveCommand()` accepts cache parameter (optional, default creates singleton)
    - Check cache before filesystem read
    - Cache populated after successful resolution
    - Inline the singleton: `const defaultCache = new CommandCache()` at module level
    - Add `fuzzyFindCommand` import and wire into resolveCommand as fallback when exact name not found

    **Tests (inline in task):**
    - Write tests in tests/tools/execute-slash-command.test.ts covering:
      1. CommandCache.set/get with valid TTL
      2. CommandCache entry expires after TTL
      3. CommandCache.invalidate removes specific entry
      4. fuzzyFindCommand returns matches for partial name
      5. fuzzyFindCommand respects namespace filter
      6. fuzzyFindCommand <50ms TTK (measured with Date.now())
  </action>

  <action>
    **src/command-engine/agent-resolver.ts:**
    - `resolveAgent(commandName: string, options?: DispatchContext): { agent: string; subtask: boolean }`
      - Reads command frontmatter for `agent:` field
      - If `agent:` present in command config: use that agent
      - If `agent:` in DispatchContext override: use that (takes precedence)
      - Default: no agent override
    - `resolveSubtask(commandName: string, options?: DispatchContext): boolean`
      - Reads command frontmatter for `subtask:` field
      - If explicitly set in context override: use that
      - Default: true (subtask:true for safety)

    **Integration in dispatch-command.ts:**
    - Add `dispatchWithAgentSwitch(commandName, args, context)` implementing P21.2 one-shot agent-switch path:
      1. Call resolveAgent() to determine target agent
      2. Call resolveSubtask() to determine subtask mode
      3. Dispatch via SDK session.prompt() with `subtask:false + agent` override
      4. After dispatch completes (result received), restore calling context by re-attaching to parent session
      5. Return delegation result with metadata: { result, agentUsed, restored: true }
    - The one-shot path is: synthetic parent prompt via session.prompt() → agent runs one turn → context restored
    - Must handle errors: if agent switch fails, fall back to current agent dispatch

    **Tests (inline in task):**
    - Write tests in tests/tools/execute-slash-command.test.ts covering:
      1. resolveAgent returns agent from command config
      2. resolveAgent prefers context override over command config
      3. resolveSubtask returns default true
      4. resolveSubtask respects subtask: false from context
      5. dispatchWithAgentSwitch calls resolveAgent
  </action>

  </verify>
