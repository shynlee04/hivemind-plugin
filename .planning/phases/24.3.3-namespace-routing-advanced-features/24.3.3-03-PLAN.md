  <action>
    **src/command-engine/intent-resolver.ts:**
    - `class IntentResolver`:
      - `resolve(text: string): { commandName?: string; arguments?: string; confidence: number }`
        - Checks if text matches a known command name
        - If exact match: return command name directly (confidence 1.0)
        - If partial match via fuzzyFindCommand: return best match (confidence 0.7)
        - If no match: return undefined
        - NL detection: if text does NOT start with known command prefix, check for NL patterns
      - `isNLQuery(text: string): boolean` — heuristic check for natural language vs explicit command
        - Returns true if text is a question, request, or non-command phrase
        - Checks against command prefix patterns

    - `function resolveIntent(text: string, options?: { cache?: CommandCache }): { commandName: string; arguments: string; confidence: number } | undefined`
      - Wrapper around IntentResolver.resolve
      - Uses fuzzyFindCommand for NL→command mapping
      - Returns undefined if no command matches with confidence >= 0.5

    **Integration in dispatch-command.ts:**
    - Add `dispatchWithNLIntent(text, context)`:
      1. Call resolveIntent() to detect command from NL text
      2. If intent resolved: dispatch the resolved command with extracted arguments
      3. If not resolved: return error "command not found"

    **Tests (inline in task):**
    - Write tests in tests/tools/execute-slash-command.test.ts covering:
      1. IntentResolver resolves exact match
      2. IntentResolver resolves partial match via fuzzy
      3. IntentResolver returns undefined for unknown text
      4. isNLQuery distinguishes NL from commands
      5. resolveIntent wrapper returns correct shape
  </action>
