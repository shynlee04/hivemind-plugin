<decisions>
## Implementation Decisions

### Semantic Agent Selection
- **Hybrid matching strategy:** Keyword extraction (fast path) → LLM fallback (if no match)
- **Integration point:** Extend `resolveCommand()` (no new module created)
- **Keyword matching:** Extract entities from user input (e.g., "gsd command" → "gsd") → filter commands/agents by keyword in name/description
- **LLM fallback:** Use `client.llm.generateText()` to parse intent when keyword matching fails
- **OpenCode fields validation:** All OpenCode-based fields must match exact SDK conventions:
  - `agent` field: must match regex `^[a-z][a-z0-9-]*$` (OpenCode agent naming)
  - `subtask` field: must be boolean (true/false), defaults to false
  - Agent existence: must exist in OpenCode agent registry (exact match required)
  - Temporary overrides: When manipulating `agent`/`subtask` fields, validate against actual OpenCode agent list — if agent doesn't exist, fail with exact agent name

### Two-Stage Routing
- **Parallel execution:** Stage 1 (command resolution) and Stage 2 (agent selection) run simultaneously
- **Independent matching:** Command resolved independently, agent selected independently
- **Fallback behavior:**
  - If command not found → ask user to specify exact command name
  - If agent not found → use default agent (current TUI selection)
- **List both first:** List all available commands AND all available agents before matching

### Contract Validation
- **Structural fields (always present):**
  - `name` — command name (required)
  - `description` — always present (required)
  - `triggers` — array of trigger strings (min 1, required)
- **Common OpenCode fields (if present):**
  - `agent` — must match regex `^[a-z][a-z0-9-]*$`
  - `subtask` — must be boolean (true/false)
  - `model` — if present, must be valid model name (string)
- **Temporary override validation:**
  - Validate `agent` against OpenCode agent registry (exact match)
  - If agent doesn't exist → error with exact agent name
  - Validate `subtask` is boolean, default to false if not specified
- **Non-gsd commands:** Lenient mode — always valid, no structural validation

### Namespace Routing
- **Status:** DEFERRED to P24.3.3.2 or later phase
- **Schema field:** `namespace?: string` added (optional, no hardcoded values, no enum)
- **No routing logic:** Namespace field present in schema but no routing/validation enforcement
- **Reason:** User wants to defer namespace routing entirely

### Agent Discovery
- **List agents first:** Before matching, list all available agents from OpenCode agent registry
- **Filter by intent:** Use semantic matching to filter agents based on user intent
- **Default fallback:** If no agent matches → use default agent (from TUI or `commandBundle?.agent`)

### Parallel vs Sequential
- **Parallel routing:** Command and agent resolution happen simultaneously (not sequential)
- **Independent matching:** Each stage can succeed/fail independently
- **Error handling:** If command fails → ask user; if agent fails → use default

</decisions>
