## Implementation Decisions (locked)

### D-01: Hook Point
**Decision:** `ToolDelegation.recordChildTaskDelegation()` at line 227.
**Rationale:** Single integration point covers both `task` and `delegate-task`. Earliest point with child session ID available. All delegation metadata (agent, description, depth, rootMain) present. Session-tracker already performs write-side mutations (childWriter, sessionIndexWriter, manifestWriter, projectIndexWriter) — adding trajectory/contract writes is consistent.
**Verified:** Line 227-360 in tool-delegation.ts. Line 312 in index.ts confirms both tools routed here.

### D-02: projectRoot Propagation
**Decision:** Add `projectRoot: string` to `ToolDelegationDeps` interface AND `HandlerDeps` interface.
**Rationale:** Both `attachTrajectoryEvidence()` and `createAgentWorkContract()` require `projectRoot`. Currently neither `ToolDelegationDeps` (line 23-32) nor `HandlerDeps` (types.ts:33-45) has this field.
**Files to modify:**
1. `src/features/session-tracker/tool-delegation.ts:23-32` — add `projectRoot: string` to `ToolDelegationDeps`
2. `src/features/session-tracker/tool-delegation.ts:41-49` — add `private readonly projectRoot: string` to `ToolDelegation` class
3. `src/features/session-tracker/capture/handlers/types.ts:33-45` — add `projectRoot: string` to `HandlerDeps`
4. `src/features/session-tracker/initialization.ts` — wire `projectRoot` to both `ToolDelegation` and `EventCapture` constructors

### D-03: Trajectory Creation at Dispatch
**Decision:** Call `attachTrajectoryEvidence()` + `eventTrajectory()` inside `recordChildTaskDelegation()` after `projectIndexWriter.addSession()` (line 331), before the `taskResult` block (line 336).
**Rationale:** Trajectory must exist before contract links to it. Both calls are idempotent (upsert pattern). Insertion point is after all session-tracker writes complete but before completion handling.
**Field mapping:**
- `trajectoryId`: `traj-${childSessionID}` (deterministic)
- `rootSessionId`: `rootMain` (already resolved at line 247)
- `sessionId`: `childSessionID` (extracted at line 233)
- `parentTrajectoryId`: `traj-${input.sessionID}` (parent session)
- `evidenceRef`: `session-tracker:delegation:${input.tool}:${childSessionID}`
- `eventType`: `"delegation_dispatch"`
- `summary`: `${input.tool} delegation to ${subagentType}: ${description.slice(0, 200)}`

### D-04: Contract Creation at Dispatch
**Decision:** Call `createAgentWorkContract()` with deterministic ID `awc-${childSessionID}`.
**Rationale:** Deterministic ID ensures idempotency on retry. Contract links to trajectory via `trajectoryId` field.
**Field mapping:**
- `id`: `awc-${childSessionID}` (deterministic)
- `owner.agent`: `subagentType` (resolved at line 239-242)
- `owner.sessionId`: `childSessionID`
- `owner.parentSessionId`: `input.sessionID`
- `scope.taskBoundary`: `description.slice(0, 500) || "Delegated task"`
- `evidence.minimumEvidenceLevel`: `"L4_IMPLEMENTATION_TRACE"` (least strict)
- `compaction.briefing`: `description.slice(0, 200) || ""`
- `trajectoryId`: `traj-${childSessionID}` (links to trajectory)

### D-05: Completion Event in Session-Idle Handler
**Decision:** Call `eventTrajectory()` in `session-idle-handler.ts` after `manifestWriter.updateChildStatus()` (line 34), inside the child session branch (line 27-46).
**Rationale:** Session-idle is the canonical completion signal. The handler already updates child status, session index, and manifest — adding trajectory event is consistent.
**Field mapping:**
- `trajectoryId`: `traj-${sessionID}` (matches dispatch trajectory)
- `eventType`: `"delegation_completed"`
- `summary`: `Child session ${sessionID} completed`
- `evidenceRef`: `session-tracker:idle:${sessionID}`

### D-06: Both Tools Covered
**Decision:** Single integration point in `recordChildTaskDelegation()` covers both `task` and `delegate-task`.
**Rationale:** Line 312 in index.ts: `if (input.tool === "task" || input.tool === "delegate-task")` — both tools route to the same method. No tool-specific branching needed.
**Verified:** The method already handles both tools identically (line 239-242 resolves `subagent_type` OR `agent`).

### D-07: Error Handling Strategy
**Decision:** Log and continue — trajectory/contract failures MUST NOT break delegation.
**Rationale:** Delegation is the critical path. Trajectory and contract are supplementary metadata. If they fail, the delegation should still succeed. Use `try/catch` with `this.client.app?.log?.()` for warnings.
**Pattern:**
```typescript
try {
  // trajectory/contract call
} catch (err) {
  void this.client.app?.log?.({
    body: {
      service: "session-tracker",
      level: "warn",
      message: `[Harness] Trajectory/contract creation failed for delegation ${childSessionID}`,
      extra: { error: err instanceof Error ? err.message : String(err) },
    },
  })
}
```
**Applies to:** Both dispatch creation (trajectory + contract) and completion event.

### D-08: Contract Defaults for Auto-Created Contracts
**Decision:** Use minimal defaults for optional contract fields. Contracts created by this integration are "seed" contracts — they establish the delegation link but don't prescribe scope constraints.
**Defaults:**
- `scope.allowedSurfaces`: `[]` (empty — no restrictions)
- `scope.dependencies`: `[]` (empty — no dependencies declared)
- `scope.nonGoals`: `[]` (empty — no non-goals declared)
- `evidence.requiredProof`: `[]` (empty — no proof required)
- `evidence.minimumEvidenceLevel`: `"L4_IMPLEMENTATION_TRACE"` (least strict)
- `evidence.verificationCommands`: `[]` (empty — no commands)
- `evidence.blockedStateRules`: `[]` (empty — no rules)
- `compaction.summary`: `""` (empty)
- `compaction.anchors`: `[]` (empty)
- `compaction.reinjectionPayload`: `""` (empty)
- `compaction.sourceRefs`: `[]` (empty)
**Rationale:** Auto-created contracts are placeholders. The child session may later update them via explicit `hivemind-agent-work-create` calls. Starting with minimal defaults avoids over-constraining the delegation.
