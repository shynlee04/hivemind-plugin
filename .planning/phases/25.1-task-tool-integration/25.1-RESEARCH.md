## Implementation Sketch

### Helper method in `ToolDelegation` class

```typescript
/**
 * Creates trajectory record and agent-work-contract for a delegation.
 * Best-effort: failures are logged but do not break delegation.
 */
private async createDelegationTrajectoryAndContract(
  childSessionID: string,
  parentSessionID: string,
  subagentType: string,
  description: string,
  rootMain: string,
  tool: string,
): Promise<void> {
  const trajectoryId = `traj-${childSessionID}`
  const contractId = `awc-${childSessionID}`

  // --- Trajectory ---
  try {
    const { attachTrajectoryEvidence, eventTrajectory } = await import(
      "../../task-management/trajectory/index.js"
    )
    attachTrajectoryEvidence({
      projectRoot: this.projectRoot,
      trajectoryId,
      rootSessionId: rootMain,
      sessionId: childSessionID,
      parentTrajectoryId: `traj-${parentSessionID}`,
      evidenceRef: `session-tracker:delegation:${tool}:${childSessionID}`,
    })
    eventTrajectory({
      projectRoot: this.projectRoot,
      trajectoryId,
      eventType: "delegation_dispatch",
      summary: `${tool} delegation to ${subagentType}: ${description.slice(0, 200)}`,
      evidenceRef: `session-tracker:child-json:${childSessionID}`,
    })
  } catch (err) {
    void this.client.app?.log?.({
      body: {
        service: "session-tracker",
        level: "warn",
        message: `[Harness] Trajectory creation failed for delegation ${childSessionID}`,
        extra: { error: err instanceof Error ? err.message : String(err) },
      },
    })
  }

  // --- Contract ---
  try {
    const { createAgentWorkContract } = await import(
      "../../features/agent-work-contracts/index.js"
    )
    createAgentWorkContract({
      projectRoot: this.projectRoot,
      id: contractId,
      owner: {
        agent: subagentType,
        sessionId: childSessionID,
        parentSessionId: parentSessionID,
      },
      scope: {
        taskBoundary: description.slice(0, 500) || "Delegated task",
        allowedSurfaces: [],
        dependencies: [],
        nonGoals: [],
      },
      evidence: {
        requiredProof: [],
        minimumEvidenceLevel: "L4_IMPLEMENTATION_TRACE",
        verificationCommands: [],
        blockedStateRules: [],
      },
      compaction: {
        briefing: description.slice(0, 200) || "",
        summary: "",
        anchors: [],
        reinjectionPayload: "",
        sourceRefs: [],
      },
      trajectoryId,
    })
  } catch (err) {
    void this.client.app?.log?.({
      body: {
        service: "session-tracker",
        level: "warn",
        message: `[Harness] Contract creation failed for delegation ${childSessionID}`,
        extra: { error: err instanceof Error ? err.message : String(err) },
      },
    })
  }
}
```

## Risks and Mitigations

### Risk 1: LOC constraint on `tool-delegation.ts` (502/500)

**Severity:** HIGH — blocks merge
**Mitigation:** Extract pure utility functions (`pruneToolInput`, `pruneToolOutput`, `extractTaskID`, `extractTaskResult`, `deriveSubagentType`, `deriveAgentNameFromSession`, `asRecord`) to `tool-delegation-utils.ts`. These are already exported and independently testable. This frees ~120 lines, making room for the integration helper. The existing tests import these functions — update imports accordingly.

### Risk 2: `createAgentWorkContract()` pressure gate blocks auto-created contracts

**Severity:** MEDIUM — contracts may not be created if runtime pressure is high
**Mitigation:** The `createAgentWorkContract()` function calls `detectRuntimePressure()` at line 23-27 of operations.ts. If `pressureScore` and `pressureTier` are not provided (both optional), the pressure check defaults to allowing the write. Since we don't pass these fields, the pressure gate will not block. Verified: `detectRuntimePressure({ score: undefined, tier: undefined, toolName: "..." })` returns `{ outcome: "allow" }`.

### Risk 3: `upsertTrajectory()` throws if `rootSessionId` is missing on new trajectory

**Severity:** LOW — `rootSessionId` is always available at the hook point
**Mitigation:** `rootMain` is resolved at line 247 via `hierarchyIndex.getRootMain(input.sessionID) ?? parentID`. The fallback to `parentID` ensures it's never undefined. The `upsertTrajectory()` function at store-operations.ts:145-147 only throws when creating a NEW trajectory without `rootSessionId` — our code always provides it.

### Risk 4: `eventTrajectory()` dedup by `eventId` may lose events

**Severity:** LOW — each call generates a unique `eventId` via `randomUUID()`
**Mitigation:** The `eventTrajectory()` function at store-operations.ts:77-78 generates `eventId: input.eventId ?? \`event-${randomUUID()}\``. Since we don't pass `eventId`, each call gets a unique ID. The filter at line 84 (`trajectory.events.filter(item => item.eventId !== event.eventId)`) only removes duplicates when the same `eventId` is passed — no data loss.

### Risk 5: Import path correctness

**Severity:** LOW — verified against barrel exports
**Mitigation:** Trajectory barrel: `src/task-management/trajectory/index.ts` exports `attachTrajectoryEvidence`, `eventTrajectory` from `store-operations.js`. Contract barrel: `src/features/agent-work-contracts/index.ts` exports `createAgentWorkContract` from `operations.js`. Both are re-exported through barrel files. Import paths from `tool-delegation.ts` would be `../../task-management/trajectory/index.js` and `../../features/agent-work-contracts/index.js`. From `session-idle-handler.ts`: `../../../../task-management/trajectory/index.js`.

### Risk 6: `session-idle-handler.ts` completion event fires for non-delegation sessions

**Severity:** LOW — trajectory lookup is idempotent
**Mitigation:** `eventTrajectory()` uses `upsertTrajectory()` which creates the trajectory if it doesn't exist. For main sessions (not child delegations), `traj-${sessionID}` would create a spurious trajectory. **Fix:** Guard the completion event with `if (childRoute)` — the idle handler already resolves `childRoute` at line 26. Only record the completion event when the session is a child (has a parent). This is already the case: the trajectory completion code is inside the `if (childRoute)` block (lines 27-46).

## Architecture Notes

### CQRS Boundary

Session-tracker is labeled "read-side observer" (index.ts:9) but already performs write-side mutations to 6+ stores. Adding trajectory/contract writes extends an existing pattern — not introducing a new violation. This is documented in CONTEXT.md D-01 rationale.

### Idempotency

Both `attachTrajectoryEvidence()` and `createAgentWorkContract()` use upsert patterns:
- Trajectory: `upsertTrajectory()` at store-operations.ts:136-163 — creates or updates by `trajectoryId`
- Contract: `upsertAgentWorkContract()` at store.ts — creates or overwrites by `id`

With deterministic IDs (`traj-${childSessionID}`, `awc-${childSessionID}`), retry is safe.

### Error Isolation

Trajectory and contract creation are independent — each wrapped in its own try/catch. A trajectory failure does not prevent contract creation, and vice versa. Both are best-effort — failures are logged but never break the delegation path.
