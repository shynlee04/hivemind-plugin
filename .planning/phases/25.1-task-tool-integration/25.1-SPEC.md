# Phase 25.1: Task Tool Integration — Specification

**Created:** 2026-05-29
**Ambiguity score:** 0.10 (gate: ≤ 0.20)
**Requirements:** 5 locked

## Goal

Wire trajectory event recording and agent-work-contract creation into OpenCode's native task tool lifecycle via session-tracker's `recordChildTaskDelegation()`. When a delegation occurs (via either `task` or `delegate-task`), a trajectory record and agent-work-contract are automatically created — no explicit tool call required.

## Background

### Current State: Disconnected Modules

The trajectory module (`src/task-management/trajectory/`) and agent-work-contract module (`src/features/agent-work-contracts/`) are **disconnected from the delegation lifecycle**. Both require explicit tool calls (`hivemind-trajectory` and `hivemind-agent-work-create`) to create records. When a user delegates via the native `task` tool or custom `delegate-task` tool, no trajectory or contract is created automatically.

### Hook Point: `recordChildTaskDelegation()`

Session-tracker handles BOTH `task` and `delegate-task` tools at a single integration point:

- **File:** `src/features/session-tracker/tool-delegation.ts:227-360`
- **Trigger:** `src/features/session-tracker/index.ts:312` — `if (input.tool === "task" || input.tool === "delegate-task")`
- **Data available:** `childSessionID`, `parentSessionID`, `subagentType`, `description`, `rootMain`, `depth`, `input.tool`
- **Missing:** `projectRoot` (not in `ToolDelegationDeps`)

## Requirements

### REQ-25.1-01: Trajectory Auto-Creation on Dispatch

When a delegation occurs, a trajectory record is automatically created.

| Dimension | Value |
|-----------|-------|
| **Current** | Trajectory only created via explicit `hivemind-trajectory` tool call |
| **Target** | `recordChildTaskDelegation()` calls `attachTrajectoryEvidence()` + `eventTrajectory()` |
| **Acceptance** | (1) After delegating via `task` tool, trajectory record exists in ledger with `trajectoryId: "traj-${childSessionID}"`, (2) Trajectory has `delegation_dispatch` event with correct summary, (3) `rootSessionId` matches resolved `rootMain`, (4) `parentTrajectoryId` is `"traj-${parentSessionID}"`, (5) Typecheck passes |

### REQ-25.1-02: Contract Auto-Creation on Dispatch

When a delegation occurs, an agent-work-contract is automatically created.

| Dimension | Value |
|-----------|-------|
| **Current** | Contract only created via explicit `hivemind-agent-work-create` tool call |
| **Target** | `recordChildTaskDelegation()` calls `createAgentWorkContract()` |
| **Acceptance** | (1) After delegating via `task` tool, contract exists with `id: "awc-${childSessionID}"`, (2) `owner.agent` matches `subagentType`, (3) `owner.sessionId` matches `childSessionID`, (4) `owner.parentSessionId` matches `input.sessionID`, (5) `scope.taskBoundary` populated from description, (6) Contract has `trajectoryId` linking to trajectory record, (7) Typecheck passes |

### REQ-25.1-03: Trajectory Completion Event

When a delegation completes, a trajectory completion event is recorded.

| Dimension | Value |
|-----------|-------|
| **Current** | No trajectory event on delegation completion |
| **Target** | `session-idle-handler.ts` calls `eventTrajectory()` on completion |
| **Acceptance** | (1) After delegation completes (session becomes idle), trajectory has `delegation_completed` event, (2) Event references correct `trajectoryId: "traj-${sessionID}"`, (3) `evidenceRef` points to idle handler, (4) Typecheck passes |

### REQ-25.1-04: Both Tools Covered

Both native `task` and custom `delegate-task` create trajectory + contract.

| Dimension | Value |
|-----------|-------|
| **Current** | Neither tool creates trajectory or contract |
| **Target** | Single integration point in `recordChildTaskDelegation()` covers both tools |
| **Acceptance** | (1) Delegating via `task` tool creates trajectory + contract, (2) Delegating via `delegate-task` tool creates trajectory + contract, (3) Both use same code path (no tool-specific branching), (4) Typecheck passes |

### REQ-25.1-05: projectRoot Propagation

`projectRoot` is available at the hook point for trajectory/contract writes.

| Dimension | Value |
|-----------|-------|
| **Current** | `ToolDelegationDeps` doesn't include `projectRoot`; `HandlerDeps` doesn't include `projectRoot` |
| **Target** | `projectRoot` added to both `ToolDelegationDeps` and `HandlerDeps`, wired through initialization |
| **Acceptance** | (1) `ToolDelegationDeps` interface includes `projectRoot: string`, (2) `HandlerDeps` interface includes `projectRoot: string`, (3) `ToolDelegation` class stores `projectRoot` as private field, (4) `initialization.ts` passes `projectRoot` to both `ToolDelegation` and `EventCapture` constructors, (5) `recordChildTaskDelegation()` can access `this.projectRoot`, (6) `session-idle-handler.ts` can access `this.deps.projectRoot`, (7) Typecheck passes |
