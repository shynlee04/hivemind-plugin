## 3. Design Error Analysis

### DE-01: No Status State Machine — CONFIRMED with L1 evidence
All 7 contracts are stuck in `status: "created"`. No contract has ever transitioned to `running`, `blocked`, `completed`, or `cancelled`.

- **Schema** (`agent-work-contract.schema.ts`): Defines `AgentWorkStatus` enum with 5 states, and `AgentWorkMutationInput` with optional `status` field
- **Tool** (`hivemind-agent-work.ts`): Only creates contracts — no mutation/update tool exists
- **Ledger persistence**: No mutation module in the trajectory stack that could update contract status
- **Evidence**: All 7 contracts at `agent-work-contracts.json` confirm this — none transitioned
- **Impact**: HIGH — contracts are write-once, cannot be used for lifecycle tracking

### DE-02: No Concurrent Write Protection — CONFIRMED
- `src/task-management/trajectory/ledger.ts` uses atomic rename (`writeFileAtomicSync`) but no file locking
- Two concurrent agent sessions writing to `trajectory-ledger.json` will race
- Evidence: Read-modify-write pattern in `writeTrajectoryLedger` with no synchronization

### DE-03: Checkpoint vs Event Semantic Overlap — CONFIRMED via runtime evidence
- `traj_uat_phase16_02` has BOTH checkpoints and events for overlapping concerns
- `chk_uat_03_polling` (checkpoint) tracks state before `evt_uat_01` (event for "delegation dispatched") — temporal confusion
- Existing audit already identified this as DE-03

### DE-04: Trajectory Traversal Default Unsorted — CONFIRMED
- `store-operations.ts` has `inspectTrajectoryLedger` which returns raw data
- No sorting guarantee on events or checkpoints
- Evidence: `traj_thinking_001` events in creation order but no explicit sort contract

### DE-05: No Contract ↔ Trajectory Linkage in Runtime — PARTIALLY MITIGATED
- Runtime evidence shows `awc_e8392ff5` has `trajectoryId: "traj_uat_phase16_02"` — the linkage field EXISTS
- However, the link is one-way (contract → trajectory). No trajectory → contract reverse lookup
- Also: linkage is manual/optional, not enforced at the schema or tool level

### DE-06: Mutation Input Allows Simultaneous Evidence+Status Conflict — CONFIRMED
- `AgentWorkMutationInput` schema allows setting `status` AND `evidence` in same mutation
- No validation against incompatible combinations
- Evidence: Code review of schema definition

## 9. Summary & Recommendations

### Critical blockers before Phase 25 can begin execution:
1. **DE-01: Status state machine** — Must implement contract mutation tool and lifecycle transitions (created → running → completed/cancelled)
2. **FG-01: Notification subsystem** — Must implement notification-router, notification-formatter, notification-handler before cross-agent features depend on it
3. **DE-02: Concurrent write protection** — File locking or transactional writes required for trajectory ledger

### Medium priority:
4. **DE-03: Checkpoint/event semantic** — Document clear semantics; checkpoint = state snapshot at a point in time, event = something that happened
5. **DE-04: Sort contract** — Add `sort: "asc" | "desc"` parameter to trajectory traversal
6. **DE-05: Bidirectional linkage** — Add trajectory → contract query capability
7. **DE-06: Mutation validation** — Add Zod refinement preventing conflicting mutation inputs

### Low priority:
8. State machine enforcement at schema level (Zod refinements on contract transitions)
9. Trajectory evidence deduplication
