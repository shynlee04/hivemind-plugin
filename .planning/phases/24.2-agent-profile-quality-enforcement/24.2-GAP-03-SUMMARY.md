## Agent-Specific Enhancements

### hm-intent-loop (56 → 120 lines)
Progressive disclosure methodology (5 levels: broad → specific → confirm → edge → close), max-question guard (10 per session), expanded 10-step execution flow with redirect detection.

### hm-ecologist (55 → 100 lines)
Dependency graph rules (explicit, directed, leveled), expanded 10-step execution flow with topological sort, circular dependency detection, impact analysis.

### hm-shipper (57 → 119 lines)
Pre-release checklist (10 items: typecheck, tests, lint, build, CHANGELOG, migrations, env vars, rollback, manifest, git tag), expanded 10-step execution flow.

### hm-nyquist-auditor (55 → 100 lines)
Evidence audit rules (L1-L3 acceptable, L4-L5 = gap, mock-only = deceptive), expanded 10-step execution flow with gap-filling test creation.

### hm-intel-updater (54 → 90 lines)
Intel JSON format specification (module, path, exports, patterns, conventions, last_updated, status), expanded 8-step execution flow.

### hm-security-auditor (54 → 130 lines)
STRIDE template with 6 categories (Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, Elevation of Privilege), package_legitimacy_gate for new dependencies, expanded 10-step execution flow with trust boundary mapping.

### hm-debugger (57 → 130 lines)
Debug workflow (Symptom → Hypothesis → Evidence → Eliminate → Confirm), hypothesis_template (predicted evidence, test, result, evidence), expanded 10-step execution flow.

### hm-user-profiler (54 → 98 lines)
8 behavioral profile dimensions (Communication, Decisions, Explanations, Debugging, UX, Vendors, Frustrations, Learning) with 1-10 scoring and confidence levels (HIGH/MEDIUM/LOW/UNSCORED), expanded 8-step execution flow.

### hm-ui-researcher (58 → 102 lines)
UI-SPEC template with component state coverage table (default, hover, active, disabled, error, loading, empty), expanded 10-step execution flow.

### hm-ui-checker (55 → 80 lines)
BLOCK/FLAG/PASS verdict methodology, expanded 8-step execution flow with completeness, consistency, and actionability checks.

### hm-ui-auditor (57 → 113 lines)
6-pillar audit methodology (Layout accuracy, State coverage, Responsive behavior, Animation, Accessibility, Token compliance), severity-classified findings (ERROR/WARNING/INFO), expanded 10-step execution flow.

### hm-doc-writer (55 → 103 lines)
Doc quality standards (6 standards: API claims verified, file paths verified, commands verified, no aspirational content, reproducible setup, real code examples), expanded 10-step execution flow with self-verify claims step.

### hm-doc-verifier (54 → 89 lines)
Claim verification categories (API signatures, Config keys, File paths, Commands, Behavior descriptions, Version numbers), VERIFIED/FALSE/UNVERIFIABLE classification, expanded 8-step execution flow.

### hm-orchestrator (59 → 160 lines)
Intent classification table (12 domains with delegation targets and triggers), delegation rules (NEVER implement, fresh context, task tool, max 3 retries), quality gate triad (lifecycle → spec compliance → evidence truth), expanded 12-step execution flow.
