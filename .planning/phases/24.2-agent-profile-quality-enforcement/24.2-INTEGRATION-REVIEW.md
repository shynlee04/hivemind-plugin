### 2.4 Conflict Issues — 1 Pair Flagged

#### hm-codebase-mapper ↔ hm-architect — ARCHITECTURE.md Collision (WARNING)

**hm-codebase-mapper** produces:
- `.planning/codebase/ARCHITECTURE.md` — Documenting existing codebase structure (observational)

**hm-architect** produces:
- `.planning/` or `.planning/codebase/ARCHITECTURE.md` — Designing new architecture (prescriptive)

Both produce an artifact named `ARCHITECTURE.md` in overlapping locations. The mapper's ARCHITECTURE.md describes "what exists" while the architect's describes "what should be." When both run on the same codebase, they could overwrite each other's work or create confusion about which ARCHITECTURE.md is authoritative.

**Impact:** MEDIUM — The two agents have different purposes (observational vs prescriptive) but produce the same artifact name. This could lead to:
1. One agent overwriting the other's output
2. Confusion about which ARCHITECTURE.md is current
3. A reader not knowing if the doc describes reality or design intent

**Recommendation:** Disambiguate by:
- **hm-codebase-mapper**: Uses `.planning/codebase/ARCHITECTURE.md` (CODEBASE-ARCHITECTURE.md or CODEBASE-MAP.md) — already correctly uses `.planning/codebase/` subdirectory
- **hm-architect**: Uses `.planning/ARCHITECTURE.md` or `.planning/architecture/ARCHITECTURE.md` — should be explicit about NOT writing to `.planning/codebase/`

The hm-codebase-mapper already specifies `.planning/codebase/` which provides some separation. The hm-architect's contract says `.planning/` or `.planning/codebase/` — remove the `.planning/codebase/` option from hm-architect to eliminate the overlap.
