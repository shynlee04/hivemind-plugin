# Phase 24.2 — Agent Profile Quality Enforcement

## Domain

Upgrade the 30 hm-* agent profile **bodies** from TODO/draft stubs to production-grade expert instructions. Agent frontmatter (mode: all, hidden: true) is already fixed from Phase 24.1. This phase fills in:

- Execution flows (steps, deviation rules, analysis paralysis guard)
- Artifact contracts (what each produces, naming, format)
- Success criteria (what proves completion)
- Delegation boundaries (when to switch agents)
- GSD-inspired patterns adapted for Hivemind (no procedural logic in profiles)

## Requirements

| ID | Description | Verification |
|----|-------------|-------------|
| Q-01 | All 30 hm-* agents have complete execution flow sections (steps, deviation rules, analysis paralysis guard) | Read each file, verify `<execution_flow>` or equivalent present |
| Q-02 | All 30 hm-* agents have artifact output specifications in their body | Each agent's body documents what files it creates, naming convention, location |
| Q-03 | All 30 hm-* agents have clear success criteria | Each agent ends with a success_criteria checklist |
| Q-04 | No agent body contains looping/gating/hierarchy logic | Grep for "loop", "gate", "checkpoint" — these belong in orchestrator/programmatic features |
| Q-05 | No agent body contains tool/permission references | Grep for "tools:", "permission:", "temperature:", "steps:" — removed per D-24-02 |
| Q-06 | Each agent has delegation/boundary rules (when to switch to another agent) | Each agent documents "if task is not my domain → signal orchestrator" |
| Q-07 | GSD patterns adopted: artifact naming, gate sub-files, deviation rules | Pattern verified from RESEARCH.md synthesis |
| Q-08 | Artifact schema reference created | Canonical Artifact Registry document in `.planning/references/` |

## Out of Scope

- Commands infrastructure (Phase 24.3-24.6)
- hf-* line agents
- Programmatic feature implementation (session-tracker, etc.)
- Old hm-l2-* agent archiving
