    ## Section 2: YAML Frontmatter Field Definitions
    Standard fields used across artifacts:
    - `phase`: Phase identifier (e.g., `24.2-agent-profile-quality-enforcement`)
    - `plan`: Plan number within phase (e.g., `01`, `02`)
    - `type`: Artifact type (`plan`, `research`, `spec`, `summary`, `review`, `verification`, `security`, `ui-spec`)
    - `status`: Lifecycle status (`draft`, `review`, `approved`, `superseded`, `archived`)
    - `depends_on`: Array of plan IDs this artifact depends on
    - `requirements`: Array of requirement IDs addressed
    - `date`: Creation date (YYYY-MM-DD format)
    - `author`: Agent or author name

    ## Section 3: Lifecycle Rules
    - **Create:** When a phase starts or research begins
    - **Update:** Only by the producing agent or human edit. Version via `status: draft → review → approved`
    - **Archive:** When phase is complete and milestone archived. Move to `.planning/archive/`
    - **Supersede:** When a newer version replaces an older one. Update old artifact to `status: superseded`
    - **Delete:** Never delete artifacts (git preserves history). Supersede or archive instead.

    ## Section 4: Naming Convention Rules
    - Phase-level artifacts: `{NN}-{TYPE}.md` where NN = zero-padded phase number (e.g., `24.2-RESEARCH.md`)
    - Plan-level artifacts: `{NN}-{MM}-{TYPE}.md` where MM = zero-padded plan number (e.g., `24.2-01-PLAN.md`)
    - Research artifacts: `{name}-{YYYY-MM-DD}.md` for date-stamped files
    - Debug artifacts: `{timestamp}-debug-{id}.md`
    - Root-level artifacts: Fixed names (e.g., `ROADMAP.md`, `PROJECT.md`, `STATE.md`, `CHANGELOG.md`)

    ## Section 5: Cross-Reference
    Per-agent artifact summary linking each hm-* agent to its produced artifact pattern(s). Quick lookup table.
