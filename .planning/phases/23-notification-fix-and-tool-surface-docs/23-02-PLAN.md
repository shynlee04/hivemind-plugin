# Plan 23-02: Periodic Silent Injection for Delegation Progress

## Objective

Implement the PeriodicNotifier class (Option C from design doc) that provides periodic progress updates to parent sessions during running child delegations, solving the "silent gap" where parents receive no feedback between dispatch and completion.

**Purpose:** Users currently see a frozen parent session during long-running delegations. PeriodicNotifier fills this gap with thin, deduplicated, batch-coalesced status lines injected via `sendPromptAsync()`.

**Output:** New `periodic-notifier.ts` (~150 LOC), wiring in `coordinator.ts` (+5 LOC) and `plugin.ts` (+8 LOC), full unit tests (~200 LOC).

### Locked Design Decisions (from design doc)

| Decision | Value | Why |
|----------|-------|-----|
| Architecture option | **Option C** — new PeriodicNotifier class | SRP, testable, decoupled |
| Cadence | Reuse monitor's POLLING_CADENCE | No new timers |
| Injection mechanism | `sendPromptAsync()` | Fire-and-forget, agent-visible, no turn |
| Toast | `showTuiToast()` alongside each injection | Best-effort, configurable |
| Thin line format | `[ses_xxx → agent] running | tools:N bash:N actions:N | XmYs` | < 120 chars |
| Batch window | 2-second coalescing | Prevents flooding with 10+ delegations |
| Dedup | Strip duration before comparing | Only inject when actions/tools/messages change |
| Module placement | `src/coordination/delegation/periodic-notifier.ts` | Coordination layer per 9-surface model |
| Import boundary | PeriodicNotifier imports ONLY from `shared/` and `notification-formatter.ts` | Leaf-like dependency |
| Error handling | `sendPromptAsync` failures logged, never thrown | Fire-and-forget resilience |

```
PeriodicNotifier
  handlePollTick
    ✓ should inject on first tick when no previous state exists
    ✓ should skip injection when only duration changed (dedup strips duration)
    ✓ should inject when tool count changed
```

**Action:**

1. **RED phase:** Create test file `tests/lib/coordination/delegation/periodic-notifier.test.ts`. Write ALL tests listed above. Mock `sendPromptAsync` and `showTuiToast` from `session-api.ts`. Use `vi.useFakeTimers()` for the 2s batch window tests. Run tests — ALL must fail (no implementation yet).

2. **GREEN phase:** Create `src/coordination/delegation/periodic-notifier.ts` implementing:

   - `PeriodicNotifierConfig` interface with `maxTrackedDelegations` (default 20), `dedupThreshold` (default 0), `showToast` (default true), `maxLineLength` (default 120)
   - `DelegationSnapshot` interface (delegationId, parentSessionId, agent, childSessionId, status, actionCount, toolCallCount, messageCount, stablePollCount, elapsedSeconds)
   - `PeriodicNotifier` class with:
     - Constructor takes `client: OpenCodeClient` and optional `Partial<PeriodicNotifierConfig>`
     - Private state: `lastInjected: Map<string, string>`, `pendingUpdates: Map<string, { parentSessionId, line }>`, `trackedDelegations: Map<string, DelegationSnapshot>`, `flushTimer: NodeJS.Timeout | undefined`
     - `handlePollTick(parentSessionId, line, delegationId?)`: Skip if no delegationId. Strip duration from line using regex `/\s*\|\s*\d+m?\d*s$/`. Compare stripped line vs `lastInjected`. If changed, update `lastInjected`, add to `pendingUpdates`, call `scheduleFlush()`.
     - `register(delegationId, parentSessionId, agent, childSessionId)`: Store in `trackedDelegations`. Initialize `lastInjected` entry as empty string (forces first injection).
     - `deregister(delegationId)`: Remove from `trackedDelegations`, `lastInjected`, `pendingUpdates`.
     - `flush()`: Group pending updates by parentSessionId. For each group: if 1 delegation, inject single line; if 2-3, format combined `[N running] ...`; if 4+, use compact format. Call `sendPromptAsync()` and optionally `showTuiToast()`. Clear pending and timer.
     - `scheduleFlush()`: If no timer, set 2s `setTimeout` to call `flush()`.
     - `stripDuration(line)`: Regex to remove trailing `| XmYs` or `| Xs`.
     - `activeCount` getter: Return `trackedDelegations.size`.
   - Import ONLY from `../../shared/session-api.js` (for `sendPromptAsync`, `showTuiToast`) and `./notification-formatter.js` (for `formatDuration`).
   - All thrown errors use `[Harness]` prefix. No `any` types.
   - JSDoc on all exported functions and class.

3. **REFACTOR phase:** Review for code clarity, remove any redundancy, ensure module stays under 200 LOC. Re-run tests — all must pass.

## Threat Model

### Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PeriodicNotifier → OpenCode SDK | `sendPromptAsync` and `showTuiToast` calls cross the plugin↔host boundary |

### STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-23-02-01 | I (Info Disclosure) | `sendPromptAsync` injection content | mitigate | Line content contains only delegation ID, agent name, tool/action counts — no user data, no file contents, no secrets |
| T-23-02-02 | D (Denial of Service) | Batch flush with 20+ delegations | accept | `maxTrackedDelegations: 20` caps tracking; batch coalescing limits to 1 injection per 2s regardless of delegation count |
| T-23-02-03 | T (Tampering) | Injection to wrong parent session | mitigate | `parentSessionId` is tracked per-delegation at registration, not per-tick; dedup map keys include delegation ID |
