<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Tool args (agent, brief, title) | Untrusted input from agent caller, validated by Zod schema |
| Session API (create, prompt) | Trusted SDK calls to OpenCode runtime |
| TUI toast | One-way notification, no data flow back |
| Git commit via execSync | Best-effort; failure must not propagate to caller |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24.3.1-01 | Spoofing | Tool args | mitigate | Zod safeParse validation at execute entry — rejects missing/invalid fields |
| T-24.3.1-02 | Tampering | execSync git command | mitigate | Wrapped in try/catch — git failure never propagates; returns success even if git fails |
| T-24.3.1-03 | Information Disclosure | brief in prompt | accept | Governance brief is intended content for child session; no PII expected |
| T-24.3.1-04 | Denial of Service | SDK call failure | mitigate | All SDK calls wrapped in try/catch — individual failures return error message, never crash the tool |
</threat_model>
