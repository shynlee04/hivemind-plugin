## Test 1: Session Naming Service — Generate & Parse

### Mục tiêu
Xác nhận naming service hoạt động đúng, format title machine-readable + human-readable.

### Steps

1. **Chạy unit test naming service:**
```bash
npx vitest run tests/features/governance-engine/ -t "naming"
```

Kỳ vọng: ✅ PASS — generate + parse round-trip hoạt động.

2. **Kiểm tra format output:**

Mở `src/shared/session-naming.ts` và xác nhận:

| Input | Expected Output |
|-------|----------------|
| `{framework:"hm", workflow:"governance", classification:"root", agent:"gsd-auditor", purpose:"audit-phase23", depth:0}` | `hm/governance/root/gsd-auditor/audit-phase23@0` |
| `{framework:"hm", workflow:"delegate", classification:"child", agent:"gsd-researcher", purpose:"research-mcp-patterns", depth:1}` | `hm/delegate/child/gsd-researcher/research-mcp-patterns@1` |
| `{framework:"hm", workflow:"spawn", classification:"grandchild", agent:"gsd-executor", purpose:"implement-fix", depth:2}` | `hm/spawn/grandchild/gsd-executor/implement-fix@2` |

3. **Parse reverse:**
```bash
node -e "
const { generateSessionTitle, parseSessionTitle } = require('./dist/shared/session-naming.js');
const title = generateSessionTitle({framework:'hm', workflow:'delegate', classification:'child', agent:'gsd-researcher', purpose:'research-mcp', depth:1});
console.log('Generated:', title);
const parsed = parseSessionTitle(title);
console.log('Parsed:', JSON.stringify(parsed, null, 2));
console.log('Round-trip:', parsed.agent === 'gsd-researcher' && parsed.workflow === 'delegate' ? '✅ PASS' : '❌ FAIL');
"
```

Kỳ vọng: ✅ Round-trip pass.

## Test 3: `create-governance-session` — Root Session Real

### Mục tiêu
Xác nhận governance session là ROOT session thật (không phải phantom), hiển thị trong `opencode session list`.

### Steps

1. **Chạy tool trong OpenCode:**

Paste vào prompt:
```
Create a governance sub-session to audit the notification architecture in Phase 23 before I continue with P24. Use the create-governance-session tool with:
- agent: "gsd-auditor"
- brief: "Audit the notification architecture fix implemented in Phase 23. Check: 1) Are synthetic notifications properly handled? 2) Is the notification router correctly filtering types? 3) Are there any remaining bugs in the delegation notification loop? Read src/plugin.ts lines 82-84 and src/coordination/delegation/notification-router.ts. Report findings back to this session."
- title: "audit-phase23-notifications"
```

2. **Verify session tồn tại thật:**
```bash
opencode session list --format json | python3 -c "
import sys, json
data = json.load(sys.stdin)
for s in data:
    title = s.get('title', '')
    if 'governance' in title.lower() or 'audit' in title.lower():
        print(f'✅ FOUND: {s.get(\"id\",\"?\")} -> {title}')
"
```

Kỳ vọng: ✅ Session hiển thị trong list với title format `hm/governance/root/gsd-auditor/audit-phase23-notifications@0`.

3. **Verify naming convention trong title:**
```bash
opencode session list --format json | python3 -c "
import sys, json
data = json.load(sys.stdin)
for s in data:
    title = s.get('title', '')
    parts = title.split('/')
    if len(parts) == 5 and '@' in parts[4]:
        framework, workflow, classification, agent, purpose_depth = parts
        depth = purpose_depth.split('@')[1]
        print(f'Framework: {framework}')
        print(f'Workflow: {workflow}')
        print(f'Classification: {classification}')
        print(f'Agent: {agent}')
        print(f'Depth: {depth}')
        has_id = bool(s.get('id','').startswith('ses_'))
        if has_id:
            print('✅ REAL session with naming convention')
"
```

4. **Switch vào session và verify brief:**
```bash
opencode session select <session-id>
```
Sau đó trong session mới paste: `Continue the audit task.`

Kỳ vọng: ✅ Session có brief đã inject, agent xử lý audit task.

5. **Rời session và quay lại session chính:**
```bash
opencode session select <main-session-id>
```

## Test 4: Session Tracker — Title trong Frontmatter

### Mục tiêu
Xác nhận session `.md` file có title trong YAML frontmatter.

### Steps

```bash
# Tìm session mới nhất

echo "Session dir: $LATEST_SESSION"
SESSION_ID=$(basename "$LATEST_SESSION")
```

Kỳ vọng: ✅ Frontmatter có field `title:` với giá trị không phải "unknown".

```bash

  if [ -z "$title" ]; then
    echo "❌ MISSING title: $f"
  else
    echo "✅ $title"
  fi
done
```

Kỳ vọng: ✅ Không còn session nào thiếu title (trừ sessions cũ tạo trước bản fix này).

### Steps

```bash

import sys, json
data = json.load(sys.stdin)
# Find latest delegations
for d in data[-5:]:
    agent = d.get('agent', '?')
    child = d.get('childSessionId', '?')
    status = d.get('status', '?')

    unknowns = []
    if agent == 'unknown' or not agent: unknowns.append('agent')
    if d.get('subagentType', '') == 'unknown': unknowns.append('subagentType')
    
    if unknowns:
        print(f'❌ {child}: unknown fields: {unknowns}')
    else:
        print(f'✅ {child}: agent={agent}, status={status}')
"
```
