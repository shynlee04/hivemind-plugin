<objective>
Create `src/shared/session-naming.ts` — a SINGLE source of truth for generating and parsing all OpenCode session titles across the harness.

**Purpose:** Currently every session creation point constructs titles independently with ad-hoc patterns: `Delegation: ${agent}`, `args.request.title`, `hm-governance:{agent}-{title}`.   <action>
    Create `src/shared/session-naming.ts` as a pure-logic, zero-dependency utility module.

    **Type definitions:**

    ```typescript
    export interface NamingInput {
      framework: string       // "hm" | "gsd"
      workflow: string        // "governance" | "delegate" | "planning" | "execute" | "spawn"
      classification: "root" | "child" | "grandchild" | "fork"
      agent: string           // "gsd-auditor", "gsd-planner", "hm-l2-researcher"
      purpose: string         // short kebab-case description, e.g. "audit-phase23"
      depth: number           // 0 = root, 1 = child, 2 = grandchild, etc
    }

    export interface ParsedNaming {
      framework: string
      workflow: string
      classification: string
      agent: string
      purpose: string
      depth: number
    }
    ```

    **generateSessionTitle(input: NamingInput): string**
    
    Format: `{framework}/{workflow}/{classification}/{agent}/{purpose}@{depth}`
    
    Rules:
    - All fields are required (non-empty validation)
    - purpose is sanitized: lowercase, spaces→hyphens, strip non-alphanumeric (keep hyphens)
    - depth is coerced to non-negative integer
    - Returns the formatted string
    
    Example outputs:
    - `generateSessionTitle({framework:"hm",workflow:"governance",classification:"root",agent:"gsd-auditor",purpose:"audit-phase23-notifications",depth:0})`
      → `"hm/governance/root/gsd-auditor/audit-phase23-notifications@0"`
    - `generateSessionTitle({framework:"hm",workflow:"delegate",classification:"child",agent:"gsd-researcher",purpose:"research-mcp-patterns",depth:1})`
      → `"hm/delegate/child/gsd-researcher/research-mcp-patterns@1"`
    - `generateSessionTitle({framework:"hm",workflow:"planning",classification:"root",agent:"gsd-planner",purpose:"plan-gap-closure",depth:0})`
      → `"hm/planning/root/gsd-planner/plan-gap-closure@0"`

    **parseSessionTitle(title: string): ParsedNaming | null**
    
    Reverse parsing: splits on `/`, extracts 5 parts, splits last part on `@` for purpose/depth.
    
    Returns null for:
    - Fewer than 5 slash-separated segments
    - Last segment missing `@` character
    - Empty segments
    - Non-numeric depth value
    
    Examples:
    - `parseSessionTitle("hm/delegate/child/gsd-researcher/research-mcp-patterns@1")`
      → `{framework:"hm", workflow:"delegate", classification:"child", agent:"gsd-researcher", purpose:"research-mcp-patterns", depth:1}`
    - `parseSessionTitle("invalid")` → `null`
    - `parseSessionTitle("hm/delegate/child/gsd-researcher/research@bad")` → `null`

    **Implementation requirements:**
    - Pure module — NO file I/O, NO runtime dependencies, NO imports from other harness modules
    - Only import from standard library: no import needed (pure TypeScript)
    - JSDoc on all exports per jsdoc-typescript-docs skill
    - `[Harness]` prefix NOT needed (this is a utility, not an error message)
    - Export both functions and both interfaces

    Register the directory:

