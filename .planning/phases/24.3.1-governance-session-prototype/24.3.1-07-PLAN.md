<objective>
Fix delegation metadata propagation in the core chain — `tool-delegation.ts`, `tool-capture.ts`, and `child-recorder.ts` — so that `delegatedBy.agentName`, `mainAgent.name`, and `subagentType` contain actual agent values instead of `"unknown"`.

**Purpose:** Currently all governance sessions and delegated sessions record `"unknown"` for `delegatedBy.agentName`, `mainAgent.name`, and `subagentType` in the session tracker. This breaks traceability. With naming service titles now standardized (Plan 06/06b), the agent name, workflow, and classification are encoded IN THE TITLE — the metadata propagation fix extracts these values from the title via `parseSessionTitle()` rather than relying on ad-hoc fallback chains.

**Key insight:** Since Plan 06/06b standardized ALL session titles to the format `{framework}/{workflow}/{classification}/{agent}/{purpose}@{depth}`, the session tracker can now use `parseSessionTitle()` to extract agent name, workflow, classification, and depth from any session title — eliminating the need for separate metadata propagation for these fields in the core chain.

**Output:**
- Fixed delegation metadata in tool-delegation.ts, tool-capture.ts, child-recorder.ts — uses `parseSessionTitle()` as primary source
</objective>
