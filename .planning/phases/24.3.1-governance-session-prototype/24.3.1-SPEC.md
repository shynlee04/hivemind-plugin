# Phase 24.3.1: Governance Session Prototype — SPEC

## Purpose

Validate technical feasibility of agent-initiated governance session creation using OpenCode SDK. Prove that an agent can programmatically create a named child session, inject governance context via system prompt, persist handoff state via git commit, and notify the user via TUI — before investing in full Commands Infrastructure (P24.4-P24.6).

## Scope

**IN SCOPE:**
- Custom OpenCode tool `create-governance-session` (TypeScript, `@opencode-ai/plugin`)
- Session creation via `sdk.session.create()` with explicit `hm-governance:` title naming
- System prompt injection via `sdk.session.prompt()` or `sdk.session.promptAsync()`
- Git commit of current context before session handoff
- TUI notification via `showToast()` and `appendTuiPrompt()`
- Plugin registration in `src/plugin.ts`
- Unit test with mocked SDK calls

**OUT OF SCOPE:**
- Full governance workflow engine (deferred to P24.4-P24.6)
- Session auto-naming fix for child sessions (deferred — use explicit title)
- Multi-agent coordination (deferred to P24.5-P24.6)
- Production hardening (error handling, retry, circuit breaker)
- User session switching UX (user manually switches)
