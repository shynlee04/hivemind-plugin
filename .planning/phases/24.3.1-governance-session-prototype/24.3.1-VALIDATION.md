## Validation Dimensions

### Session Naming Service
- `generateSessionTitle` produces correct format for all workflow types (governance, delegate, planning, execute, spawn)
- `parseSessionTitle` correctly reverses `generateSessionTitle` output (roundtrip)
- Edge cases: empty fields, non-numeric depth, too few segments, empty segments
- Purpose sanitization: spaces→hyphens, lowercase, strip non-alphanumeric

### Session Tracker Title
- `SessionRecord.title` field exists and is optional
- `initializeSessionFile` writes title into YAML frontmatter
- `updateFrontmatter` passes through title correctly
- Re-initialization preserves existing title
- KernelPacket title resolves from continuity metadata → agent name → sessionID (no more "unknown")

### Governance Config
- `readGovernanceConfig()` validates JSON schema with Zod
- `resolveAgentForBrief()` matches keywords case-insensitively
- `validateNamingTitle()` validates framework, workflow, classification against allowed lists
- Config missing/failure handled gracefully (non-fatal)

### Creation Points (all 4 paths)
- **SDK child session starter** — `"hm/delegate/child/{agent}/{purpose}@1"`
- **PTY spawner session creator** — `"hm/spawn/child/{agent}/{purpose}@1"` (fallback only)
- **Spawn request builder** — `"hm/delegate/child/{agent}/{purpose}@1"` (fallback only)
- **Governance tool** — `"hm/governance/root/{agent}/{purpose}@0"` (root session, no parentID)

### Metadata Propagation (no more "unknown")
- `tool-delegation.ts` — `delegatedBy.agentName` from `parseSessionTitle`
- `tool-capture.ts` — `delegatorAgentName` and `mainAgent.name` from `parseSessionTitle`
- `child-recorder.ts` — `effectiveActor` from `parseSessionTitle`
- `event-capture.ts` — `agentName` from `parseSessionTitle`
- `hierarchy-manifest.ts` — `delegatedBy` and `subagentType` from `parseSessionTitle`
