<objective>
Add `title` field to `SessionRecord` type, propagate it through session writer frontmatter, and fix the hardcoded `title: "unknown"` in the kernel packet compaction hook.

**Purpose:** Every session `.md` file must carry a title in YAML frontmatter so the session tracker can display meaningful session names. Currently 100% of session files lack titles because the `SessionRecord` interface has no `title` field, `initializeSessionFile` never writes one, and `session-hooks.ts` hardcodes `"unknown"`. The naming service from Plan 00 generates standardized titles — this plan ensures those titles are captured and persisted.

**Output:**
- Updated `SessionRecord` type with `title?: string`
- Fixed `initializeSessionFile` accepting and persisting title from naming service
- Fixed `KernelPacket` construction with real title from continuity metadata
- Unit tests verifying title capture through frontmatter lifecycle
</objective>
