<objective>
Create the governance configuration plane: a JSON config file at `.hivemind/governance/config.json` plus a typed config reader utility that validates the schema and provides typed access — now including a `naming_standards` section that defines the project-wide session naming convention.

**Purpose:** The governance config plane maps available agents, commands, and templates so the `create-governance-session` tool can resolve which agent to dispatch from the brief content. The new `naming_standards` section documents the accepted `framework` prefixes, `workflow` types, and `classification` terms that the naming service (Plan 00) uses — making naming convention configurable and auditable.

**Output:**
- `.hivemind/governance/config.json` — config with agents, commands, templates, naming_standards
- `.hivemind/governance/.gitkeep` — directory registration
- `src/features/governance-engine/config-reader.ts` — typed reader + Zod schema + naming validation
- Unit tests for config reader including naming standards
</objective>

Config file location (per Q6): `.hivemind/governance/config.json`

Naming convention format (from approved design):
```
{framework}/{workflow}/{classification}/{agent}/{purpose}@{depth}
```

New naming_standards section in config:
```yaml
naming_standards:
  allowed_frameworks: ["hm", "gsd"]
  allowed_workflows: ["governance", "delegate", "planning", "execute", "spawn"]
  allowed_classifications: ["root", "child", "grandchild", "fork"]
  naming_format: "{framework}/{workflow}/{classification}/{agent}/{purpose}@{depth}"
  description: "Project-wide session naming convention. Every session created through OpenCode SDK must use this format."
```

Schema shape (extended):
```typescript
interface GovernanceConfig {
  version: string
  defaultAgent: string
  agents: Record<string, AgentConfig>
  commands: Record<string, CommandConfig>
  templates: Record<string, TemplateConfig>
  naming_standards: {
    allowed_frameworks: string[]
    allowed_workflows: string[]
    allowed_classifications: string[]
    naming_format: string
    description: string
  }
}
```

Env var for config path override (optional): `HIVEMIND_GOVERNANCE_CONFIG_PATH`
Default: `.hivemind/governance/config.json` from project root

  <name>Task 5.2: Create typed config reader utility with naming standards types</name>
  <files>src/features/governance-engine/config-reader.ts</files>
  <action>
    Create `src/features/governance-engine/config-reader.ts` with:

    1. **Zod schema** for the extended governance config shape including naming_standards:
    ```typescript
    import { z } from "zod"

    export const NamingStandardsSchema = z.object({
      allowed_frameworks: z.array(z.string()),
      allowed_workflows: z.array(z.string()),
      allowed_classifications: z.array(z.string()),
      naming_format: z.string(),
      description: z.string(),
    })

    const AgentConfigSchema = z.object({
      description: z.string(),
      model: z.object({ providerID: z.string(), modelID: z.string() }).optional(),
      allowedCommands: z.array(z.string()),
      defaultBrief: z.string().optional(),
    })

    const CommandConfigSchema = z.object({
      description: z.string(),
      agent: z.string().optional(),
      template: z.string().optional(),
    })

    const TemplateConfigSchema = z.object({
      content: z.string(),
    })

    export const GovernanceConfigSchema = z.object({
      version: z.string(),
      defaultAgent: z.string(),
      naming_standards: NamingStandardsSchema.optional(),
      agents: z.record(AgentConfigSchema),
      commands: z.record(CommandConfigSchema),
      templates: z.record(TemplateConfigSchema),
    })
    ```

    2. **Type exports** inferred from Zod:
    ```typescript
    export type GovernanceConfig = z.infer<typeof GovernanceConfigSchema>
    export type NamingStandardsConfig = z.infer<typeof NamingStandardsSchema>
    export type AgentConfig = z.infer<typeof AgentConfigSchema>
    export type CommandConfig = z.infer<typeof CommandConfigSchema>
    ```

    3. **`readGovernanceConfig()` function:**
       - Reads `.hivemind/governance/config.json` from project root
       - Supports `HIVEMIND_GOVERNANCE_CONFIG_PATH` env var override
       - Validates with `GovernanceConfigSchema.safeParse()`
       - Returns typed `GovernanceConfig` or throws descriptive error
       - Resolves project root from `process.cwd()` (works in plugin context)

    4. **`resolveAgentForBrief(brief: string, config: GovernanceConfig): string` helper:**
       - Keyword matching: scan `config.agents` descriptions for matches in the brief
       - Returns matched agent name or `config.defaultAgent`
       - Simple case-insensitive substring matching (no NLP needed)

    5. **`validateNamingTitle(title: string, standards: NamingStandardsConfig): boolean` helper:**
       - Parses the title using simple string splitting (not importing the naming service — keeps config reader dependency-free)
       - Verifies `framework` is in `standards.allowed_frameworks`
       - Verifies `workflow` is in `standards.allowed_workflows`
       - Verifies `classification` is in `standards.allowed_classifications`
       - Returns `true` if all validations pass, `false` otherwise

    6. **JSDoc** on all exports per jsdoc-typescript-docs skill.

    Follow existing patterns:
    - Error messages use `[Harness]` prefix
    - `import { readFile } from "node:fs/promises"` for file I/O
    - Use Zod `safeParse` per existing tool validation patterns
