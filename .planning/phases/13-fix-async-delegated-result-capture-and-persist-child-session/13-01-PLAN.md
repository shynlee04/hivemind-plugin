# Plan 13-01: Types, Result Capture Module, and Continuity Wiring

## Goal

Create the `CapturedResult` type, the `result-capture.ts` pure extraction module, and wire it into the continuity persistence layer (clone + normalize). This plan delivers the reusable extraction primitives that Plan 13-02 will call from the observer and process runner.

## Requirements

- **PH13-01**: `CapturedResult` type defined in `types.ts` as optional field on `SessionContinuityMetadata`
- **PH13-02**: `captureSubsessionResult(client, sessionID)` extracts assistant text, tool call summary, artifact paths, and git commits from session messages
- **PH13-03**: `captureProcessResult(stdout, stderr)` wraps process output into `CapturedResult`
- **PH13-04**: Deep-clone-on-read for `CapturedResult` via `continuity-clone.ts`
- **PH13-05**: Disk normalization for `CapturedResult` via `continuity-normalizers.ts`

## <threat_model>

### STRIDE Analysis

| Threat | Category | Risk | Mitigation |
|--------|----------|------|------------|
| Malicious message content injected into result text | Tampering | MEDIUM | Truncate result text to 10KB max; sanitize newlines in artifact paths |
| Continuity JSON bloat from large captured results | Denial of Service | MEDIUM | Enforce `MAX_RESULT_TEXT_LENGTH = 10240` and `MAX_TOOL_CALL_SUMMARY_LENGTH = 50` |
| Normalizer drops malformed CapturedResult on reload | Information Disclosure | LOW | Graceful fallback: return `undefined` for individual fields, preserve the rest |
| Artifact path traversal via extracted file paths | Elevation of Privilege | LOW | Extracted paths are informational strings only — never used for filesystem operations |

</threat_model>

- src/lib/session-api.ts (for `getSessionMessages` signature at line 53-66 and `OpenCodeClient` type at line 5)

<action>
Create new file `src/lib/result-capture.ts` (~120 LOC) with these exports:

```typescript
import type { OpenCodeClient } from "./session-api.js"
import type { CapturedResult, ToolCallSummary } from "./types.js"

const MAX_RESULT_TEXT_LENGTH = 10240
const MAX_TOOL_CALL_ARGS_LENGTH = 200
const MAX_TOOL_CALL_SUMMARIES = 50
```

Implement these functions:

1. `export function extractAssistantText(messages: unknown[]): string`
   - Filter messages where `msg.role === "assistant"`
   - For each assistant message, iterate `msg.parts` array
   - Concatenate all parts where `part.type === "text"` and `typeof part.text === "string"`
   - Truncate final concatenation to `MAX_RESULT_TEXT_LENGTH`
   - Return empty string if no text parts found

2. `export function extractToolCallSummary(messages: unknown[]): ToolCallSummary[]`
   - Filter assistant messages
   - Iterate `msg.parts` for parts where `type === "tool-call" || type === "tool_call" || type === "tool"`
   - For each: extract `part.name` as `tool`, truncate `JSON.stringify(part.arguments ?? part.args)` to `MAX_TOOL_CALL_ARGS_LENGTH` as `args`
   - Cap result at `MAX_TOOL_CALL_SUMMARIES` entries
   - Return empty array if none found

3. `export function extractArtifactPaths(messages: unknown[]): string[]`
   - Filter assistant messages, iterate parts for tool-call parts
   - For tools named `"Write"`, `"Edit"`, `"read"`, `"write"`: extract `part.arguments.filePath` or `part.arguments.path` or `part.args.filePath` or `part.args.path`
   - Deduplicate with `Array.from(new Set(...))`
   - Return empty array if none found

4. `export function extractGitCommits(messages: unknown[]): string[]`
   - For each message (all roles), iterate parts
   - Look for tool-result parts where output contains `/[0-9a-f]{7,40}/` (git SHA pattern)
   - Also check tool-call parts for tools named "bash" or "Bash" where arguments contain "git commit" — extract SHA from result
   - Deduplicate
   - Return empty array if none found

5. `export async function captureSubsessionResult(client: OpenCodeClient, sessionID: string): Promise<CapturedResult>`
   - Call `getSessionMessages(client, sessionID)` from session-api.ts (import it)
   - Wrap entire body in try/catch — on failure return empty CapturedResult with `partial: true`
   - Pass messages to extractAssistantText, extractToolCallSummary, extractArtifactPaths, extractGitCommits
   - Return assembled CapturedResult with `messageCount: messages.length`, `capturedAt: Date.now()`

6. `export function captureProcessResult(stdout: string, stderr: string): CapturedResult`
   - Return CapturedResult where `resultText` = stdout truncated to MAX_RESULT_TEXT_LENGTH, `artifactPaths: []`, `gitCommits: []`, `toolCallSummary: []`, `messageCount: 1`, `capturedAt: Date.now()`

7. `export function truncateResultText(text: string): string`
   - If `text.length > MAX_RESULT_TEXT_LENGTH` return `text.slice(0, MAX_RESULT_TEXT_LENGTH) + "...[truncated]"`
   - Otherwise return text

All message parsing must be defensive: use `typeof` checks, optional chaining, and graceful fallback to empty arrays/strings. Never throw from extraction functions.
</action>

```typescript
function normalizeToolCallSummary(value: unknown): ToolCallSummary | undefined {
  if (!isRecord(value)) return undefined
  const tool = asString(value.tool)
  if (!tool) return undefined
  return { tool, args: asString(value.args) }
}

function normalizeCapturedResult(value: unknown): CapturedResult | undefined {
  if (!isRecord(value)) return undefined
  const resultText = asString(value.resultText)
  if (resultText === undefined) return undefined
  const artifactPaths = Array.isArray(value.artifactPaths)
    ? value.artifactPaths.map((e) => asString(e)).filter((e): e is string => Boolean(e))
    : []
  const gitCommits = Array.isArray(value.gitCommits)
    ? value.gitCommits.map((e) => asString(e)).filter((e): e is string => Boolean(e))
    : []
  const toolCallSummary = Array.isArray(value.toolCallSummary)
    ? value.toolCallSummary.map(normalizeToolCallSummary).filter((e): e is ToolCallSummary => Boolean(e))
    : []
  const messageCount = asNumber(value.messageCount)
  const capturedAt = asNumber(value.capturedAt)
  if (messageCount === undefined || capturedAt === undefined) return undefined
  return {
    resultText,
    artifactPaths,
    gitCommits,
    toolCallSummary,
    messageCount,
    capturedAt,
    partial: asBoolean(value.partial),
  }
}
```
