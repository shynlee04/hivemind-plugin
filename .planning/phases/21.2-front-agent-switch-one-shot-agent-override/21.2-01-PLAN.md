    | Case 1 — Synthetic parent prompt | gsd-progress | gsd-executor | subtask:false | mode:synthetic-parent-prompt, scheduled:true |

    **Execution protocol:**

    1. **Case 1 (primary UAT):** Call `execute-slash-command { command: "gsd-progress", agent: "gsd-executor", subtask: false }`.

       - After the tool returns, wait for the deferred `dispatchPromptAfterToolReturn` callback (fires after ~50ms via setTimeout).
       - Observe what happens next in the session. If OpenCode accepts the prompt for `gsd-executor`, the agent should respond. If rejected, an error appears. Document ALL observable outcomes.

    2. **Case 2 (baseline):** Call `execute-slash-command { command: "gsd-progress", agent: "gsd-executor", subtask: true }`.
       - Capture the immediate tool output. Verify metadata includes mode:"subtask".
       - Observe the deferred dispatch. This should create a SubtaskPartInput with `type: "subtask"`.

    3. **Case 3 (TUI control):** Call `execute-slash-command { command: "gsd-progress" }` — no agent or subtask override.
       - Capture immediate tool output. This should follow the TUI `clearPrompt → appendPrompt → submitPrompt` path.
       - Observe that the command text appears in the TUI prompt.

    Record ALL outputs, metadata fields, and post-dispatch observations for each case.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| execute-slash-command tool → client.session.prompt() | The synthetic parent prompt path sends a text part to a target agent via deferred setTimeout callback. This bypasses native command parsers and hook events. |
| prototype dispatch process | The setTimeout-based dispatch runs outside the synchronous tool execution boundary — errors are caught by a generic promise catch handler, not by the tool's error classification. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21.2-01 | Spoofing | `dispatchPromptAfterToolReturn` | accept | The prototype explicitly labels dispatch as `mode: "synthetic-parent-prompt"` — not native slash-command. No authentication claim is made. |
| T-21.2-02 | Information Disclosure | `console.error` in catch handler | accept | The error log only exposes error messages from OpenCode SDK, which are standard API error strings. No user secrets are logged. |
| T-21.2-03 | Denial of Service | Deferred prompt fails silently | accept | The catch handler logs the failure. The prototype explicitly states it is not production-ready. Production mitigation (retry, alerting) is deferred until after UAT verdict. |
| T-21.2-04 | Tampering | Non-mutation constraint | mitigate | The prototype plan (21.2-prototype-plan-2026-05-22.md) explicitly lists forbidden write surfaces. Validate by grep-checking that no `.opencode/commands/**` or `.opencode/agents/**` files were modified during this plan. |
</threat_model>
