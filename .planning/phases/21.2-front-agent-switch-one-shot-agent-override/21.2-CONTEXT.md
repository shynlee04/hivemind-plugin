<domain>
## Phase Boundary

Phase 21.2 is a feasibility and prototype-planning phase. It must not claim production runtime readiness until live UAT establishes whether the desired behavior is supported, partially supported, or impossible without upstream OpenCode changes.

The motivating case is `.opencode/commands/gsd-progress.md` with `gsd-executor`:

1. Run as a subtask with `gsd-executor` (proven in Phase 21.1).
2. Run in the main/front-facing context by temporarily treating `gsd-executor` as a primary/all agent for that one command execution (Phase 21.2 target).

</domain>

<decisions>
## Implementation Decisions

### D-01: Research-Locked Verdict
Phase 21.2 starts with verdict **PARTIAL / NEEDS PROTOTYPE** — research confirmed no SDK request-time `mode`/`subtask` override field exists. The only non-mutating prototype path is a synthetic parent-session `session.prompt({ agent, parts })` fallback using the expanded command body.

### D-02: Prototype Implemented (Option B)
The prototype path has been implemented: `execute-slash-command` treats explicit `subtask:false` plus a resolved agent as a synthetic parent prompt dispatch path. Metadata mode: `synthetic-parent-prompt`. Unit tests and typecheck pass at L3.

### D-03: Live UAT Required Before Production
Production implementation is blocked until L1 live UAT proves one of:
- **SUPPORTED:** `session.prompt({ agent: "gsd-executor" })` runs in parent session
- **PARTIAL:** Accepted but routes as subtask-like behavior
- **NOT SUPPORTED:** OpenCode rejects subagent-mode agents for direct parent prompts

### D-04: Non-Mutation Constraint
The prototype must not write to `.opencode/commands/**`, `.opencode/agents/**`, or global config counterparts. Any implementation requiring those mutations is out of scope for Phase 21.2.

### D-05: Four Candidate Solutions Evaluated
- **Option A** — Command overlay with `subtask:false` (preferred but blocked by SDK request shape)
- **Option B** — Synthetic main prompt with agent override (implemented as prototype)
- **Option C** — One-shot agent config overlay (deferred — no SDK seam found)
- **Option D** — File mutation / alias agent (rejected — violates non-mutation constraint)

### the agent's Discretion
- Choice of command discovery expansion approach
- Detail level of expanded command body
- Test naming and organization in the test file
- Which verification commands to include in the plan

</decisions>

## Specific Implementation Details

**Prototype path (`subtask:false + agent`):**
1. Discover command bundle through existing command discovery
2. Resolve target agent from input `agent` or command frontmatter `agent`
3. Expand `$ARGUMENTS` in command body
4. Schedule deferred `client.session.prompt()` after tool return
5. Send text part to requested agent — not literal `@agent /command` TUI text

**Changed files:**
- `src/tools/session/execute-slash-command.ts` — Added synthetic parent prompt dispatch
