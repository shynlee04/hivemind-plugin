# Phase 21.2 Prototype Plan — One-Shot Front-Agent Override

**Created:** 2026-05-22
**Status:** Prototype plan, not production implementation authorization
**Evidence level:** L4 source/type inspection plus live MCP documentation lookup; L1 live UAT still required

## Purpose

This plan defines the smallest safe prototype path for testing whether `execute-slash-command` can run a command body under a configured `mode: subagent` agent in the parent session, without mutating command or agent primitives.

## Current Evidence

| Evidence | Finding |
|----------|---------|
| Installed packages | `@opencode-ai/sdk@1.15.5` and `@opencode-ai/plugin@1.15.5` are installed. |
| SDK type `SessionCommandData` | Request body exposes `messageID`, `agent`, `model`, `arguments`, and `command` only. No `subtask`, `mode`, or `agentMode` field exists. |
| SDK type `SessionPromptData` | Request body exposes `agent`, `model`, `noReply`, `system`, `tools`, and `parts`; parts may include text, file, agent, or subtask inputs. |
| DeepWiki live source query | OpenCode command execution computes subtask behavior from resolved `agent.mode` and command config `cmd.subtask`; no request-time mode/subtask override is exposed. |
| Context7 SDK docs | SDK prompt/chat style APIs accept an agent field for direct prompt dispatch, but command overlay semantics are not documented. |

## Prototype Options

### Option A — Native Command Overlay

Attempt to make `session.command()` see an effective command config where `subtask:false` and `agent: gsd-executor` are applied only for one execution.

**Current status:** Blocked by SDK request shape. `SessionCommandData.body` does not expose a `subtask` field, and OpenCode command loading reads command config from the command registry.

**Stop condition:** If no plugin hook or command registry overlay seam exists before `isSubtask` is computed, do not implement Option A.

### Option B — Deferred Synthetic Parent Prompt

Load the command primitive through existing command discovery, expand `$ARGUMENTS`, and defer a direct `client.session.prompt()` call with:

```ts
body: {
  agent: "gsd-executor",
  parts: [{ type: "text", text: expandedCommandBody }]
}
```

This bypasses native slash-command execution and is explicitly a synthetic fallback. It does not mutate `.opencode/commands/**` or `.opencode/agents/**`.

**Expected value:** Tests whether OpenCode accepts `session.prompt({ agent: "gsd-executor" })` for an agent configured as `mode: subagent`, and whether that dispatch stays in the parent session.

**Risks:** It bypasses command parser events and native command hooks. It must be labeled in tool metadata as `mode: "synthetic-parent-prompt"`, not as native slash-command execution.

**Stop condition:** If OpenCode rejects subagent-mode agents for direct parent prompts or routes them as subtasks anyway, mark Phase 21.2 as NOT SUPPORTED without upstream OpenCode changes.

### Option C — Runtime Agent Config Overlay

Temporarily override resolved agent config so `gsd-executor.mode` becomes `all` or `primary` only for one dispatch.

**Current status:** Not authorized for implementation. No SDK or plugin seam has been confirmed.

**Stop condition:** Do not implement unless live OpenCode source inspection finds a supported in-memory config overlay point with cleanup guarantees.

## Recommended Prototype Sequence

1. Add unit coverage for explicit `subtask:false` plus `agent` on `execute-slash-command` to verify the tool schedules a synthetic parent prompt rather than TUI `@agent /command` text.
2. Implement only the minimal Option B prototype path behind the existing `subtask:false` input.
3. Verify no `.opencode/commands/**` or `.opencode/agents/**` files are changed.
4. Run focused unit tests: `npx vitest run tests/tools/execute-slash-command.test.ts`.
5. Run typecheck: `npm run typecheck`.
6. Restart OpenCode/harness and run live UAT with `gsd-progress`, `gsd-executor`, and `subtask:false`.
7. Inspect session/delegation evidence to determine whether the result was parent-session execution or subtask routing.

## Live UAT Matrix

| Case | Command | Agent | Input | Expected evidence |
|------|---------|-------|-------|-------------------|
| Baseline subtask | `gsd-progress` | `gsd-executor` | `subtask:true` | Child subtask/delegation evidence exists. |
| Synthetic parent prompt | `gsd-progress` | `gsd-executor` | `subtask:false` | Parent session receives the expanded command body as a direct prompt, or OpenCode rejects it with an explicit error. |
| TUI control | `gsd-progress` | `gsd-executor` | no `subtask` override | TUI prompt text is used; no claim of front-agent switching. |

## Acceptance for Prototype Completion

- Unit tests prove the tool selects the intended dispatch path and uses deferred prompt scheduling.
- Typecheck passes after prototype changes.
- Live UAT records whether `session.prompt({ agent: "gsd-executor" })` is accepted and where execution occurs.
- Runtime readiness is not claimed unless live UAT proves parent-session execution.
- If rejected or routed as subtask, the phase result becomes NOT SUPPORTED / UPSTREAM GAP.

## Non-Mutation Guard

The prototype must not write to:

- `.opencode/commands/**`
- `.opencode/command/**`
- `.opencode/agents/**`
- `.opencode/agent/**`
- `~/.config/opencode/commands/**`
- `~/.config/opencode/agents/**`

Any implementation that requires those mutations is out of scope for Phase 21.2.
