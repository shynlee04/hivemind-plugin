# Phase 21.2 Research — One-Shot Front-Agent Mode Override Feasibility

**Created:** 2026-05-22  
**Status:** Research locked — verdict PARTIAL / NEEDS LIVE UAT  
**Research quality:** B — live OpenCode source plus local codebase scan and subagent research agreement  

## Research Question

Can Hivemind run a command under a configured `mode: subagent` agent as a main/front-facing agent for exactly one execution, without mutating either the command primitive or the agent primitive?

## Evidence Sources

| Source | Evidence |
|--------|----------|
| `package.json` | `@opencode-ai/sdk` and `@opencode-ai/plugin` `^1.15.5` |
| `.hivemind/STACKS-REFERENCES.md` | Active OpenCode source repo is `anomalyco/opencode` |
| Context7 `/anomalyco/opencode` | Commands support `agent` and `subtask`; subagent agents default to subtask invocation; `subtask:false` disables at command config level |
| Raw OpenCode source `session/prompt.ts` | `command()` computes `isSubtask = (agent.mode === "subagent" && cmd.subtask !== false) \|\| cmd.subtask === true` |
| Raw OpenCode source `agent/agent.ts` | Agent mode is part of resolved config; default mode for new config agents is `all` |
| Local `.opencode/agents/gsd-executor.md` | `gsd-executor` is configured as `mode: subagent` |
| Subagent research `ses_1b0f660b5ffe5mzCW3YQrvpw4C` | Independent verdict: PARTIAL / NEEDS PROTOTYPE |

## Key Source Finding

OpenCode command execution computes subtask behavior:
```ts
const agent = agentName ? yield* agents.get(agentName) : yield* agents.defaultInfo()
const isSubtask = (agent.mode === "subagent" && cmd.subtask !== false) || cmd.subtask === true
const userAgent = isSubtask ? (input.agent ?? (yield* agents.defaultInfo()).name) : agent.name
```

## Feasibility Verdict

**Verdict:** PARTIAL / NEEDS LIVE UAT.

The platform supports one-shot subtask dispatch (Phase 21.1). The prototype for synthetic parent prompt (`session.prompt({ agent, parts })`) has been implemented with L3 evidence. Live UAT must still confirm whether OpenCode accepts a subagent-mode agent for direct parent prompt execution.

## Candidate Solutions

| Option | Approach | Status |
|--------|----------|--------|
| **A** | Command overlay with `subtask:false` | Blocked — SDK `SessionCommandData` has no `subtask` request field |

| **C** | Runtime agent config overlay | Deferred — no SDK seam confirmed |
