<objective>
Convert the tmux integration factory at `src/features/tmux/integration.ts:139-179` to use a runtime `existsSync` check for the fork package directory, replacing the current implicit assumption that any fork-bridge wiring will only run when the fork is actually present.

Purpose: Currently, `createTmuxIntegrationIfSupported` directly imports `setForkSessionManager` from `./fork-bridge.js` and unconditionally calls it (line 165) when the caller passes an adapter. This is safe when the fork is absent (the import resolves to a Hivemind-side stub) but fragile when the caller passes an adapter that comes from the fork — if the fork's plugin entry is not loaded (e.g., fork package not installed, or HMR-style re-init failed), the adapter may be stale or undefined. The `existsSync` check makes the fork presence a runtime decision, not an implicit import-time decision.

Output: One import line added (existsSync is already imported at L9 — the import line is already there; the action is to USE it in the factory). One new conditional check in the factory. No new files, no new tests, no logic changes beyond the existsSync guard.
