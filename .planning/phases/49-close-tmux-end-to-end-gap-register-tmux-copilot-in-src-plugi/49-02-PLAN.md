<objective>
Replace the unconditional noop fallback at the tmux event observer site (`src/plugin.ts:594-595`) with a nullish-coalescing lookup that prefers the real fork adapter (set via `setForkSessionManager` by the fork's plugin entry) and falls back to the existing noop stub. This closes the runtime-injection boundary for REQ-05.

Purpose: Currently, even when the fork is fully wired and a real `ForkSessionManager` is set via `setForkSessionManager(adapter)`, the plugin's observer still calls `buildNoopForkSessionManager()` and discards the enriched event. This plan wires the observer to use the real adapter if present, while preserving the noop as a safety net for builds where the fork is absent. This is the Hivemind-side mirror of the fork's `setForkSessionManager(adapter)` call.

Output: ONE line edit in `src/plugin.ts` (L595 only — L594 unchanged). One new import line (L52 area). No new files, no new tests, no logic changes.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-49-02-01 | Spoofing | getForkSessionManager singleton | accept | The singleton is process-local; an attacker who can set it has already compromised the OpenCode process. The threat surface is the same as the fork's plugin entry itself. |
| T-49-02-02 | Denial of Service | Real adapter throws on dispatch | mitigate | The noop fallback ensures plugin init never blocks on a missing/faulty adapter. The fork's adapter is responsible for its own try/catch (already P43-mitigated via tmux-copilot's structured error results). |
| T-49-02-03 | Information Disclosure | Fork adapter receives enriched event | accept | The enriched event contains delegation metadata + lastMessage excerpt. This is intentional — the fork needs the metadata to propagate `hivemindMeta` on respawn (REQ-06). No secrets in the event by design. |
| T-49-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages. Only an import of an existing function from the monorepo. |
