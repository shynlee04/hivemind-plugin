## Implementation Details

### `scripts/sync-assets.js` (228 insertions, 79 deletions)

**New sections:**

1. **Mode detection** (top of file): Parses `process.argv.slice(2)` for `"--mode=install"`, sets `installMode` boolean and `logPrefix` (`[postinstall]` vs `[Harness Build]`).

2. **Dual-root resolution**: `stage` object with `sourceRoot` (always `process.cwd()`) and `consumerRoot` (`process.env.INIT_CWD ?? process.cwd()` in install mode, `process.cwd()` otherwise). `PRIMITIVE_MAP` targets resolve from `stage.consumerRoot`.

3. **Version drift hot path** (install mode only): Reads `package.json` version from `stage.sourceRoot`, checks `.hivemind/state/version.json` at `stage.consumerRoot`. If versions match, logs and calls `process.exit(0)`. Wrapped in try/catch тАФ falls through on error.

4. **Non-destructive merge loop**: When `installMode` is true, replaces the destructive `rmSync`+`mkdirSync`+`cpSync` pattern with per-file content comparison. For each primitive:
   - Target missing тЖТ copy from assets
   - Target exists, content same тЖТ skip (no-op)
   - Target exists, content differs тЖТ backup to `.backup`, skip
   - Binary/unreadable тЖТ skip

5. **Command mirror non-destructive**: Mirror section (commands/ тЖТ command/) also gated on `installMode`. In install mode: compare content, skip identical, backup+skip different. Build mode: unchanged.

6. **Version stamp write**: After successful install-mode sync, writes `.hivemind/state/version.json` with the current package version.

7. **Try/catch wrapper**: Entire main body wrapped so postinstall errors are caught and logged without failing the npm install (npm treats non-zero exit on postinstall as non-fatal).
