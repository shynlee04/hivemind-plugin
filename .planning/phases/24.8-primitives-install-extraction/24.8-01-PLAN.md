- Modified `package.json` with `"postinstall"` hook

    With a mode-gated branch:
    - If `installMode`: call a non-destructive merge function (see below) — do NOT `rmSync`, do NOT `mkdirSync`
    - If `!installMode`: keep the existing build-mode cleanup + `rmSync` + `mkdirSync` as-is (identical logic)

    2. **Install-mode merge logic** (replaces the destructive cleanup):

    ```js
    if (installMode) {
      // INSTALL MODE: Non-destructive per-file merge
      const label = "[postinstall]";
      mkdirSync(targetDir, { recursive: true });  // Ensure target exists
      for (const entry of readdirSync(sourceDir)) {
        if (entry === ".gitkeep" || entry.startsWith("gsd-") || entry === "gsd") continue;
        const srcPath = join(sourceDir, entry);
        const destPath = join(targetDir, entry);
        if (existsSync(destPath)) {
          try {
            const existingContent = readFileSync(destPath, "utf-8");
            const sourceContent = readFileSync(srcPath, "utf-8");
            if (existingContent === sourceContent) {
              continue;  // Identical — skip (no-op)
            }
            // Content differs — user has modified this file
            const backupPath = destPath + ".backup";
            if (!existsSync(backupPath)) {
              cpSync(destPath, backupPath, { recursive: true });
              console.log(`${label} ⚠ Preserved user-modified ${entry} → ${entry}.backup`);
            }
            console.log(`${label} ⚠ Skipping ${kind}/${entry} — user-modified. Backup: ${entry}.backup`);
            continue;
          } catch {
            // Binary or unreadable — skip diff check
            continue;
          }
        }
        // Target doesn't exist — fresh copy
        cpSync(srcPath, destPath, { recursive: true });
        if (kind === "commands") mirrorCount++;
      }
      console.log(`${label} Reflected ${kind} (install-mode, non-destructive)`);
      continue;  // Skip the build-mode cleanup below
    }
    // BUILD MODE: ...existing logic unchanged...
    ```

    3. **Command mirror section changes** (lines 69-104):

    The mirror section already has partial backup logic but assumes destructive overwrite (`cpSync` without checking). For install mode, wrap the entire mirror section:
    ```js
    if (mirrorCount > 0) {
      const commandsDir = PRIMITIVE_MAP.commands;
      const commandMirrorDir = join(stage.consumerRoot, ".opencode", "command");
      ...
      // Inside the entry loop:
      if (existsSync(destPath)) {
        if (installMode) {

          try {
            const existingContent = readFileSync(destPath, "utf-8");
            const sourceContent = readFileSync(srcPath, "utf-8");

            // Differs — backup + skip
            const backupPath = destPath + ".backup";
            if (!existsSync(backupPath)) {
              cpSync(destPath, backupPath, { recursive: true });
            }
            continue;
          } catch { /* skip */ }
        } else {
          // Build mode: existing backup + overwrite logic (unchanged)
          ...
        }
      }
      if (!installMode || !existsSync(destPath)) {
        cpSync(srcPath, destPath, { recursive: true });
        mirrored++;
      }
    ```

## Trust Boundaries

|----------|-------------|
| package `assets/` → consumer `.opencode/` | Shipped files are copied from the npm package to the consumer project |
| postinstall → filesystem | npm lifecycle script has write access to consumer project |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24.8-01 | Tampering | scripts/sync-assets.js install-mode | mitigate | Source files are read from the package's own `assets/` directory (bundled in tarball, signed by npm); consumer target is writable only by the user running npm install |
| T-24.8-02 | Denial of Service | postinstall script | mitigate | Entire script wrapped in try/catch — npm treats postinstall errors as non-fatal (logs but continues install) |
| T-24.8-03 | Information Disclosure | .backup files | accept | Backup files are plain text copies of user's own files in their own project; no sensitive data leakage |
| T-24.8-04 | Repudiation | File overwrite / user data loss | mitigate | Non-destructive merge: user-modified files are backed up (`.backup`) and skipped — no data loss |
| T-24.8-SC | Tampering | npm package install | mitigate | slopcheck: assets/ files are shipped by npm registry; postinstall script is declared in package.json manifest |
