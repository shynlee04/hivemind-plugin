**Created:** 2026-05-29

## Goal

Register all 23 plugin tools in the pressure authority matrix. Eliminate `unknownToolFallback` usage by giving every tool an explicit `pressureBehavior` entry. Add a coverage test to prevent future regressions.

## Background

### Current State: Partial Matrix

The authority matrix (`src/features/runtime-pressure/authority-matrix.ts`) registers 16 of 23 tools. When `detectRuntimePressure()` receives an unregistered tool name, it falls back to `unknownToolFallback()` — a conservative band default that treats every unknown tool as if it were a mutating executor. This is correct as a safety net but masks tool-specific risk profiles.

### Existing Patterns

The matrix uses 4 reusable behavior presets:

| Preset | steady | advisory | gated | blocking | Used by |
|--------|--------|----------|-------|----------|---------|
| `HIVEMIND_STATE_WRITER` | allow | advise | require_approval | block | session-patch, hivemind-trajectory, hivemind-pressure, hivemind-agent-work-create |
| `OPENCODE_PRIMITIVE_WRITER` | allow | advise | require_approval | block | configure-primitive |
| `READ_ONLY_INSPECTOR` | allow | allow | advise | defer | delegation-status, prompt-skim, prompt-analyze, session-journal-export, hivemind-doc, hivemind-sdk-supervisor, hivemind-command-engine, hivemind-agent-work-export, validate-restart |
| `EXTERNAL_COMMAND_RUNNER` | allow | advise | require_approval | block | delegate-task, run-background-command |

### Decision Logic

`detectRuntimePressure()` in `control-plane.ts`:
1. Classifies pressure into tier + band
2. Looks up tool via `getToolAuthority(toolName)`
3. If tool found → uses `tool.pressureBehavior[band]`
4. If tool NOT found → calls `unknownToolFallback(band)` (conservative default)

After this phase, step 4 should never trigger for registered tools.
