  <action>
    Create `scripts/sync-fork.sh` (`chmod +x`) with the following implementation. Follow the `set -euo pipefail` pattern from `scripts/sync-oss.sh`. Use explicit error handling for `git merge-tree` (do NOT rely solely on `set -e`). Reference the complete code pattern from 45-RESEARCH.md §Code Examples:Example 1.

    ## Requirements to implement:

    1. **Shebang and strict mode:** `#!/usr/bin/env bash` + `set -euo pipefail`

    2. **Configuration constants:**
       - `REMOTE_NAME="hivemind-fork-temp"` — temp remote name for fetch
       - `REMOTE_URL="${SYNC_FORK_REMOTE_URL:-https://github.com/shynlee04/opencode-tmux.git}"` — overridable via env var for CI/testing
       - `FORK_BRANCH="${SYNC_FORK_BRANCH:-main}"` — overridable via env var
       - `VENDOR_DIR="opencode-tmux"` — vendored directory
       - `PINNED_FILES` array with exactly 4 entries (per D-01): `src/session-manager.ts`, `src/grid-planner.ts`, `src/__tests__/grid-planner.test.ts`, `src/__tests__/session-manager.test.ts`
       - Add a comment above PINNED_FILES: `# If you add or remove Hivemind-pinned files, update this list`.

    3. **Argument parsing:** Only accept `--dry-run`. Any other argument → `echo "Usage: $0 [--dry-run]" >&2; exit 1`.

    4. **Cleanup trap:** `trap cleanup EXIT` where `cleanup()` removes the temp remote (`git remote remove "$REMOTE_NAME" 2>/dev/null || true`). Also handle the case where a `git merge` was aborted mid-flight — add `git merge --abort 2>/dev/null || true` in cleanup.

    5. **Step 1 — Add temp remote:** Check existence via `git remote get-url "$REMOTE_NAME" &>/dev/null`. If absent, `git remote add --no-fetch "$REMOTE_NAME" "$REMOTE_URL"`. Print status to stdout. (per REQ-04 idempotency)

    6. **Step 2 — Fetch upstream:** `git fetch "$REMOTE_NAME" "refs/heads/$FORK_BRANCH"`. Print progress. Handle fetch failure gracefully (exit with error message).

    7. **Step 3 — Conflict detection via `git merge-tree --write-tree --name-only --no-messages HEAD FETCH_HEAD`:**
       - Capture output and exit code separately.
       - Exit code 0: clean — proceed.
       - Exit code 1 (has conflicts): parse output (skip first line = tree OID), iterate `PINNED_FILES` using `grep -Fxq "$pinned"` against conflicted files. If ANY pinned file found, print `"ERROR: Pinned file 'opencode-tmux/$pinned' has merge conflicts."` (per file) to stderr, then `exit 1`. If conflicts exist but none are pinned files, print `"→ Non-pinned conflicts detected (safe to merge)."` and proceed.
       - Exit code 2+: print `"ERROR: git merge-tree failed (exit $rc): $merge_output"` to stderr, exit with `$rc`.
       - **Important:** `git merge-tree` output root path is repo-root-relative. The PINNED_FILES paths are relative to `opencode-tmux/`. When checking, construct the full path: `"$VENDOR_DIR/$pinned"`.

    8. **Step 4 — Count incoming commits:** `git log --oneline HEAD..FETCH_HEAD 2>/dev/null | wc -l | tr -d ' '`. Print count.

    9. **Step 5a — Dry-run mode:** If `--dry-run`, print summary including commit count and "No changes made." Exit 0 if no pinned conflicts, exit 1 if pinned conflicts were detected (code already handles this). (per REQ-03)

    10. **Step 5b — Merge:** If not dry-run, `git merge FETCH_HEAD --no-edit --no-ff -m "sync: merge upstream opencode-tmux changes"`. Print success message with commit count. (per REQ-01, REQ-04)

    11. **Cleanup via trap:** temp remote removal + merge abort cleanup runs automatically on EXIT.

    ## Edge cases to handle:
    - Temp remote already exists (user has it for some reason) → reuse it
    - Fetch fails (no network) → exit with meaningful error
    - merge-tree returns unexpected format → don't crash, check exit code first
    - SIGINT during merge → trap handles cleanup
    - `opencode-tmux/` directory doesn't exist → fail early with clear message
    - Script called from wrong directory (not monorepo root) → fail if `opencode-tmux/` not found

    ## What to AVOID:
    - ❌ Do NOT hardcode REMOTE_URL without env var override (breaks testability per Assumption 6 resolution)
    - ❌ Do NOT use `git merge --abort` as conflict detection mechanism — use `merge-tree` (zero side effects per RESEARCH)
    - ❌ Do NOT attempt auto-resolution of pinned file conflicts — SPEC explicitly forbids it
    - ❌ Do NOT leave the temp remote on exit in any code path (trap guarantees this)
    - ❌ Do NOT use `git merge-tree` output parsing as the sole conflict indicator — check exit code FIRST (per RESEARCH Pitfall 3)
    - ❌ Do NOT use `command substitution $()` without explicit exit code check for merge-tree (per RESEARCH Pitfall 2)
    - ❌ Do NOT add `-v` or `--verbose` flags — only `--dry-run` is supported (per SPEC)
