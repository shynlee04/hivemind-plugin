<objective>
Fix `.github/workflows/sync-oss.yml` to prevent internal artifacts leaking to the public OSS branch. Switch from auto-detect (exclusion-based) to explicit whitelist (inclusion-based) model.

Purpose: The current sync-oss.yml auto-detects ALL non-dot top-level directories and explicitly adds `.opencode/`, causing internal dev tooling (GSD, get-shit-done/), runtime state (`.hivemind/`, `.planning/`), and external-tool symlinks (`/tmp/arxitect/`) to sync to the public `oss-dev` branch. 

Key insight: `.opencode/` is NOT source code — it is a runtime manifestation. Users get it via `npm install` → postinstall → `scripts/sync-assets.js --mode=install` copies from `assets/`. The OSS branch should contain only the SOURCE that generates the package: `src/`, `assets/`, `bin/`, `scripts/`, `tests/`, and public documentation.

Output: Rewritten sync-oss.yml with explicit whitelist, GSD/internal artifact exclusion, symlink dereference verification.
</objective>
