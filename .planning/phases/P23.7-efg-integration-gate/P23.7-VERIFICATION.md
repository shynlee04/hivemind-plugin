## Additional Minor Issues

### Issue A1: `configure-primitive compile --dryRun` output validity not verified

**Risk:** LOW — the `configure-primitive` tool is registered at `plugin.ts:182`, its Zod input schema has 11 fields, and it imports `compileAgent`/`compileCommand`/`compileSkill` from `src/config/compiler.ts`. The compile → OpenCode validation cycle was not executed in the research session.

**Recommendation:** Add `configure-primitive compile --dryRun` followed by `validate-restart` call in P28 phase gate verification to confirm compiled output passes OpenCode runtime validation. Tracking: DEFERRED to P28.

**Evidence ref:** `P23-7-efg-integration-research-2026-05-29.md` lines 203, 209-212.

### Issue A2: Governance engine fallback path creates sessions but doesn't auto-dispatch

**Risk:** LOW — the `create-governance-session` tool falls back to raw `sendPrompt` if no coordinator is provided (`plugin.ts` registers it without coordinator). The fallback path creates a session and injects a prompt but doesn't dispatch to the named agent — the user must manually interact.

**Recommendation:** Acceptable for P23.7 gate — the tool is wired correctly. Agent dispatch via coordinator is a P27/P28 concern. Tracking: DEFERRED to P27.

**Evidence ref:** `P23-7-efg-integration-research-2026-05-29.md` lines 214-217.
