## Minor Issues

### A1 (LOW): `configure-primitive compile --dryRun` output validity not formally verified

**Issue:** The `configure-primitive` tool's `compile` → `dryRun` → `write` cycle was verified through source code inspection (tool registered at plugin.ts:182 with 11-field Zod schema, imports `compileAgent`/`compileCommand`/`compileSkill` from `src/config/compiler.ts`), but a runtime dryRun execution was not performed in this verification.

**Risk:** LOW — the tool is properly registered with full schema validation. The compiler imported from `src/config/compiler.ts` is used by the same code path as other verified tools. A dryRun validation would only confirm the output format matches schema expectations.

**Deferral target:** P28 — Hook Injection Plane Redesign will exercise the full compile→write cycle during hook-command integration testing.

### A2 (LOW): Governance engine fallback without auto-dispatch

**Issue:** The `create-governance-session` tool's fallback path (used when no coordinator is provided) creates a session and injects a governance prompt via `session.prompt()`, but does not auto-dispatch to the named agent. The user must manually interact with the session.

**Risk:** LOW — the tool is wired correctly. Creating the session + injecting the prompt is the expected behavior for a prototype phase. Agent dispatch without a coordinator is a workflow design decision, not a tool defect. The coordinator path (P27/P28) will add auto-dispatch capability.

**Deferral target:** P27 — Routing/Intent Loop will formalize coordinator-based dispatch. P28 will add hook-command integration.
