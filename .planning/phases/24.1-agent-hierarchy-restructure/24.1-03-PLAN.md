## Workflow Format

Workflow files define the end-to-end process for a command. They contain:

1. **Goal** — What the workflow achieves
2. **Agent routing table** — Which agents are involved and in what order
3. **Execution phases** — Step-by-step process with gates
4. **Checkpoint protocol** — How different checkpoints are handled
5. **Output contract** — What artifacts are produced

Workflows are the authoritative dispatch layer:
- Commands reference workflows via `<execution_context>`
- Workflows contain all routing, gate, and checkpoint logic
- Agents are spawned by the workflow, not the command
