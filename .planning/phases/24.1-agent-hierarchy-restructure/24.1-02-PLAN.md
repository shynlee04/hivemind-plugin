## Command Format Reference

Based on existing gsd-* command files, the format is:

```yaml
---
description: [one-line command purpose]
argument-hint: "<phase-number> [flags]"
requires: [list of required preconditions]
execution_context:
  @path/to/workflow.md
---
<objective>
[What this command achieves]
</objective>

<context>
Phase: $ARGUMENTS
[relevant context for execution]
</context>

<process>
Execute end-to-end.
Preserve all workflow gates.
</process>
```

Key differences from gsd commands:
- hm-* commands reference hm-* workflows, not GSD workflows
- hm-* commands route to hm-* agents (not gsd-* agents)
- All commands use the `.opencode/command/` directory
