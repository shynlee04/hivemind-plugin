# Phase 25.2: Trajectory Immutability Guard — Context

**Gathered:** 2026-05-29
**Status:** Ready for spec

<domain>
## Phase Boundary

Prevent mutations on closed trajectories. All trajectory mutation operations (eventTrajectory, attachTrajectoryEvidence, checkpointTrajectory) must throw when the trajectory status is "closed". Currently, closeTrajectory() sets status to "closed" but no guard prevents subsequent mutations — evidence integrity is compromised.
</domain>

<decisions>
## Implementation Decisions

- **D-01:** Guard check at top of each mutation function
- **D-02:** Error message format: `[Harness] trajectory is closed: {trajectoryId}`
- **D-03:** `closeTrajectory()` itself should throw if already closed (idempotent close is a bug)
- **D-04:** Tests for each guarded operation (3+ test cases)
