### `upsertTrajectory()` Helper (line 136)

Returns existing trajectory without checking status. This is the root cause — all three mutation functions delegate to it.

```typescript
function upsertTrajectory(ledger, input, now): TrajectoryRecord {
  const existing = ledger.trajectories[input.trajectoryId]
  if (existing) {
    // Updates lineage fields but NEVER checks status
    existing.rootSessionId = input.rootSessionId ?? existing.rootSessionId
    // ...
    return existing
  }
  // Creates new trajectory with status: "active"
}
```

## Guard Placement Strategy

### Option A: Guard in `upsertTrajectory()` (rejected)

Would require changing the helper signature to accept a `requiredStatus` parameter. Over-engineering for 4 call sites.

### Option B: Guard at top of each mutation function (selected)

Each function reads the ledger, then checks status before calling `upsertTrajectory()`. Clean, explicit, no shared state changes.

**Pattern for `attachTrajectoryEvidence`, `checkpointTrajectory`, `eventTrajectory`:**

```typescript
export function attachTrajectoryEvidence(input) {
  const ledger = readTrajectoryLedger(input.projectRoot)
  const existing = ledger.trajectories[input.trajectoryId]
  if (existing?.status === "closed") {
    throw new Error(`[Harness] trajectory is closed: ${input.trajectoryId}`)
  }
  // ... rest of function
}
```

**Pattern for `closeTrajectory()`:**

```typescript
export function closeTrajectory(input) {
  const ledger = readTrajectoryLedger(input.projectRoot)
  const trajectory = ledger.trajectories[input.trajectoryId]
  if (!trajectory) {
    throw new Error(`[Harness] Trajectory "${input.trajectoryId}" not found`)
  }
  if (trajectory.status === "closed") {
    throw new Error(`[Harness] trajectory is already closed: ${input.trajectoryId}`)
  }
  // ... rest of function
}
```

### Helper Extraction Decision

**Decision: Inline checks.** Only 4 guard sites. A shared `assertTrajectoryOpen()` helper adds indirection without meaningful deduplication. The `closeTrajectory()` guard has a different error message ("already closed") which further reduces shared logic.
