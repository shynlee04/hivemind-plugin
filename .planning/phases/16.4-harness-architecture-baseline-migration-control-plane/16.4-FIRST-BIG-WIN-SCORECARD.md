# Phase 16.4 First Big Win Scorecard

Reader: future roadmap/planning agents deciding the next architecture-aligned build after Phase 16.4. Post-read action: pick the first build/migration target without resuming paused or risky work prematurely.

## Scoring Criteria

Scores use 1–5 where 5 is best. For `State Risk`, lower risk receives higher score.

| Criterion | 1 | 3 | 5 |
|---|---|---|---|
| User Value | Mostly internal cleanup | Improves agent/operator visibility | Clearly improves daily human/agent collaboration |
| Architecture Leverage | Local improvement only | Helps one subsystem | Unlocks multiple later phases safely |
| Reversibility | Hard to revert or migrates canonical state early | Reversible with effort | Projection/contract-first and easy to discard |
| State Risk | Writes new canonical state or creates dual authority | Adds derived state with some coupling | Audit/projection first; no runtime authority replacement |
| OpenCode Fit | Fights platform lifecycle | Uses platform with adapters | Aligns with SDK/hooks/tools/permissions boundaries |
| Testability | Hard to assert | Artifact/contract tests possible | Clear unit/static/integration tests and guard checks |

## Recommendation

Select `session journal + execution lineage bridge` as the first architecture-aligned migration/build win after Phase 16.4. This implements D-21 and D-22: it provides delegation awareness, human context, future sidecar read models, product-detox concept migration support, and cross-session continuity while remaining audit/projection-first instead of replacing runtime truth.

`delegation manifest/notification mediation` remains important but paused until state ownership, runtime classification, and journal/lineage decisions are locked per D-23. Product-detox concepts are migrated as concepts/contracts first and no product-detox code is copied as-is per D-24.

## Deferred Candidate Rationale

- `delegation manifest/notification mediation`: defer until journal/lineage can provide mediated context rather than raw reminder firehose.
- `doc intelligence`: defer because Phase 16.4 guard/register artifacts already cover immediate stale-doc prevention.
- `agent-work-contract schema`: defer until journal entries and delegation evidence define the minimum contract fields.
- `task/trajectory graph`: defer until projections are rebuildable and not competing canonical workflow state.
- `sidecar visibility`: defer until read-model boundaries and source projections are stable.
