# SPEC: Delegations.json

> **Date:** 2026-05-31
> **Status:** Draft
> **Source:** Code audit of 12 source files in `src/`

## 1. Purpose (one sentence)

`delegations.json` is the durable persistence file that records all delegation operations (SDK child sessions and command processes) so that in-memory state can survive harness restarts, tools can query delegation history, and recovery flows can resume pending delegations.

## 4. Data Schema

```typescript
interface DelegationRecord {
  // --- Identity & Linkage ---
  /** Unique identifier. v1: crypto.randomUUID(). v2: "dt-{timestamp}-{random}". Written by coordinator / manager-runtime at dispatch time. */
  id: string
  /** Parent OpenCode session ID (ses_xxx). Written at dispatch. */
  parentSessionId: string
  /** Child OpenCode SDK session ID (ses_xxx). Written at dispatch, may be updated by DelegationCoordinator.attachChildSession(). */
  childSessionId: string
  /** Agent name resolved at dispatch (e.g., "hm-l2-scout"). Written at dispatch. */
  agent: string

  // --- Status & Lifecycle ---
  /** Current delegation status. Written by DelegationStateMachine.transition() and transitionToTerminal(). */
  status: "dispatched" | "running" | "completed" | "error" | "timeout"
  /** Unix ms timestamp of record creation. Written at dispatch. */
  createdAt: number
  /** Unix ms timestamp when delegation reached terminal state. Written by transitionToTerminal(). */
  completedAt?: number

  // --- Terminal Detail ---
  /** Machine-readable terminal subcategory. Written by transitionToTerminal() or markCommandCancellationForPtySession(). */
  terminalKind?: "completed" | "error" | "timeout" | "cancelled" | "restarted" | "runtime-dispatch-unsupported" | "interrupted-by-signal" | "non-resumable-after-restart"
  /** OS signal that terminated the child (SIGINT, SIGTERM, etc.). Written by transitionToTerminal(). */
  terminationSignal?: string
  /** True when the user explicitly cancelled. Written by transitionToTerminal() or markCommandCancellationForPtySession(). */
  explicitCancellation?: boolean

  // --- Execution Mode & Guarantees ---
  /** How the delegation runs. Written at dispatch. */
  executionMode: "sdk" | "pty" | "headless"
  /** Derived from executionMode. Written at dispatch or normalized on read. */
  surface?: "agent-delegation" | "command-process"
  /** Recovery guarantee classification. Written at dispatch or normalized on read. */
  recoveryGuarantee?: "resumable" | "best-effort" | "non-resumable-after-restart"
  /** Working directory for the delegated session. Written at dispatch. */
  workingDirectory: string
  /** PTY session ID for pty-mode delegations. Written by CommandDelegationHandler. */
  ptySessionId?: string
  /** Why headless fallback was used. Written by CommandDelegationHandler. */
  fallbackReason?: string
  /** Concurrency queue key. Written at dispatch. */
  queueKey: string

  // --- Nesting & Control Chain ---
  /** Delegation depth (1 = top-level). Written at dispatch. */
  nestingDepth: number
  /** Source delegation id when redirected. Written by control operations. */
  redirectedFrom?: string
  /** Source delegation id when restarted. Written by DelegationManager.controlDelegation(). */
  restartedFrom?: string
  /** Source delegation id when resumed. Written by DelegationManager.controlDelegation(). */
  resumedFrom?: string
  /** Source delegation id when chained. Written by DelegationManager.controlDelegation(). */
  chainedFrom?: string

  // --- Progress Tracking ---
  /** Last observed message count from child session. Written by SdkDelegationHandler stability poll. */
  lastMessageCount: number
  /** Consecutive stable polls. Written by SdkDelegationHandler. */
  stablePollCount: number
  /** Timestamp when last message count changed. Written by DelegationStateMachine. */
  lastMessageCountChangeAt?: number
  /** Grace period expiry for in-memory cleanup. Written by scheduleGracePeriodCleanup(). */
  gracePeriodExpiresAt?: number

  // --- Execution Signals ---
  /** Execution confirmation state. Written by DelegationCoordinator.recordExecutionSignal(). */
  executionState?: "pending" | "confirmed" | "unconfirmed" | "stalled"
  /** Timestamp of first observed action. Written by DelegationCoordinator. */
  firstActionAt?: number
  /** Source of execution signal. Written by DelegationCoordinator. */
  signalSource?: "action" | "event" | "message" | "polling" | "tool"
  /** Action count. Written by DelegationCoordinator. */
  actionCount?: number
  /** Message count. Written by DelegationCoordinator. */
  messageCount?: number
  /** Tool call count. Written by DelegationCoordinator. */
  toolCallCount?: number
  /** Evidence confidence level. Written by DelegationCoordinator. */
  evidenceLevel?: "accepted-only" | "status-only" | "message" | "tool" | "message-and-tool" | "file-change"
  /** Excerpt of the final assistant message. Written by DelegationCoordinator. */
  finalMessageExcerpt?: string

  // --- Results ---
  /** Result text (truncated for persistence). Written by transitionToTerminal() completion path. NOT persisted raw — redacted by persistDelegations(). */
  result?: string
  /** True when result was truncated. Written by projection code. */
  resultTruncated?: boolean
  /** Error message. Written by transitionToTerminal() error path. NOT persisted raw — redacted by persistDelegations(). */
  error?: string

  // --- Prompt & Versioning ---
  /** Original delegation prompt. Written at dispatch. Persisted but redacted by persistDelegations(). */
  prompt?: string
  /** v2 coordinator marker. Written by DelegationCoordinator.createRecord(). */
  v2?: boolean
}
```

### Persistence Redaction Policy

When writing `delegations.json`, `persistDelegations()` applies `redactBoundaryFields()` to redact the following fields from the persisted JSON:
- `result` — redacted
- `error` — redacted
- `fallbackReason` — redacted
- `prompt` — redacted (not in the redaction list explicitly, but the normalization layer may strip it)

These fields are available in-memory but are scrubbed on disk to prevent leaking sensitive content through the durable file.

### Read Normalization

When reading, `normalizePersistedDelegation()` applies defaults for missing or invalid fields:
- `executionMode`: defaults to `"sdk"` (unless `fallbackReason` is set → `"headless"`)
- `surface`: derived from `executionMode` if missing/invalid
- `recoveryGuarantee`: derived from `executionMode` if missing/invalid
- `status`: falls back to `"error"` with a synthetic error message if invalid
- `terminalKind`: set to `"error"` if status was invalid
- `workingDirectory`: defaults to `process.cwd()` if missing
- `nestingDepth`: defaults to `1`
- `lastMessageCount`: defaults to `0`
- `stablePollCount`: defaults to `0`

## 5. Lifecycle States

```
                         ┌──────────┐
                         │dispatched│◄── Record created, child session prompted
                         └────┬─────┘
                              │
                     session.idle event
                     (or explicit transition)
                              │
                              ▼
                         ┌──────────┐
                         │ running  │◄── Child session processing, monitoring active
                         └────┬─────┘
                              │
              ┌───────────────┼───────────────┐
              │               │               │
              ▼               ▼               ▼
        ┌──────────┐   ┌──────────┐   ┌──────────┐
        │completed │   │  error   │   │ timeout  │
        └──────────┘   └──────────┘   └──────────┘
```

### Transition Table

| From | To | Trigger | Actor | Notes |
|------|----|---------|-------|-------|
| `dispatched` | `running` | Session idle event or explicit transition | `DelegationManager.handleSessionIdle()` / `DelegationCoordinator.dispatch()` | PromptAsync returns 204, session becomes idle |
| `dispatched` | `completed` | Immediate completion (rare) | Completion detector | Possible for very short tasks |
| `dispatched` | `error` | Session deleted or dispatch failure | `handleSessionDeleted()` / `failDispatch()` | |
| `dispatched` | `timeout` | Safety ceiling exceeded | `handleSafetyCeiling()` | 30 minutes |
| `running` | `completed` | Dual-signal completion detected | Completion detector / `transitionToTerminal()` | Normal success path |
| `running` | `error` | Session error/deleted, SDK failure | `handleSessionError()` / `handleSessionDeleted()` | |
| `running` | `timeout` | Safety ceiling exceeded while running | `handleSafetyCeiling()` | 30 minutes from creation |
| `completed` | — | Terminal — no further transitions | N/A | Grace period (10 min) then in-memory cleanup |
| `error` | — | Terminal — no further transitions | N/A | Grace period then in-memory cleanup |
| `timeout` | — | Terminal — no further transitions | N/A | Grace period then in-memory cleanup |

### Grace Period Cleanup (R-LC-03)

After a delegation reaches terminal state, a 10-minute grace period timer starts (`TASK_CLEANUP_DELAY_MS`). When the timer fires:
- The delegation is removed from the **in-memory Map only**
- The delegation record **remains on disk** in `delegations.json`
- The record persists on disk until the parent's grace period timer or batch pruning removes it

### Batch Pruning

When the in-memory store exceeds `MAX_DELEGATIONS_BEFORE_PRUNE` (50 entries), `pruneCompletedDelegations()` removes terminal delegations older than `DEFAULT_PRUNE_MAX_AGE_MS` (30 minutes). Pruning:
1. Removes from in-memory Map
2. Calls `cleanupTracking()` (clears timers and session→delegation reverse lookup)
3. Calls `persistDelegations()` with the **remaining** in-memory delegations (effectively removing pruned records from the file)

## 6. Cross-File Dependencies

### Session-Tracker (`.hivemind/session-tracker/`)

- **Direction:** delegations.json → session-tracker (read-only, no write)
- **Mechanism:** The delegation-status tool reads session-tracker `hierarchy-manifest.json` to discover child sessions when delegations.json records are incomplete. It merges three sources: in-memory, persisted, and session-tracker.
- **Link field:** `childSessionId` (`ses_xxx` format) is the common key — it matches child session JSON files under `session-tracker/`.
- **ToolDelegation** creates child session records in the session-tracker but does **not** write to delegations.json. The session-tracker generates child files under `sessions/{root}/{child}.json` and updates `hierarchy-manifest.json`.

### Session Metadata / Continuity (`.hivemind/state/session-metadata.json`)

- **Direction:** delegations.json → continuity (indirect, via notification handler)
- **Mechanism:** When a delegation reaches terminal state, `notifyDelegationTerminal()` produces a `PendingNotification` that gets appended to the parent session's continuity record. The notification payload includes `delegationId` and `terminalState`.
- **Link field:** `id` (delegation id) is stored in `PendingNotification.metadata.delegationId`.

### Trajectory Ledger (`.hivemind/trajectory/`)

- **Direction:** session-tracker → trajectory (independent of delegations.json)
- **Mechanism:** `ToolDelegation.createDelegationTrajectoryAndContract()` creates trajectory records using `traj-{childSessionId}` as the key. It never reads delegations.json.
- **Link field:** `childSessionId` is shared between delegation records and trajectory IDs (`traj-{childSessionId}`).

## 7. Acceptance Criteria

### AC-1: Real SDK Session IDs
All new delegation records MUST use real OpenCode SDK session IDs (`ses_*` format) for `parentSessionId`, `childSessionId`, and the `Delegation` fields that store them. The `assertValidSessionID()` function in `src/shared/session-api.ts` enforces this at the SDK interaction layer.

### AC-2: Source Field
Every delegation record MUST track its dispatch `source` to distinguish:
- `delegate-task` tool → SDK child session via `DelegationManager.dispatch()` / `DelegationCoordinator.dispatch()`
- `task` tool → native OpenCode Task (captured by session-tracker)
- `execute-slash-command` → command delegation via `dispatchCommand()`
- `programmatic` → control actions (resume, chain) and recovery

Note: The current `Delegation` type does NOT have an explicit `source` field. This is a **gap** — the source must be inferred from `executionMode` and `v2` flag. Consider adding an explicit `source: "task" | "delegate-task" | "execute-slash-command" | "programmatic"` field.

### AC-3: Session-Tracker Linkage
Every delegation record's `childSessionId` MUST be discoverable in the session-tracker's `hierarchy-manifest.json` or child session file directory. The delegation-status tool's `getSessionTrackerDelegation()` and `getSessionTrackerChildren()` functions perform this link.

### AC-4: Typecheck and Test Compliance
After delegations.json redesign:
- `npm run typecheck` MUST pass
- `npm test` MUST pass (all existing tests)
- Existing test helpers that create delegation records MUST be updated to use the new schema
- The 35 test artifact records SHOULD be cleaned or migrated as part of this phase

### AC-5: Persistence Roundtrip
A delegation record MUST survive a roundtrip:
1. Created in-memory via `registerDelegation()`
2. Persisted to disk via `persistAll()`
3. Read back via `readPersistedDelegations()`
4. Normalized via `normalizePersistedDelegation()`
5. All fields preserved (with expected redaction of `result`, `error`, `fallbackReason`, `prompt`)

### AC-6: Recovery Correctness
After a harness restart:
- `recoverPending()` MUST correctly hydrate non-terminal (`dispatched`/`running`) delegations
- SDK-mode delegations MUST be handed to `sdkHandler.recoverSdkDelegation()`
- PTY-mode delegations MUST be handed to `commandHandler.recoverPtyDelegation()`
- Headless delegations MUST be transitioned to `error` with `terminalKind: "non-resumable-after-restart"`

### AC-7: No Test Artifact Pollution
After the redesign:
- No delegation with `childSessionId` not matching `ses_*` format shall be persisted
- The initial cleanup SHOULD remove or migrate the 35 test artifact records
- A migration script OR first-write cleanup in `persistDelegations()` SHOULD filter out records with invalid session IDs

## 8. Open Questions

1. **Should we add an explicit `source` field?**
   - Current: inferred from `executionMode` + `v2` flag + tool context
   - As-is is fragile — `executionMode="sdk"` could mean delegate-task or task tool
   - Recommendation: Add `source: "task" | "delegate-task" | "execute-slash-command" | "programmatic"` to the `Delegation` type

2. **Should we add a schema version marker?**
   - Current: no version field on individual records; inferred from `v2` boolean
   - Adding `schemaVersion: number` would future-proof migrations
   - Recommendation: Add `schemaVersion: 1` to the `Delegation` type for the redesign

3. **How should the 35 test artifact records be cleaned?**
   - Option A: Manual deletion as part of phase execution
   - Option B: First-write guard in `persistDelegations()` that filters records with invalid `parentSessionId` or `childSessionId`
   - Recommendation: Option B — add a one-shot migration that removes records where `parentSessionId` doesn't start with `ses`
