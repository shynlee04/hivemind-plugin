# Phase 2.1: Feature Architecture Migration (INSERTED) - Context

**Gathered:** 2026-03-18

<domain>
## Phase Boundary

Migrate the codebase to a hybrid feature structure to solve discoverability issues and isolate behavior. Establish `src/features/*` as the owner of business logic, while converting `tools/`, `hooks/`, `commands/`, and `plugin/` into thin OpenCode adapter surfaces.
</domain>

<decisions>
## Implementation Decisions

### Code Organization
- Move core logic from `src/cli/`, `src/control-plane/`, and `src/shared/` into `src/features/*`.
- Establish specific feature folders: `runtime-entry`, `session-entry`, `workflow`, `trajectory`, `handoff`, `doc-intelligence`, `runtime-observability`.

### Thin Adapters
- Modify existing `src/tools/`, `src/hooks/`, `src/commands/`, and `src/plugin/` to act exclusively as input/output validators that delegate to the new `features/` folders.

### Shared Contracts
- Enforce the `src/shared/contracts/` boundary so both the backend and future Bun TUI share the exact same types.
</decisions>
