# Phase 24.4: References & Templates System — Summary

**Status:** CANCELLED
**Date:** 2026-05-29

## Decision
This phase is CANCELLED. The "References & Templates System" as originally
conceived (template engine + reference resolver) was overengineering.

## Correct Architecture
The Command → Workflow → Agent routing paradigm (YAML frontmatter in command files,
process body in workflow files, agent dispatch via routing engine) handles everything.
Templates and references are STATIC documentation files, NOT runtime engines.

## What HM Already Has
- 118 commands with YAML frontmatter in `.opencode/commands/`
- 106 workflows in `.opencode/workflows/` (103 hm-*)
- 42 agents in `.opencode/agents/`
- `.planning/references/artifact-schema.md` (static reference, built by 24.2)

- `src/features/references/` (entire directory)
