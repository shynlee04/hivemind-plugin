# Phase 24.4: References & Templates System — SPEC

## Requirements

### REQ-01: Template Format
Define a template format supporting:
- Markdown body with `{{variable}}` placeholders
- YAML frontmatter with metadata (type, description, version, required_vars)
- Inline code blocks and structured sections

### REQ-02: Template Storage
Create `assets/templates/` with subdirectories:
- `assets/templates/agent-outputs/` — output format templates per agent type
- `assets/templates/reports/` — report templates (phase summary, verification, audit)
- Sync via `sync-assets.js` to `.opencode/templates/`

### REQ-03: Template Rendering
Build `src/features/templates/renderer.ts`:
- Load template by name
- Populate `{{variable}}` placeholders with provided values
- Return rendered markdown string
- Validate required variables are provided

### REQ-04: Reference Resolution
Build `src/features/references/resolver.ts`:
- Resolve `@ref:artifact-schema.md` style references
- Load referenced content from `.planning/references/`
- Support cross-agent output linking

### REQ-05: Integration
Wire into agent output contracts:
- Agent SUMMARY.md generation uses template system
- Phase CONTEXT.md generation uses template system
- Verification reports use template system

## Success Criteria
- [ ] REQ-01: Template format documented in `.planning/references/template-format.md`
- [ ] REQ-02: Template storage created with 5+ starter templates
- [ ] REQ-03: Renderer function working (unit tested)
- [ ] REQ-04: Reference resolver working (unit tested)
- [ ] REQ-05: At least 1 agent type uses template system
