# Phase 24.4: References & Templates System — Context (CORRECTED)

**Gathered:** 2026-05-29
**Status:** CANCELLED — architecture correction
**Mode:** Retroactive closure

## Correction

The GSD architecture proves that templates and references are STATIC markdown files,
NOT a runtime engine. The Command → Workflow → Agent routing handles everything:

- **Command profile** (YAML frontmatter in `.opencode/commands/*.md`): defines description,
  argument-hint, requires, tools, agent routing
- **Workflow file** (XML body or `.opencode/workflows/*.md`): process blueprint
- **Agent routing**: via command frontmatter, resolved by `execute-slash-command.ts`

Templates are just reference markdown files in `.opencode/get-shit-done/templates/` (GSD)
or static files. NO template engine, NO reference resolver needed.

What HM already has:
- 118 commands with YAML frontmatter → chains to workflows
- 106 workflows in `.opencode/workflows/` → process blueprints
- 42 agents in `.opencode/agents/` → execution targets
- `.planning/references/artifact-schema.md` → static reference

## Decision

Phase 24.4 is CANCELLED as originally conceived. No template engine, no resolver.
