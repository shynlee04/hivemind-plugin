# P40.03: Wire Auto-Loop / Ralph-Loop to Hook Chain

## Problem

Phase 39 designed `runAutoLoop` and `runRalphLoop` as orchestration primitives. They are imported in `plugin.ts` and passed to hook deps, but no hook consumer actually invokes them. The `session-hooks.ts` auto-loop is a simpler inline version that only handles delegation packets ‚Äî it doesn't use the spawner infrastructure at all.

- **Determine:** What events should trigger `runAutoLoop` vs `runRalphLoop`
- **Determine:** How do dispatcher/verifier/fixer callbacks get wired

- **Trigger:** When a delegation completes and the parent session needs to continue
- **File:** `src/hooks/lifecycle/session-hooks.ts`

- **If keeping both:** Document when each is used
