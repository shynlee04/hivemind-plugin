# P40.04: Wire Governance Evaluation

## Problem

The governance infrastructure exists at multiple levels:
- `createGovernanceSessionTool` in `features/governance-engine/`
- Rules/violations in the continuity store
- `configs.schema.json` for governance configuration

But the actual evaluation in `tool-guard-hooks.ts:190` always returns empty arrays. Governance rules configured by users are silently ignored.

## Scope

### Task 1: Research — Map governance engine current state
- **Read:** `src/features/governance-engine/index.ts` and all files in that directory
- **Read:** `src/hooks/guards/tool-guard-hooks.ts` governance section
- **Read:** Config schema for governance fields
- **Determine:** What was "stripped in 14-01" and what needs restoration

### Task 2: Wire governance-engine to tool execution hooks
- **File:** `src/hooks/guards/tool-guard-hooks.ts:190`
- **Change:** Replace empty arrays with actual evaluation call to governance engine
- **Wire:** Tool name, parameters, and context → governance rules → blocks/warnings/escalations

### Task 3: Wire governance rules loading from config
- **File:** `src/config/subscriber.ts`
- **Ensure:** Governance rules from `configs.json` are loaded and available to evaluation

### Task 4: Add tests for governance evaluation
- **Test:** Rule that blocks specific tool
- **Test:** Rule that warns on specific parameter
- **Test:** No rules → no blocks/warnings

## Acceptance Criteria

- [ ] Governance rules in config are evaluated on every tool execution
- [ ] Blocks prevent tool execution
- [ ] Warnings are surfaced to user
- [ ] Escalations trigger appropriate handlers
- [ ] No rules → no-op (backward compatible)
- [ ] All existing tests pass
- [ ] New tests cover: block, warn, escalate, no-rules paths
