## Evaluator Logic Summary (for planner reference)

The `evaluateGovernance(toolName, sessionID, rules)` function:
1. Returns empty result immediately if `rules` is empty/falsy (L33-35)
2. Iterates all rules, skipping `enabled: false` (L39-42)
3. Matches on `condition.toolNames` (array includes toolName) and/or `condition.sessionIDs` (array includes sessionID)
4. If only one condition specified: OR match. If both specified: AND match (L62-67)
5. On match: dispatches to `block` (throws later), `warn` (recorded), or `escalate` (recorded)
6. All violations persisted to continuity store (L95-103)
7. Result includes `blocked: boolean`, `blocks: string[]`, `warnings: string[]`, `escalations: Array<{ruleId, escalation}>`
