# P40.01: Fix Fire-and-Forget Dispatch & Silent Failures

## Problem

Three critical dispatch paths use `setTimeout` fire-and-forget patterns that:
1. Return `{ success: true }` BEFORE the actual SDK call completes
2. Catch errors into `console.error` only — user never sees them
3. Allow invalid agent names to pass validation on API failures

Additionally, `handleSessionError` in `manager.ts` only forwards to coordinator, not runtime adapter — delegations can stay "running" forever on child errors.

## Scope

### Task 1: Convert dispatch-command to await-based dispatch
- **File:** `src/tools/session/dispatch-command.ts:64-114`
- **Change:** Replace `setTimeout(async () => { ... }, 50)` with direct `await` of the SDK call
- **Return:** Actual success/failure based on SDK response
- **Error handling:** Return error message to user via tool response

### Task 2: Convert execute-slash-command agent dispatch to await-based
- **File:** `src/tools/session/execute-slash-command.ts:500-515`
- **Change:** Same pattern — await the `client.session.command()` call
- **Return:** Actual result from SDK

### Task 3: Convert execute-slash-command command dispatch to await-based
- **File:** `src/tools/session/execute-slash-command.ts:453`
- **Change:** Same pattern for the non-agent command path

### Task 4: Fix handleSessionError forwarding
- **File:** `src/coordination/delegation/manager.ts:106-108`
- **Change:** Add `this.runtime?.handleSessionError?.(sessionId, error)` alongside the coordinator call
- **Test:** Verify error events reach both paths

### Task 5: Fix validateAgentExists to fail on API error
- **File:** `src/tools/session/dispatch-command.ts:26-29`
- **Change:** Return `false` on API failure instead of `true`
- **Alternative:** Return `{ valid: false, reason: "api-unreachable" }` to distinguish from "agent not found"

## Acceptance Criteria

- [ ] All three dispatch paths return actual SDK results, not premature success
- [ ] Errors from SDK calls propagate to user via tool response
- [ ] `handleSessionError` forwards to both coordinator AND runtime adapter
- [ ] `validateAgentExists` returns false when API is unreachable
- [ ] All existing tests pass
- [ ] New tests cover: dispatch failure propagation, error forwarding, validation fallback
