## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] npm files array overrides .npmignore**
- **Found during:** Task 4 (gate verification)
- **Issue:** .npmignore entries for assets/.hivemind/ and *.map were ignored by npm because the files array explicitly listed 'assets' and 'dist'. npm's files whitelist takes precedence over .npmignore for included paths.
- **Fix:** Replaced blanket `"assets"` in files array with explicit subdirectory entries (assets/agents, assets/commands, assets/references, assets/skills, assets/templates, assets/workflows, assets/agent-instructions). Added `"!dist/**/*.map"` negation pattern in files array to exclude source maps. Kept .npmignore as defense-in-depth for non-files-listed paths.
- **Files modified:** package.json
- **Commit:** f8235d37

## Key Decisions

1. **D-01 (from plan):** Package renamed to hivemind-3.0 per user decision.
2. **D-02 (deviation):** Explicit subdirectory listing in files array instead of blanket "assets" — prevents any future hidden directory from leaking into the package.
3. **D-03 (deviation):** Source map exclusion via files array negation `!dist/**/*.map` rather than .npmignore — npm behavior requires it.
