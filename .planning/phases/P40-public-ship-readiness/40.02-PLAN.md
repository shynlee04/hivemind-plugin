# P40.02: Unify Delegation State Machines

## Problem

Two delegation state machines coexist:
- **v1 Runtime** (`manager-runtime.ts`): Used by `dispatchCommand()`, maintains `this.state.delegations`
- **v2 Coordinator** (`coordinator.ts`): Used by `delegate-task` tool, maintains `this.active`

The `manager.ts` facade tries to merge both, but:
- `getAllDelegations()` uses `??` fallback — only returns one system's data
- `handleSessionIdle` forwards to both, but `handleSessionError` only to coordinator
- `delegation-status` tool's `mergeAllDelegations` may produce inconsistent results

Additionally, the continuity store (singleton JSON file) and session-tracker (per-session files) track overlapping data with no sync.

## Scope

- **Approach:** When `manager-runtime.ts` creates a delegation, emit an event that coordinator can observe
- **File:** `src/coordination/delegation/manager-runtime.ts`
- **Goal:** `delegation-status list` shows ALL delegations regardless of creation path

### Task 2: Unify `getAllDelegations` merge logic
