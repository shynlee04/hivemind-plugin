# P40.06: npm Package Publishing & Public Repo Readiness

## Problem

### Name Collision
The name `hivemind` is already taken on npm by an unrelated package. The package CANNOT be published under this name.

### Dependency Issues
- `bun-pty` is in `dependencies` but should be `optionalDependencies` — it only works on Bun runtime
- `bun-types` is in `dependencies` — it's a type-only package, should be devDependencies
- Both will cause install failures on Node.js environments

### Missing Publishing Config
- No scoped package name (e.g., `@hivemind/opencode` or `@shynlee/hivemind`)
- No `publishConfig` section
- No `.npmignore` — `.gitignore` may not be sufficient
- `scripts/` not in `files` array but postinstall references it

## Scope

### Task 1: Resolve package naming
- **Options:**
  - A) `@hivemind/opencode` — scoped, clean
  - B) `@shynlee/hivemind` — scoped, owner namespace
  - C) `hivemind-opencode` — unscoped, descriptive
  - D) `opencode-hivemind` — unscoped, platform first
- **Decision:** Needs owner input — DO NOT implement without confirmation
- **Update:** package.json name, README, docs references

### Task 2: Fix dependency classification
- **Move** `bun-pty` from `dependencies` to `optionalDependencies`
- **Move** `bun-types` from `dependencies` to `devDependencies`
- **Verify:** `npm install` works on Node.js without Bun
- **Verify:** PTY features gracefully degrade when bun-pty unavailable

### Task 3: Add publish configuration
- **Add** `publishConfig` section with access/registry
- **Add** `.npmignore` to exclude dev files
- **Verify** `files` array includes everything needed:
  - `dist/` (compiled output)
  - `bin/` (CLI)
  - `assets/` (primitives source)
  - `scripts/` (postinstall script)
  - `.hivemind/configs.schema.json` (config schema)

### Task 4: Verify sync-oss.yml produces publishable package
- **Read:** `.github/workflows/sync-oss.yml` whitelist
- **Verify:** All files in `files` array are in PUBLIC_PATHS
- **Add CI step:** `npm pack --dry-run` to verify package contents
- **Add CI step:** `npm publish --dry-run` to verify publish config

### Task 5: Create first npm publish workflow
- **File:** `.github/workflows/publish.yml`
- **Trigger:** Tag push (e.g., `v0.1.0`)
- **Steps:** build → test → pack → publish
- **Dry-run option:** For testing without actual publish
