<task type="auto" tdd="false">
  <name>Task 2: Create .npmignore to fix 90 MB package bloat</name>
  <files>.npmignore</files>
  <action>
Create .npmignore at project root to exclude internal governance and development files from the npm package.

The CRITICAL issue: `files: ["assets"]` in package.json captures the hidden directory `assets/.hivemind/` (90 MB, 410 governance skill markdown files — gate-l3-*, hf-l2-*, hm-l2-*). npm's files field is a whitelist, and .npmignore is a blacklist that removes from that whitelist. This is the correct way to exclude without restructuring the assets/ directory.

Write .npmignore with these contents:

```
# Internal governance and meta-builder — 90 MB, not for consumers
assets/.hivemind/
assets/.hivefiver-meta-builder/

# Source maps — ~3 MB, consumers don't need them for 0.x
*.map

# Development-only directories
.planning/
.hivemind/
.hivefiver-meta-builder/
tests/
docs/

# CI/CD
.github/

# Config files not needed at runtime
vitest.config.ts
tsconfig.json
*.tsbuildinfo
```

Key decisions (per research F5):
- assets/.hivemind/ excluded: These are internal governance skills (quality gates, meta-builder authoring) — consumers never need them. They are the single largest source of bloat at 90 MB.
- *.map excluded: Source maps add ~3 MB. Consumers don't debug our internals. Can revisit for 1.0.
- tests/ and docs/ excluded: Standard for npm packages — development artifacts.
- dist/, bin/, assets/ (minus exclusions), scripts/, .hivemind/configs.schema.json still included via the files array in package.json.

The files array in package.json is NOT modified — .npmignore acts as a post-filter on what files includes. This means assets/skills/, assets/agents/, assets/commands/, assets/workflows/, assets/references/, assets/templates/ are all still shipped.
  </action>
  <verify>
    <automated>npm pack --dry-run 2>&1 | grep -c "assets/.hivemind/" || echo "PASS: 0 matches for assets/.hivemind/"</automated>
  </verify>
  <done>.npmignore exists and npm pack --dry-run output contains zero references to assets/.hivemind/</done>
</task>
