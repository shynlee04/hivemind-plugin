# Phase P40 Plan 40.01: Fix Fire-and-Forget Dispatch Summary

Deferred-promise pattern for dispatch-command.ts and execute-slash-command.ts — replaces premature `{ success: true }` returns with real Promise-wrapped setTimeout that resolves with actual SDK outcomes.

## Changes Made

### Task 1: dispatch-command.ts — deferred-promise + validateAgentExists fix

**`src/tools/session/dispatch-command.ts`:**
- Replaced fire-and-forget `setTimeout(async () => { ... }, 50)` + premature `return { success: true }` with `return new Promise((resolve) => { setTimeout(async () => { try { resolve({ success: true }) } catch { resolve({ success: false, ... }) } }, 50) })`
- Agent restore failure now resolves with `{ success: false, output: "Dispatch succeeded but agent restore failed: ..." }` instead of swallowed `console.error`
- `validateAgentExists` catch block returns `false` (was `true`) — prevents dispatch with unvalidated agent when API is unreachable
- Updated JSDoc to reflect deferred-promise behavior

**`tests/tools/dispatch-command.test.ts`** (rewritten — 10 tests):
- `validates agent format correctly` — format check
- `validates agent existence correctly` — positive lookup
- `validateAgentExists returns false when API call throws` — NEW: covers catch block
- `dispatches synthetic prompt with agent override` — integration check
- `resolves with { success: true } after successful SDK call via deferred promise` — NEW: fake timers
- `resolves with { success: false, error: true } after SDK error via deferred promise` — NEW: fake timers
- `resolves with failure when agent restore fails` — NEW: partial SDK failure
- `throws InvalidCommandError for invalid agent format` — validation
- `throws AgentNotFoundError for non-existent agent` — validation
- `requires vi.advanceTimersByTime to resolve — confirms 50ms deadlock prevention` — NEW: regression test
