# P40.07: End-to-End Integration Verification

## Problem

Phase 39's E2E verification (39-10) tested internal checkpoints (typecheck, build, test suite, coverage). It did NOT verify the actual user experience:

1. User clones/installs hivemind in their project
2. Plugin registers with OpenCode
3. Tools appear and function
4. Hooks fire and modify behavior
5. Bootstrap copies primitives to user's `.opencode/`
6. Delegation, session tracking, governance actually work in real OpenCode session

## Scope

### Task 1: Create simulated user project test
- **Create:** `tests/integration/user-install.test.ts`
- **Simulate:** What happens when user runs `npm install hivemind` in a fresh project
- **Verify:**
  - postinstall script runs (sync-assets.js)
  - `.opencode/` primitives are copied (agents, skills, commands)
  - Plugin entry point loads without errors
  - All tools are registered

### Task 2: Create plugin registration E2E
- **Create:** `tests/integration/plugin-registration.test.ts`
- **Load:** `dist/plugin.js` as OpenCode would
- **Verify:**
  - Plugin exports correct shape (`Plugin` type)
  - All declared tools have implementations
  - All declared hooks are functions
  - No import errors or missing modules

### Task 3: Create tool execution smoke tests
- **Create:** `tests/integration/tool-smoke.test.ts`
- **For each registered tool:**
  - Call with minimal valid input
  - Verify it returns a response (not throws)
  - Verify response has expected shape
- **Tools to smoke test:**
  - `delegate-task`, `delegation-status`
  - `execute-slash-command`, `session-patch`
  - `bootstrap-init`, `bootstrap-recover`
  - `configure-primitive`, `validate-restart`
  - `hivemind-doc`, `hivemind-trajectory`, `hivemind-pressure`
  - `session-tracker`, `session-hierarchy`, `session-context`
  - `hivemind-session-view`
  - `hivemind-sdk-supervisor`, `hivemind-command-engine`
  - `hivemind-agent-work` (create + export)
  - `run-background-command`
  - `session-journal-export`
  - `prompt-skim`, `prompt-analyze`

### Task 4: Create bootstrap flow E2E
- **Create:** `tests/integration/bootstrap-flow.test.ts`
- **Simulate:** User installs hivemind in a temp directory
- **Verify:**
  - `assets/` contents are known and correct
  - `sync-assets.js` copies to `.opencode/` correctly
  - All 31 hm-* agents are present
  - All 11 hf-* agents are present
  - All 34 non-GSD skills are present
  - All 19 commands are present
  - No GSD primitives leak into user's `.opencode/`

### Task 5: Verify sync-oss produces clean public repo
- **Run:** sync-oss.yml logic locally (dry run)
- **Verify:** No banned directories (.opencode, .hivemind, .planning, .hivefiver-meta-builder)
- **Verify:** All required source files present
- **Verify:** Build + test passes from oss-dev branch content

## Acceptance Criteria

- [ ] User install simulation passes — all primitives copied
- [ ] Plugin loads without errors — correct export shape
- [ ] Every registered tool responds to minimal input (smoke test)
- [ ] Bootstrap flow copies all 75 agents, 34 skills, 19 commands
- [ ] No GSD primitives in user-facing output
- [ ] sync-oss dry run produces clean output
- [ ] All existing tests still pass

## Verification

1. `npm run typecheck` passes
2. `npm test` passes (all existing + new integration tests)
3. `npm run build` produces valid dist/
4. `npm pack --dry-run` shows correct contents
5. Manual: install in a test project → verify plugin loads in OpenCode

## Out of Scope

- Performance testing
- Load testing
- Multi-user scenarios
- OpenCode version compatibility matrix
