    ### Step 1: Preflight — Command existence check (REQ-03, D-06)
    ```typescript
    // Call client.command.list() - try with directory from ctx if available
    const commands = await client.command.list({ query: { directory: ctx.directory } })
    // commands is Array<Command> — SDK v1 Command type
    const found = commands.find((c: { name: string }) => c.name === args.command)
    if (!found) {
      return error(
        `Command not found: /${args.command}. Available: ${commands.map((c: { name: string }) => c.name).join(", ")}`,
        { availableCommands: commands.map((c: { name: string }) => c.name) },
        { command: args.command }
      )
    }
    ```

    ### Step 2: Frontmatter enrichment (REQ-04, D-07)
    Extract frontmatter fields from the found command:
    ```typescript
    const frontmatterAgent = (found as any).agent  // from SDK Command type
    const frontmatterModel = (found as any).model
    const frontmatterSubtask = (found as any).subtask
    ```
    The SDK `Command` type from `command.list()` includes `agent`, `model`, `subtask`, `template`.

    Agent precedence (SPEC says "documented precedence"): Explicit `args.agent` > frontmatter `agent`.

    ### Step 3: Subtask routing (REQ-04 subtask, D-04)
    If frontmatterSubtask === true:
    ```typescript
    // Use session.prompt() with SubtaskPartInput — NOT session.command()
    const subtaskAgent = args.agent || frontmatterAgent
    const promptText = `${found.template}${args.arguments ? ` ${args.arguments}` : ""}`
    await client.session.prompt({
      path: { id: sessionID },
      body: {
        agent: subtaskAgent,
        parts: [{
          type: "subtask" as const,
          agent: subtaskAgent,
          prompt: promptText,
          description: found.description,
        }],
      },
      query: { directory: ctx.directory },
    })
    return success(
      `Subtask dispatched: /${args.command} → ${subtaskAgent}`,
      { command: args.command, agent: subtaskAgent, dispatched: true },
      { command: args.command, mode: "subtask" }
    )
    ```
    **Key constraint per D-04:** `session.command()` does NOT accept `SubtaskPartInput`. Only `session.prompt()` with `parts: [SubtaskPartInput]` works for subtask routing.

    Per SPEC boundary: subtask routing wires to SDK-native `session.prompt()` with SubtaskPartInput. It does NOT create additional delegation through hivemind's own delegate-task tool — the SDK server handles the session lifecycle.

    ### Step 4: Primary path — SDK session.command() (REQ-01, D-03)
    For non-subtask commands:
    ```typescript
    // Determine effective agent: explicit > frontmatter (REQ-04 precedence)
    const effectiveAgent = args.agent || frontmatterAgent
    // Determine effective model: explicit > frontmatter (same precedence pattern)
    const effectiveModel = args.model || frontmatterModel

    // Mechanism (b) — run-as-then-restore (REQ-02, D-09)
    if (args.restore && effectiveAgent && currentAgent) {
      // Dispatch under target agent
      await client.session.command({
        path: { id: sessionID },
        body: {
          command: args.command,
          arguments: args.arguments || "",
          agent: effectiveAgent,
          model: effectiveModel,
        },
        query: { directory: ctx.directory },
      })
      // Restore prior agent
      await client.session.command({
        path: { id: sessionID },
        body: { command: "", arguments: "", agent: currentAgent },
        query: { directory: ctx.directory },
      })
      return success(
        `Command /${args.command} executed under ${effectiveAgent} and restored to ${currentAgent}.`,
        { command: args.command, agent: effectiveAgent, restored: true, priorAgent: currentAgent },
        { command: args.command, mode: "agent-switch-restore" }
      )
    }

    // Mechanism (a) — simple agent switch via SDK (REQ-02, D-08)
    const response = await client.session.command({
      path: { id: sessionID },
      body: {
        command: args.command,
        arguments: args.arguments || "",
        ...(effectiveAgent && { agent: effectiveAgent }),
        ...(effectiveModel && { model: effectiveModel }),
      },
      query: { directory: ctx.directory },
    })
    ```

    ### Step 5: TUI fallback (REQ-01, D-03 — hybrid approach)
    If session.command() throws (session busy, SDK error, etc.), catch and fall back to TUI path:
    ```typescript
    // Fallback: use TUI append path (preserved from current implementation)
    const parts: string[] = []
    if (effectiveAgent) parts.push(`@${effectiveAgent}`)
    parts.push(`/${args.command}`)
    if (args.arguments) parts.push(args.arguments)
    const promptText = parts.join(" ")

    await client.tui.clearPrompt()
    await client.tui.appendPrompt({ body: { text: promptText } })
    await client.tui.submitPrompt()
    ```

    ### Step 6: Standard result envelope (REQ-07, D-05)
    All returns use `success()` / `error()` from `../../shared/tool-response.js`.

    Success: `return success(message, data, metadata)`
    Error: `return error(message, data, metadata)`

    The function signature returns `ToolResponse`, but since `tool()` accepts any return, this is fine. The return type of the tool's execute function is `Promise<ToolResponse>`.

    ### Step 7: Error handling
    Wrap the entire execute body in try/catch. For SDK errors, fall back to TUI path. For TUI errors, return error via `error()`.

    ### Contamination/Risk Mitigation (all 4 items)
    1. **SDK queue behavior (Pitfall 2):** If session.command() queues, the call still returns. The tool returns success immediately but the command fires on the next turn. This is acceptable per RESEARCH.md. TUI fallback provides an alternative.
    2. **Current-agent introspection missing (D-10):** Cache `currentAgent = ctx.agent` at the top of execute(). The ToolContext provides the current agent at call time.
    3. **CommandBundle type gap:** Solved by Plan 01 — extended CommandBundle. But the tool primarily uses SDK `Command` type from `command.list()`, which already has `model` and `subtask`. Use the SDK type directly.
    4. **v1/v2 SDK mismatch (Pitfall 1):** Use v1 body shape only: `{ command, arguments, agent?, model? }`. No `parts`, no `variant`. The `client` injected at `plugin.ts:408` is `PluginInput["client"]` which is v1.
