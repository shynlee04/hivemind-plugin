## Known Runtime Issue

`session.command()` called from within tool execution may **queue silently** when session is busy — same behavior as `session.prompt()`. The TUI fallback only catches SDK throws, not queuing. Mitigation: TUI append path preserves backward compatibility when SDK path blocks. Long-term fix: implement timeout wrapper or use TUI append as primary path.
