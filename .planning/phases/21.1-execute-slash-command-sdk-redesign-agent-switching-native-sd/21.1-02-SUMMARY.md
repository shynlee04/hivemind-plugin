# Plan 21.1-02 Summary: Tool Rewrite — Deferred Command Dispatch

**Status:** Revised after live UAT failure
**Evidence level:** L2 for unit/build; L1 live UAT pending restart after deferred fix
**Date:** 2026-05-22

## Design Outcome

The original SDK-await implementation was rejected by live behavior. Calling and awaiting `client.session.prompt()` from inside `execute-slash-command` causes the tool execution to hang while the session is busy running the tool.

The revised implementation uses two paths:

| Path | Behavior |
|------|----------|
| Non-subtask command | Preserve TUI `clearPrompt -> appendPrompt -> submitPrompt` path |
| Frontmatter or one-shot subtask | Expand command body and schedule SDK `session.prompt()` after the tool returns |

## Changes

| File | Change |
|------|--------|
| `src/tools/session/execute-slash-command.ts` | Adds command discovery, `$ARGUMENTS` expansion, one-shot `agent`/`subtask` overrides, and deferred `SubtaskPartInput` scheduling |
| `src/routing/command-engine/index.ts` | Discovers commands from project/global roots and singular/plural command directories |
| `src/routing/command-engine/types.ts` | Keeps `model` and `subtask` in `CommandBundle` |
