    ## Important Testing Constraints

    1. **v1 SDK body shape:** Assert that `session.command` is called with body `{ command, arguments, agent?, model? }` — no `parts`, no `variant` (per Pitfall 1).

    2. **ToolResponse assertions:** Every test must assert against the `ToolResponse` shape (kind/message/data/metadata), NOT the old `{ output, metadata }` shape.

    3. **Mock isolation:** Each test creates its own mock client via `createMockClient()` or `vi.resetAllMocks()` to prevent state leakage.

    4. **The mock client's `command.list` returns a sample `Command` array** matching the SDK v1 `Command` type: `{ name, description?, agent?, model?, template, subtask? }`.
