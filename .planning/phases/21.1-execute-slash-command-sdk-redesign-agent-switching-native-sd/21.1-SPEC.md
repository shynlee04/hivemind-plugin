# Phase 21.1: Execute-Slash-Command SDK Redesign — Specification

**Created:** 2026-05-22
**Ambiguity score:** 0.12 (gate: ≤ 0.20)
**Requirements:** 7 locked

## Goal

`execute-slash-command` tool changes from a raw TUI prompt appender (`clearPrompt → appendPrompt → submitPrompt`) to an SDK-native command executor supporting real TUI agent switching, command frontmatter-aware preflight, primitive discovery across project/global dirs, `$ARGUMENTS`/flags/template/reference parsing, and subtask-to-native-task delegation wiring.

## Background

Current implementation at `src/tools/session/execute-slash-command.ts:49-152`:

- Builds raw text `[@agent] /command [arguments]` then calls `clearPrompt() → appendPrompt() → submitPrompt()`.
- `agent` parameter only prepends `@agent` text — no real TUI agent switch, no restore.
- `model` parameter accepted in schema but never actually injected into the prompt execution.
- No command existence preflight — dispatches blindly.
- No command frontmatter awareness (`agent`, `model`, `subtask` are registered in schema but the read-side command-engine drops them at discovery).
- `$ARGUMENTS` detection is shallow (`body.includes("$ARGUMENTS")`), no `$1/$2` positional support, no bash interpolation, no file reference resolution.
- Result envelope is raw `{ output, metadata }` — not standard `renderToolResult(...)`.
- Subtask (`subtask: true`) is not wired to native `task` tool delegation — no `SubtaskPart` creation.

OpenCode SDK v1.14.44 (source: `anomalyco/opencode`) provides native alternatives:

- `client.session.command(...)` — supports `agent`, `model`, `arguments`, `command`, `variant`, `parts`.
- `client.command.list(...)` — command discovery API.
- SDK native command parser handles `$1`/`$2`, `$ARGUMENTS`, bash interpolation (!\`cmd\`), file/agent references.
- SDK native subtask logic: `isSubtask = (agent.mode === "subagent" && cmd.subtask !== false) || cmd.subtask === true`.
- Both singular/plural dirs supported: `{command,commands}`, `{agent,agents}`, project and global.

Research evidence from 3 passes: codebase scan (ses_1b4869be7ffeuKzbfhxku4hK10), OpenCode SDK research (ses_1b4869b74ffe5G395s5zjraSIO), advisor pass (ses_1b47d0a9fffeWAlFFOVt5b2qRc).

## Requirements

1. **SDK-native command execution path**: Change primary execution from TUI `appendPrompt` to `client.session.command(...)` with agent/model/arguments/parts support.
   - Current: `clearPrompt → appendPrompt → submitPrompt` (raw text, no SDK command semantics)
   - Target: `execute-slash-command` calls `client.session.command({ command, agent, model, arguments })` when available, with TUI append fallback for backward compatibility
   - Acceptance: Tool dispatches command through SDK command path; raw prompt text is no longer the primary execution mechanism

2. **Real TUI agent switching**: Two mutually exclusive mechanisms — (a) switch front-facing TUI agent before appending command, (b) run command under target agent then restore prior agent after completion.
   - Current: Only `@agent` text prepend — no TUI agent change.
   - Target: Mechanism (a) calls `session.command({ agent: target })` which triggers SDK-side agent switch and `AgentSwitched` event; mechanism (b) records current agent, runs under target agent, restores prior agent on completion callback.
   - Acceptance: After invocation with `agent`, the TUI shows the target agent (not `@agent` text); when using mechanism (b), the TUI restores to the user's prior agent after command finishes.

3. **Command existence preflight**: Verify command exists in OpenCode's command registry before execution attempt.
   - Current: No preflight — dispatches and hopes SDK validates.
   - Target: Call `client.command.list(...)` or read from `discoverCommandBundles()` before dispatch; return graceful error for missing/unknown commands.
   - Acceptance: `execute-slash-command("/nonexistent")` returns graceful error, does not dispatch; `execute-slash-command("/gsd-help")` proceeds to execution.

4. **Command frontmatter awareness**: Read and respect `agent`, `model`, `subtask`, `template`, `description` from command frontmatter files.
   - Current: Command-engine reads `description`, `agent`, `body`, `filePath` only — drops `model` and `subtask`.
   - Target: Full frontmatter schema is available to the tool; `agent` from frontmatter can override tool-provided agent (or vice versa, with documented precedence); `subtask: true` routes to native subtask path; `model` feeds into SDK command call.
   - Acceptance: A command file with `agent: hm-operator` and `subtask: true` in frontmatter causes tool to dispatch via subtask delegation instead of foreground append.

5. **Full argument/template/reference parsing**: Support `$1`/`$2`/`$ARGUMENTS`, bash interpolation, file/agent references, and flag-based parsing.
   - Current: Only shallow `$ARGUMENTS` detection via `body.includes("$ARGUMENTS")`.
   - Target: Use OpenCode SDK's native argument parsing (regex: `argsRegex`, `placeholderRegex`, `quoteTrimRegex`, `bashRegex`) or reimplement equivalent behavior in the tool layer; `$1`/`$2` positional substitution; last numbered placeholder absorbs remaining args; `$ARGUMENTS` replaced with raw input; file references resolved; agent references become agent parts.
   - Acceptance: Command `gsd-plan-phase {phase}` with arguments `21.1 --skip research` correctly substitutes `$1`=21.1 and appends `--skip research` as remaining arguments; bash interpolation `!`which node`` resolves to path.

6. **Primitive discovery across all locations**: Discover and list available commands and agents from project + global + singular/plural directories.
   - Current: Primitive-loader scans `{agents,agent}`, `{commands,command}`, `{skills,skill}`, but command-engine only exposes command discovery, not agent choices.
   - Target: Tool exposes agent listing capability (all, primary, subagent modes) from both project `.opencode/{agents,agent}/` and global config `{agents,agent}/`; command listing respects singular/plural.
   - Acceptance: Passing invalid agent name returns a list of available agents; tool can list all available commands with their frontmatter metadata.

7. **Standard result envelope**: Return results through `renderToolResult(...)` standard wrapper instead of raw `{ output, metadata }`.
   - Current: Returns `{ output, metadata }` — not the project's standard tool response format.
   - Target: Standard `ToolResponse` envelope with status, output content, and tool metadata.
   - Acceptance: All existing tests updated to assert against standard envelope.

## Boundaries

**In scope:**
- Rewrite `src/tools/session/execute-slash-command.ts` to use SDK command execution path
- Two TUI agent switching mechanisms (switch-before-append, run-as-then-restore)
- Command existence preflight via command discovery
- Full command frontmatter awareness (agent, model, subtask, template)
- Native argument/template/reference parsing
- Primitive discovery exposure (list agents, list commands)
- Standard result envelope
- Corresponding test updates and additions

**Out of scope:**
- Changing how `hivemind-command-engine` read-side works — preflight may use it as-is or add new discovery surfaces, but its core contract stays unchanged — this phase is about the write-side tool
- Updating `gsd-*` commands to use the new tool — those are GSD framework internals and not shipped primitives
- Backend config schema changes — the existing `command-frontmatter.schema.ts` and `agent-frontmatter.schema.ts` are sufficient for frontmatter reading; no schema-kernel changes planned
- Plugin.ts composition changes — tool registration path stays the same
- Other tools (delegate-task, delegation-status, etc.) — this phase is focused on `execute-slash-command` only
- Improving GSD's own command parsing — GSD has its own `$ARGUMENTS` handling which is separate from OpenCode SDK semantics; this phase standardizes on OpenCode's.

## Constraints

- Must support both OpenCode SDK `v1.14.44` and `v1.15.7` range — actual pin in `package.json`/lockfile must be validated before locking target SDK surface.
- Must preserve backward compatibility for callers that pass only `command` + `arguments` (no `agent`/`model`).
- Tool response must match project's standard `ToolResponse` format from `src/shared/tool-response.ts`.
- TypeScript strict mode — no `any` on new types.
- Module size limit: 500 LOC per file.
