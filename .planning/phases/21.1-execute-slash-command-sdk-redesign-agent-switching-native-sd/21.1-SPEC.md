# Phase 21.1: Execute-Slash-Command Command Dispatch — Specification

**Created:** 2026-05-22
**Ambiguity score:** 0.12 (gate: ≤ 0.20)
**Requirements:** 7 locked

## Goal

`execute-slash-command` must dispatch OpenCode command primitives with the same effective payload shape observed from manual TUI command execution. Commands with `subtask: true`, or a one-shot `subtask: true` override, must expand the command body and schedule a `SubtaskPartInput` using dynamic `prompt`, `description`, `agent`, and `command` metadata. The command primitive on disk must remain unchanged.

## Design Correction — 2026-05-22

Live UAT disproved the earlier SDK-await design. Awaiting `client.session.prompt()` from inside tool execution hangs because the active session is still running the tool. Evidence: `session-ses_1b48.md:12903-12918` shows `execute-slash-command` aborting; `session-ses_1b48.md:12932-12947` shows the subtask payload only appears after cancellation.

The corrected design is:

- Non-subtask commands use the original TUI append/submit path.
- Subtask commands do not await SDK prompt calls inside the tool.
- Subtask commands schedule `client.session.prompt()` after the tool returns.
- One-shot overrides may supply `agent` and `subtask` without mutating the command file.
- Command discovery includes project and global config roots, and both singular/plural command directories.

## Background

Current implementation at `src/tools/session/execute-slash-command.ts:49-152`:

- Builds raw text `[@agent] /command [arguments]` then calls `clearPrompt() → appendPrompt() → submitPrompt()`.
- `agent` parameter only prepends `@agent` text — no real TUI agent switch, no restore.
- `model` parameter accepted in schema but never actually injected into the prompt execution.
- No command existence preflight — dispatches blindly.
- No command frontmatter awareness (`agent`, `model`, `subtask` are registered in schema but the read-side command-engine drops them at discovery).
- `$ARGUMENTS` detection is shallow (`body.includes("$ARGUMENTS")`), no `$1/$2` positional support, no bash interpolation, no file reference resolution.
- Result envelope is raw `{ output, metadata }` — not standard `renderToolResult(...)`.
- Subtask (`subtask: true`) is not wired to native `task` tool delegation — no `SubtaskPart` creation.

OpenCode SDK v1.14.44 (source: `anomalyco/opencode`) provides native alternatives:

- `client.session.command(...)` — supports `agent`, `model`, `arguments`, `command`, `variant`, `parts`.
- `client.command.list(...)` — command discovery API.
- SDK native command parser handles `$1`/`$2`, `$ARGUMENTS`, bash interpolation (!\`cmd\`), file/agent references.
- SDK native subtask logic: `isSubtask = (agent.mode === "subagent" && cmd.subtask !== false) || cmd.subtask === true`.
- Both singular/plural dirs supported: `{command,commands}`, `{agent,agents}`, project and global.

Research evidence from 3 passes: codebase scan (ses_1b4869be7ffeuKzbfhxku4hK10), OpenCode SDK research (ses_1b4869b74ffe5G395s5zjraSIO), advisor pass (ses_1b47d0a9fffeWAlFFOVt5b2qRc).

## Requirements

1. **Foreground command path preserved**: Non-subtask commands keep the TUI `clearPrompt → appendPrompt → submitPrompt` behavior.
   - Acceptance: Calling a command without `subtask: true` appends `/command args` into the TUI pipeline.

2. **Deferred subtask dispatch**: Subtask command execution must not await `session.prompt()` inside the tool.
   - Acceptance: If the SDK prompt request never resolves, `execute-slash-command` still returns a scheduled success result instead of hanging.

3. **Subtask payload parity with manual TUI**: The scheduled SDK prompt must use the same effective fields observed from manual command execution.
   - Acceptance: Payload includes expanded `prompt`, frontmatter or override `description`, resolved `agent`, and `command` metadata.

4. **Command frontmatter plus one-shot overrides**: Read `agent`, `model`, `subtask`, and `description`; allow tool arguments to override `agent` and `subtask` for one execution only.
   - Acceptance: A command file without `agent` or `subtask` can be run as a subtask by passing `agent` and `subtask: true` to the tool.

5. **Argument substitution**: Replace `$ARGUMENTS` in the command body with the raw argument string for subtask dispatch.
   - Acceptance: `test-echo` body `Echo back exactly what the user said: $ARGUMENTS` becomes `Echo back exactly what the user said: <arguments>`.

6. **Primitive discovery across all command locations**: Discover commands from project and global config roots, supporting both `.opencode/commands`, `.opencode/command`, global `commands`, and global `command`.
   - Acceptance: A command found only in the global singular `command/` directory can be discovered and dispatched.

7. **Honest scheduled result metadata**: Subtask dispatch returns scheduled metadata, not a completed-runtime claim.
   - Acceptance: Subtask tool output includes `mode: "subtask"`, `scheduled: true`, and `dispatched: true`.

## Boundaries

**In scope:**
- Rewrite `src/tools/session/execute-slash-command.ts` to use SDK command execution path
- Two TUI agent switching mechanisms (switch-before-append, run-as-then-restore)
- Command existence preflight via command discovery
- Full command frontmatter awareness (agent, model, subtask, template)
- Native argument/template/reference parsing
- Primitive discovery exposure (list agents, list commands)
- Standard result envelope
- Corresponding test updates and additions

**Out of scope:**
- Changing how `hivemind-command-engine` read-side works — preflight may use it as-is or add new discovery surfaces, but its core contract stays unchanged — this phase is about the write-side tool
- Updating `gsd-*` commands to use the new tool — those are GSD framework internals and not shipped primitives
- Backend config schema changes — the existing `command-frontmatter.schema.ts` and `agent-frontmatter.schema.ts` are sufficient for frontmatter reading; no schema-kernel changes planned
- Plugin.ts composition changes — tool registration path stays the same
- Other tools (delegate-task, delegation-status, etc.) — this phase is focused on `execute-slash-command` only
- Improving GSD's own command parsing — GSD has its own `$ARGUMENTS` handling which is separate from OpenCode SDK semantics; this phase standardizes on OpenCode's.

## Constraints

- Must support both OpenCode SDK `v1.14.44` and `v1.15.7` range — actual pin in `package.json`/lockfile must be validated before locking target SDK surface.
- Must preserve backward compatibility for callers that pass only `command` + `arguments` (no `agent`/`model`).
- Tool response must match project's standard `ToolResponse` format from `src/shared/tool-response.ts`.
- TypeScript strict mode — no `any` on new types.
- Module size limit: 500 LOC per file.

## Acceptance Criteria

- [ ] `execute-slash-command("/gsd-help")` dispatches via SDK command path, not raw TUI append
- [ ] `execute-slash-command("/gsd-help", { agent: "hm-operator" })` causes TUI agent to switch to hm-operator (mechanism a)
- [ ] `execute-slash-command("/gsd-help", { agent: "hm-operator", restore: true })` runs under hm-operator then restores prior agent (mechanism b)
- [ ] `execute-slash-command("/nonexistent")` returns graceful error without dispatching
- [ ] Command file with `agent: hm-operator` frontmatter routes to that agent when no agent is explicitly passed
- [ ] Command file with `subtask: true` frontmatter routes via subtask delegation
- [ ] `$1`/`$2`/`$ARGUMENTS` placeholders are correctly substituted in command templates
- [ ] Tool can list available agents when invalid agent name is provided
- [ ] Tool returns standard `ToolResponse` envelope
- [ ] All 91+ existing delegation/tool regression tests still pass; new tests cover agent switch, preflight, frontmatter, argument parsing
- [ ] Typecheck passes with strict mode
