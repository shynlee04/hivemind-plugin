# Phase 21.1: Execute-Slash-Command SDK Redesign — Context

**Gathered:** 2026-05-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Upgrade `execute-slash-command` tool from TUI prompt appender to SDK-native command executor with two TUI agent switching mechanisms, command frontmatter preflight, primitive discovery, argument parsing via SDK server, and subtask->native task delegation wiring. Does NOT change `hivemind-command-engine` read-side core contract, GSD commands, plugin.ts composition, schema-kernel, or other tools.

</domain>

<decisions>
## Implementation Decisions

### SDK Version
- **D-01:** Project uses SDK **v1** (`@opencode-ai/sdk` 1.15.5), NOT v2. `PluginInput.client` is `ReturnType<typeof createOpencodeClient>` from v1. v1 `session.command()` body = `{ command, arguments, agent?, model?, messageID? }` — no `parts`, no `variant`.
- **D-02:** `client` is injected via factory closure (`createExecuteSlashCommandTool(client)` at `plugin.ts`), not available from `ctx`. No change needed to this pattern.

### Command Execution
- **D-03:** Primary path = `client.session.command({ path: { id: sessionID }, body: { command, arguments, agent?, model? }, query: { directory } })`. Falls back to TUI append path if SDK command fails/queues.
- **D-04:** Subtask routing (`subtask: true`) uses `session.prompt()` with `SubtaskPartInput` — NOT `session.command()` which doesn't support subtask parts.
- **D-05:** Result output = standard `ToolResponse` envelope from `src/shared/tool-response.ts`.

### Preflight & Frontmatter
- **D-06:** Use `client.command.list()` for command existence preflight — SDK returns `Array<Command>` with `{ name, description, agent?, model?, template, subtask? }`.
- **D-07:** Extend existing command discovery or use SDK `Command` type for `model`/`subtask` frontmatter fields — `CommandBundle` type at `src/routing/command-engine/types.ts:4-17` currently drops these fields.

### Agent Switching
- **D-08:** Mechanism (a) = pass `agent` to `session.command()` — SDK handles switch server-side with `SessionNextAgentSwitched` event.
- **D-09:** Mechanism (b) = record `ctx.agent`, dispatch under target agent, then send follow-up `session.command()` with empty command and prior agent to restore.
- **D-10:** No current-agent introspection API exists in v1 SDK — restore mechanism must cache from `ToolContext`.

### the agent's Discretion
- Exact error message formats for preflight failures
- Fallback behavior between SDK command attempt and TUI retry
- Directory handling for `query: { directory }` in SDK calls
- `unwrapData()` pattern specifics

</decisions>
