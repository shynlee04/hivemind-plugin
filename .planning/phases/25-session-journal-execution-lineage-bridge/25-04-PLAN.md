# Plan 25-04: Event-Tracker-Style Automatic Session Journey Writer

## Objective

Correct Phase 25 from an explicit export-tool-only interpretation into an automatic parser/writer/meta-writer slice inspired by product-detox event-tracker evidence.

## Tasks

1. Add `session-artifact-parser` to parse markdown session artifacts into bounded header, turn, tool, delegation, and counter metadata.
2. Add `session-journey-events` to classify OpenCode hook events into bounded session journey events and write paired JSON/Markdown read models under `.hivemind/sessions/journey-events/`.
3. Wire `sessionJourneyEventObserver` into the plugin event observer chain as best-effort audit projection writer.
4. Export the parser/writer modules from the public package entrypoint.
5. Verify with focused Vitest coverage, typecheck, build, and full tests where possible.

## Acceptance Criteria

- OpenCode session events can automatically create/update `.hivemind/sessions/journey-events/{sessionId}.json` and `.md` without a tool call.
- Parser tests prove header, turn, tool call, and delegation extraction from product-detox-style markdown.
- Writer tests prove bounded JSON/Markdown output and no raw hook payload firehose.
- Plugin wiring test proves automatic observer registration.
- No new `.opencode/state` writer is introduced.

## Threat Model

| Threat | Mitigation |
|---|---|
| Raw payload information disclosure | Journey Markdown renders only bounded event metadata and summary fields. |
| Runtime event handling disruption | Plugin observer catches writer errors and never blocks canonical event processing. |
| State root regression | Writer path is hard-coded under project-root `.hivemind/sessions/journey-events`. |
