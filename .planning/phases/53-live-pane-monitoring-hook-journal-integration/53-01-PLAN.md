# Plan 53-01: Live Pane Monitoring Hook + Journal Integration

## Objective

Add a new hook `src/hooks/pane-monitor.ts` that consumes `pane-captured` events from the P52-expanded `src/features/tmux/observers.ts` observer and persists each event as a 7-field JSON entry under `.hivemind/journal/<sessionId>/<ISO-timestamp>-pane.json` (local runtime state, never committed). The hook enforces a 5s → 10s → 30s exponential backoff schedule (max 3 retries, silent drop on 4th failure) and a 100-entries-per-session-per-hour rate cap with UTC top-of-hour reset. After the hook ships, retroactively append `## L1 Backing (P53)` sections to P42 UAT.md and P43 VERIFICATION.md — these upgrade the L5-only acceptance steps of P42 and the W-01..W-04 spec-drift items of P43 to L1-backed runtime evidence.

The P53 deliverable is the **first runtime consumer** of the P52 event surface; before this hook, `pane-captured` events were emitted and dropped with no persistence path. The single 1-wave plan keeps the scope tight (6 tasks, 1 hook + 1 wiring + 3 test artifacts + 1 commit) so the EXECUTE phase completes within one context window.

```typescript
export interface PaneMonitorOptions {

export interface PaneMonitorCounters {
  written: number;
  retried: number;
  dropped: number;
}

export interface PaneMonitorHandle {
  dispose: () => Promise<void>;
  counters: PaneMonitorCounters;

}

export function createPaneMonitorHook(opts: PaneMonitorOptions): PaneMonitorHandle;
```

**7-field journal entry (D-53-04, locked by CONTEXT — see D-53-13 below):**
```typescript
{
  schemaVersion: 1,                    // number, NOT "1.0" string (CONTEXT-locked)
  eventType: "pane-captured",          // string
  sessionId: string,                   // from event
  paneId: string,                      // from event
  contentLength: number,               // from event
  capturedAt: string,                  // ISO 8601 from event.timestamp
  retryCount: number,                  // 0 on first write; increments per retry
}
```

**Filename format (D-53-03):** `<ISO-8601 timestamp with colons → dashes>-pane.json`
- Example: `2026-06-02T19-30-45-123Z-pane.json`
- Colons replaced by dashes for filesystem safety (Windows + shell quoting)
- Preserves lexicographic sort order (dashes sort before colons in ASCII)

**Backoff schedule (D-53-05, REQ-53-03):**
- Schedule: `[5000, 10000, 30000]` ms
- Max retries: 3 (4 total attempts: 1 initial + 3 retries)
- 4th failure → `counters.dropped++`, return silently, NO throw
- Use `setTimeout` from `node:timers` (built-in, P20 invariant)
- Closure-captured retry handler (D-53-08 in-memory `Map<sessionId, { attempts: number; timer: NodeJS.Timeout }>`)

**Rate limit (D-53-06, D-53-07, REQ-53-04):**
- In-memory `Map<sessionId, { hourEpoch: number; count: number }>`
- On each event: `currentHourEpoch = Math.floor(Date.now() / 3_600_000)`
- If `stored.hourEpoch !== currentHourEpoch` → reset to `{ hourEpoch: current, count: 0 }`
- If `stored.count >= 100` → `counters.dropped++`, return (NO write attempt)
- Otherwise increment `stored.count`, proceed with write

**Path-traversal guard (PATTERNS §4):** Reject `sessionId` containing `/`, `\`, or `..` before path resolution (inline check, do NOT import from `session-tracker` which would create a cycle). On rejection: log warn via `opts.logWarn` (if provided) or `console.warn` fallback, increment `counters.dropped`, return.

**Write path (D-53-04, REQ-53-02):**
1. Compute `sessionDir = path.join(opts.journalRoot, opts.sessionId)`
2. First event for session: `fs.mkdir(sessionDir, { recursive: true })` (mirror `ensureDirectory` at `atomic-write.ts:105-107`)
3. Compute `filename = event.timestamp.replace(/:/g, "-") + "-pane.json"`
4. Compute `filePath = path.join(sessionDir, filename)`
5. `fs.writeFile(filePath, JSON.stringify(entry, null, 2), { encoding: "utf-8", flag: "wx" })` — exclusive create (NO clobber, per REQ-53-02)

**Error handling (D-53-10, D-04 silent-fallback):**
- All `try/catch` boundaries return silently with `counters.dropped++` (last attempt) or `counters.retried++` (intermediate)
- `client.app.log({ level: "warn" })` is the canonical harness log channel; provide `logWarn` option for the plugin.ts wiring to inject this; fall back to `console.warn` for standalone use
- NO `throw` crosses the `dispose`/handler boundary

**`dispose()` semantics (REQ-53-01 acceptance):**
- Sets `disposed: true` flag in closure (the handler wrapper short-circuits when set — chosen over `paneCaptureListeners.splice(index, 1)` because P52 does NOT expose a `removePaneListener` API and P53 is forbidden from mutating `observers.ts`)
- Clears any in-flight retry timer
- Awaits all pending retries (via the `__waitForPendingRetries` test seam)
- After dispose: firing a `pane-captured` event does NOT write a file (BATS acceptance: file count unchanged)

**JSDoc requirements (AGENTS.md):** Full JSDoc on `createPaneMonitorHook`, `PaneMonitorHandle`, `PaneMonitorOptions`, `PaneMonitorCounters`. Brief one-liner on private helpers per P51 precedent (`delegation-consumer.ts:18-25`).

**`[Harness]` error prefix:** All throws (e.g., from path-traversal guard) use the prefix.

```bash
  tmux_bats_require_dist
  tmux_bats_make_project

# REQ-53-02 acceptance: the hook writes a journal file with 7 fields
@test "pane-monitor writes 7-field JSON journal entry on pane-captured event" {
  local journal_root="${BATS_TEST_TMPDIR}/project/.hivemind/journal"
  run tmux_node_eval "
    import('${TMUX_BATS_ROOT}/dist/features/tmux/observers.js').then(async (obs) => {
      const { createPaneMonitorHook } = await import('${TMUX_BATS_ROOT}/dist/hooks/pane-monitor.js');

      handler({ sessionId: 'test-session', paneId: '%7', contentLength: 2048, timestamp: '2026-06-02T12:34:56.789Z' });

      await hook.dispose();
      process.stdout.write('written=' + hook.counters.written);
    });

  [ "$status" -eq 0 ]

  run bash -c "ls ${journal_root}/test-session/ | grep -E '2026-06-02T.*-pane.json'"
```
