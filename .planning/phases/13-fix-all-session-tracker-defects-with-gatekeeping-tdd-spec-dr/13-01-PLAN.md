<objective>
Fix the session initialization path to eliminate cross-contamination where child sessions get incorrectly registered as main sessions.

Purpose: The current `ensureSessionReady()` method creates directories and registers ALL sessions (including children) in the project index. This is the root cause of dual registration — 4 of 5 child sessions appear in BOTH their parent's directory AND have their own top-level directory. Fixing this stops the contamination at the source.

Output: Corrected `ensureSessionReady()` that skips child sessions, plus deduplicated initialization flow where only ONE code path creates session directories.
</objective>
