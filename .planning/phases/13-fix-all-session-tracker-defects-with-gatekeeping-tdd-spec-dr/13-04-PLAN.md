<objective>
Add per-session serial write queues to SessionIndexWriter and ChildWriter to prevent silent data loss under concurrent writes.

Purpose: Both SessionIndexWriter and ChildWriter perform read-modify-write operations without serialization. When 6 concurrent sessions write simultaneously, interleaved reads and writes WILL cause data loss. This follows the proven `enqueueWrite` pattern already used in ProjectIndexWriter.

Output: SessionIndexWriter and ChildWriter each gaining a serial write queue with stale-queue detection, matching the ProjectIndexWriter pattern.
</objective>
