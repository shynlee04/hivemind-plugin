<objective>
Wire turn counting into the message capture pipeline so that `session-continuity.json` turnCount reflects actual user messages.

Purpose: Currently ALL 15 session-continuity.json files show `turnCount: 0` even though `.md` files contain hundreds of lines of user turns. The `MessageCapture.handleUserMessage` method writes user turns to `.md` files but never calls `SessionIndexWriter.incrementTurnCount()`. This fix injects the missing dependency and calls it.

Output: MessageCapture receives a SessionIndexWriter via constructor injection, and `handleUserMessage` calls `incrementTurnCount(sessionID)` after each user message capture.
</objective>
