### Critical Gap: Current System Is "Working but Useless"

The `delegate_task` tool spawns a child session and returns immediately (WaiterModel), but:
- No one polls the child session
- No notification is delivered to the parent when the child completes
- No implicit detection — user must explicitly know to say "delegate"
- No loop protection, no stale detection, no result aggregation

### OMO Patterns to Adopt

- **Two-phase spawn reservation** (`reserveSubagentSpawn` → `commit`/`rollback`) — prevents descendant count drift
- **Hook-based notification delivery** via `messages.transform` — not callback-based (leaks, races)
- **Adaptive polling** — accelerate when child is active, decelerate when idle
- **Circuit breaker** on repeated tool signatures across child sessions
- **Implicit detection** via `category-skill-reminder` hook pattern (detect after 3+ tool calls, multi-domain scope, attachments)
