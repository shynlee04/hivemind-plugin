<objective>
Close Gap 4 (PTY not the actual execution path) by rethinking the PTY integration as honest dual-mode delegation: SDK execution for agent tasks, PTY execution for command tasks. Add a standalone PTY tool for arbitrary background CLI commands — addressing the user's concern that PTY has broader applications beyond delegation.

Purpose: The current code pretends PTY is the execution path for agent delegations, but the delegated prompt actually runs through client.session.prompt(). This is architecturally dishonest and leads to the dead-end the user identified. The fix is to: (1) honestly report "sdk" for agent delegations, (2) add true PTY command delegation as a new execution mode, and (3) expose PTY as a standalone capability for arbitrary background commands.

Output: Honest dual-mode delegation system + standalone PTY command tool. Agent delegations run via SDK (executionMode "sdk"). Command delegations run via PTY (executionMode "pty"). A new run-background-command tool lets agents run arbitrary CLI commands in background PTY processes.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| run-background-command → PTY process | Agent-triggered arbitrary command execution — highest privilege boundary |
| dispatchCommand → PtyManager.spawn | Command delegation spawns child processes with controlled env |
| Tool input → PTY stdin | Interactive input flows to child process — injection risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-06-01 | E (Elevation of Privilege) | run-background-command tool | mitigate | Tool only available when PTY is supported; commands run with user's own permissions; no sudo elevation |
| T-16-06-02 | S (Spoofing) | Command delegation output | accept | Output comes from PtyManager's own buffer — trusted within process boundary |
| T-16-06-03 | D (Denial of Service) | Unbounded PTY process spawning | mitigate | DelegationManager's semaphore limits concurrent delegations; safety ceiling enforces max runtime; PtyManager buffer is bounded |
</threat_model>
