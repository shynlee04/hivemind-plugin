<objective>
Close Gap 4 by replacing the untruthful "PTY metadata alongside SDK execution" design with a truthful dual-mode model: SDK child-session execution for agent delegations, PTY execution for command/process surfaces, and one shared PTY subsystem.

Purpose: Verification proved the earlier "PTY default for all delegations" wording produced disconnected metadata instead of a real PTY execution path. Per the D-04A amendment in CONTEXT, this plan must make the architecture truthful: (1) agent delegations report/use the SDK path they actually run on, (2) command delegations use PTY for real process execution under the same queue/safety rules, and (3) the standalone PTY tool reuses the same canonical PTY/session universe instead of creating a second owner.

Output: Truthful dual-mode delegation system + shared-state standalone PTY tool. Agent delegations run via SDK (`executionMode: "sdk"`). Command/process execution runs via PTY (`executionMode: "pty"`) with shared queue governance, shared PTY session tracking, and persistence normalization.
When PTY is unavailable on command/process surfaces, the same governance path must degrade truthfully to `executionMode: "headless"` with `fallbackReason`, without relabeling agent work or inventing fake PTY state.
</objective>
