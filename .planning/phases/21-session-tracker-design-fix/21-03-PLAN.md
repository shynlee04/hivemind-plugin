## Scope Boundary

Only child `.json` metadata. The `agentName`, `subagentType`, `model`, and `description` fields in the persisted `ChildSessionRecord`. Do NOT touch delegation system, dispatch, or completion logic — only the metadata that flows into event-capture at child creation time.

## Design Decision

The metadata flows as follows:
1. `tool-capture.ts` receives delegate-task dispatch → extracts `agent`, `model`, `description` from tool args
2. Passes to `event-capture.ts:writeImmediateChildFile()`
3. If metadata is unavailable at creation (race condition), store `"pending"` as before
4. Delegation completion handler → calls `child-writer.ts:backfillChildMetadata()` → replaces `"pending"` with real values

## Exit Criteria

- Integration test: dispatch delegate-task with `hm-l2-researcher` → child `.json` shows `agentName: "hm-l2-researcher"`
- Unit test: backfill replaces `"pending"` → `"hm-l2-researcher"` after completion
- All existing tests pass
