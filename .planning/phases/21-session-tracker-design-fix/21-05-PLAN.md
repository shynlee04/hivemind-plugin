<objective>
Fix two preconditions that block Phase 22-25: (G-3) stop `addSession()` from overwriting session status to "active" on every hook callback, and (G-4) remove the `commit_docs` gate from `persistDelegations()` so delegation records are always persisted.

**Purpose:** P22 (status unification) needs `project-continuity.json` to be a trustworthy canonical status store. Currently, every hook callback that calls `addSession()` resets `existing.status = "active"` — a session set to "idle" by the lifecycle manager is silently reverted to "active" on the next tool call. Separately, delegation persistence is gated behind `config.commit_docs` — a config flag whose name describes git commit behavior, not delegation persistence. Since `commit_docs` defaults to `false`, delegation records are NEVER persisted by default. P23 (dispatch redesign) needs delegation records to be written reliably.

**Output:** `addSession()` preserves existing status rather than setting to "active"; `persistDelegations()` always writes regardless of config.
</objective>

Lines 167-170 are the problematic code:

```typescript
existing.updated = now

```

This block runs EVERY time a hook callback calls `addSession()` for an already-registered session. The only guards are for "error" and "deleted" — all other status values (including "idle", "completed", "running") get overwritten to "active".

```typescript
  if (!config.commit_docs) {
    return  // Skip document persistence
  }
```

The early return at L62-63 prevents ALL delegation persistence when `commit_docs` is false (the default). The function never reaches the write logic below.
