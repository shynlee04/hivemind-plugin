## Scope Boundary

Manifest becomes read-time generated. `writeManifest()` remains as an optimization (write-generated copy to disk for fast reads), but the source of truth is ALWAYS the continuity tree. `childCount`/`totalDelegationDepth` are computed from the hierarchy index at write time, not stored as separate fields.

## Exit Criteria

- Unit test: continuity tree → `generateFromContinuity()` → manifest matches
- Integration test: root + 5 children → `project-continuity.json` shows `childCount: 5`
- Integration test: root + L1 + L2 → `totalDelegationDepth: 2`
- All existing tests pass
