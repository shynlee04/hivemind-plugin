<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| OpenCode hooks → harness manager | Runtime event facts enter coordination layer; hooks must not mutate durable state. |
| child session facts → parent notification | Child output/status becomes parent-visible text; risk of leakage/noise. |
| timer callbacks → session context injection | Automated status injections can pollute context if unbounded. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-14-05 | Denial of Service | `DelegationMonitor` timers | mitigate | Cadence bounded to 30/45/60/90/120/180 and final checkpoint stops injection after level 4. |
| T-14-06 | Information Disclosure | Notification message body | mitigate | Use deterministic thin-line/system-reminder format and preserve redaction in later status rendering; no raw secret dump in notification formatter. |
| T-14-07 | Tampering | Hook signal forwarding | mitigate | Hooks call injected manager signal methods only; no durable writes from hook body. |
| T-14-08 | Repudiation | Completion/failure events | mitigate | Integration tests require delegation ID and timestamp/idempotency key in routed notifications. |
</threat_model>
