<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Caller session → delegation control | A user/session attempts to control a delegation that may belong to another parent/child lineage. |
| Control tool → child session SDK | Tool can abort or append prompts to child sessions. |
| Parent slot accounting → dispatch admission | Parent session gets concurrent delegation capacity; incorrect accounting can starve or bypass cap. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-14-09 | Elevation of Privilege | `delegation-status.ts` control action | mitigate | Keep `canSessionAccessDelegation()` check before manager control call; tests cover unauthorized denial. |
| T-14-10 | Tampering | `resume` / `chain` prompt append | mitigate | Require same delegation/parent lineage; same-parent chain only; no cross-parent append. |
| T-14-11 | Denial of Service | per-parent slots | mitigate | Enforce max 10 active delegations per parent session and release slots on terminal cleanup. |
| T-14-12 | Spoofing | `changeAgent` / resume agent override | mitigate | Validate agent/changeAgent through existing agent resolver/dispatch checks; do not bypass permission resolution. |
</threat_model>
