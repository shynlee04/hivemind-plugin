<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| OpenCode tool caller → delegate-task tool | Untrusted tool args and plugin context enter harness write-side. |
| delegate-task tool → DelegationManager | Validated request crosses from tool boundary into coordination state machine. |
| child session dispatch → parent TUI route | Child delegation status is routed back to parent session and could leak if session ownership is wrong. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-14-01 | Tampering | `manager-runtime.ts` delegation record persistence | mitigate | Register/persist delegation before monitor start; prompt failure transitions to terminal error instead of leaving corrupt running state. |
| T-14-02 | Information Disclosure | `NotificationRouter.register` parent mapping | mitigate | Register by exact delegation ID and parentSessionId; tests assert parent ID passed from dispatch params. |
| T-14-03 | Denial of Service | Monitor startup on failed dispatch | mitigate | `monitor.start` only after successful `sendPromptAsync`; failure test asserts no timer starts on prompt failure. |
| T-14-04 | Spoofing | ToolContext session identity | mitigate | Continue using `context.sessionID` from OpenCode plugin context; do not invent unsupported `context.task` seam. |
</threat_model>
