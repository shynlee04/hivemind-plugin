## Commit Strategy Note

The plan's Atomic Commit Strategy section listed 23 commits. The actual
delivery is 17 commits. The reductions came from:

1. **R3 (vitest fixes) collapsed to 1 commit** — The plan listed
   `phase-58-9(test): fix 5 vitest regressions for REQ-02 full-suite-only
   timeouts` as a single commit. The actual delivery has 3 separate
   commits (one per test file), per atomicity. The 5 test-file changes
   could not be split further because each test is a single
   logical change.

2. **R4 (regression guards) collapsed to 1 atomic commit** — The 4
   regression guard test files + 1 baseline fixture are tightly coupled
   (the test references the fixture). Splitting them would create
   intermediate state where the no-new-deps test references a
   non-existent fixture. The single atomic commit preserves
   test-fixture co-evolution.

3. **Wave 2D verification note** — The plan called for 5 separate
   commits (I14-I18), but the 4 test files + 1 fixture landed in 1
   atomic commit (R4). A separate verification note was added
   (`58.9-WAVE-2D-VERIFY.md`) as a docs-only commit to document the
   AC coverage.

The total logical changes match the plan (23 distinct actions),
delivered in 17 atomic commits. Each commit passes `npx tsc --noEmit`
and the affected test file in isolation. The substance is identical;
the commit count reduction is purely a hygiene optimization.
