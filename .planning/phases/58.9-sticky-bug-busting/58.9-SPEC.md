## Goal

P58.9 is the **sticky-bug-busting follow-up to the P58.8 gap-fix**. A P51+ regression hunt at `.planning/debug/p51-plus-sticky-bugs-2026-06-04.md` (153 lines, gsd-debugger READ-ONLY investigation) found **12 bugs** that survived the P58.8 gap-fix: 3 critical, 4 high, 3 medium, 2 low. P58.9 closes the 3 critical bugs and adds regression guards for the categories that, while currently PASS, are not under automated guard.

The 3 critical sticky bugs to fix:

1. **P53 journal hook broken** — `SessionManager.startPolling` at `src/features/tmux/session-manager.ts:328-356` captures content but never emits `pane-captured` events. The P53 pane-monitor hook receives nothing; `.hivemind/journal/<sid>/` stays empty despite the JSDoc at L316-317 claiming the tick emits events.
2. **5 new vitest regressions introduced by P58.8** — `eval/coherence.test.ts:37,106`; `bootstrap-tools-registration.test.ts:59`; `delegate-task.test.ts:197,239` all time out in full-suite. P58.8 INCREASED the failure count from 2 to 7 without addressing the pre-existing 2.
3. **BATS structural bypass for tmux runtime** — 7 of 8 tmux-related BATS slots (62, 63, 64, 71, 72, 73, 74) use `cat` or inline mocks and never exercise the real `opencode attach` TUI path. The user's S1 PUSH complaint cannot be reproduced in BATS.

P58.9 also adds **automated regression guards** for 5 other categories (P52 action count, P51 in-tree tmux contamination, AC#10/AC#11 manualOverride, 27-tool-key, P20 no-new-deps) so that the next gap-fix doesn't need to find them by hand.

P58.9 preserves the 27-tool-key invariant, the P20 no-new-deps invariant, the AC#10/AC#11 manualOverride guards, and the 3,303+ vitest baseline (target: net ≤ 2 pre-existing failures, ZERO new regressions).

## Requirements

### 1. **Wire `pane-captured` event emit in `SessionManager.startPolling`** (REQ-58.9-01) — LOCKED 2026-06-04

**User-Pain symptom:** P53 pane-monitor journal hook (`src/hooks/pane-monitor.ts:107-115` 7-field `JournalEntry` shape) subscribes to `pane-captured` events via `tmuxObserver.onPaneCaptured` (`src/plugin.ts:772-786`) and persists entries to `.hivemind/journal/<sid>/<ts>-pane.json`. The P58.8 polling tick at `src/features/tmux/session-manager.ts:328-356` claims via JSDoc (L316-317) to emit these events but **does not** — verified by `grep "emit|setObserver|onPaneCaptured" src/features/tmux/session-manager.ts` returning no matches. The result: `.hivemind/journal/` directory is empty (only `.gitkeep` exists) despite the polling loop running.

- **Current:** `src/features/tmux/session-manager.ts:328-356` `startPolling` calls `multiplexer.capturePaneContent(paneId, 5000)`, stores result in `this.latestCapture`, updates `this.lastCaptureHash`, but never calls any observer/emit/event-bus dispatch. `SessionManager` does not have an observer field; it is constructed without a `tmuxObserver` dependency.
- **Target:**
  - Add a `private observer: PaneObserver | null = null` field to `SessionManager` class.
  - Add a `setObserver(observer: PaneObserver): void` method to wire the observer into `SessionManager`.
  - In `startPolling` `tick()` after `this.latestCapture.set(tracked.paneId, capture)`, emit a `pane-captured` event with shape `{ eventType: "pane-captured", sessionId: tracked.sessionId, paneId: tracked.paneId, contentLength: capture.byteLength, content: capture.content, capturedAt: capture.capturedAt }`.
  - The emitted event shape MUST be compatible with the P53 hook's `tmuxObserver.onPaneCaptured` consumer (which expects `{ sessionId, paneId, contentLength, capturedAt, content }` per `src/hooks/pane-monitor.ts:107-115` + 7-field journal shape).
  - `PaneObserver` interface defined in `src/features/tmux/observers.ts` (existing file) with method `onPaneCaptured(event: PaneCapturedEvent): void`.
  - Wire `sessionManager.setObserver(tmuxObserver)` in `src/plugin.ts` at the same place that constructs `SessionManager` (search for `new SessionManager(`).
  - **P53 hook is unchanged.** The existing P53 hook at `src/hooks/pane-monitor.ts` already calls `journal.write({...})` for `pane-captured` events; the new emit just feeds it.
- **Acceptance:**
  - **AC-58.9-01-01:** After `startPolling` captures content, `pane-captured` event fires to the registered observer (verified by mock observer receiving the event in vitest).
  - **AC-58.9-01-02:** P53 hook receives the event; `.hivemind/journal/<sid>/<ts>-pane.json` file is created with 7 fields per the locked `JournalEntry` interface (`schemaVersion: 1`, `eventType: "pane-captured"`, `sessionId`, `paneId`, `contentLength`, `capturedAt`, `retryCount`).
  - **AC-58.9-01-03:** RED BATS `tests/scripts/tmux/75-pane-captured-journal.bats` (slot 75) — assert the journal file exists with correct schema after the polling tick fires at least once.
  - **AC-58.9-01-04:** RED-FIRST enforced — BATS 75 committed with `(red)` marker BEFORE the implementation.
  - **AC-58.9-01-05:** 27-tool-key invariant preserved — no new tool registrations in `src/plugin.ts`; the event is emitted via the existing `tmuxObserver` instance.

### 3. **BATS slot 75 — real `opencode attach` TUI in pane** (REQ-58.9-03) — LOCKED 2026-06-04

- **Target:**

  - The scenario starts the `SessionManager` polling loop on the tmux pane.
  - The scenario writes a synthetic event via the SDK to drive the opencode attach TUI (e.g., a `chat.message` part injected into the test session).
  - The scenario asserts the pane content updates within 1 second of the synthetic event (via `tmux capture-pane -t <paneId> -p | grep -c <probe-text> >= 1`).
  - The scenario teardown kills the tmux session and stops the polling loop.
- **Acceptance:**
  - **AC-58.9-03-01:** `tests/scripts/tmux/75-pane-real-runtime.bats` exists.
  - **AC-58.9-03-02:** BATS spawns real `opencode attach` in pane (NOT `cat` mock).
