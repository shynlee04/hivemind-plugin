    ```
    ...(tmuxIntegration
      ? [createTmuxEventObserver({
          onSessionCreated: async (enriched: EnrichedSessionEvent) => {
            // The enriched event is forwarded to the fork plugin's SessionManager
            // via OpenCode's standard event dispatch. The Hivemind-side observer
            // enriches with metadata; the fork plugin (loaded as separate OpenCode
            // plugin in opencode.json) receives the same event and spawns panes.
            // No direct method call needed — OpenCode event system handles routing.
            // This observer exists for cases where we need Hivemind-side control
            // over the enriched event before the fork handles it.
            void enriched // intentional: OpenCode event dispatch reaches fork plugin
          },
        })]
      : []),
    ```

    **Key design note:** The fork plugin is loaded as a SEPARATE OpenCode plugin in the user's `opencode.json` configuration. Both Hivemind's event observer and the fork's event handler receive the same `session.created` event via OpenCode's standard event dispatch. The Hivemind observer is a metadata enrichment point. The fork's own event handler (registered via its own plugin factory) receives the original un-enriched event — the enrichment here is for Hivemind's own awareness. The actual enriched event shape (`EnrichedSessionEvent`) is the contract that Hivemind's metadata layer agrees with the fork's SessionManager.

    **However** — since OpenCode dispatches the SAME event object to ALL plugins, and the fork plugin is a separate plugin in `opencode.json`, the fork receives the original `EventSessionCreated` from OpenCode SDK, NOT the enriched version from Hivemind's observer. The fork's enrichment path relies on the fork being loaded inside Hivemind (not as a separate plugin). 

    **Recommended approach for Phase 42:** Keep the fork as a separate OpenCode plugin AND create a thin Hivemind-owned bridge. The fork's SessionManager methods are invoked from Hivemind's observer instead of relying on OpenCode's event dispatch:

    Replace with:
    ```
    ...(tmuxIntegration
      ? [createTmuxEventObserver({
          onSessionCreated: async (enriched: EnrichedSessionEvent) => {
            // Hivemind-side metadata enrichment
            // In current Phase 42, this is a placeholder that logs/records
            // the enriched event for future Phase 43 co-pilot integration.
            // The fork plugin receives un-enriched events from OpenCode and
            // handles pane lifecycle independently.
            // Phase 43 will wire Hivemind's enriched events into fork's
            // SessionManager for copilot-aware pane spawning.
          },
        })]
      : []),
    ```

    This keeps Phase 42 clean: fork handles its own events, Hivemind observer enriches metadata for future use. Phase 43 will bridge the two.
