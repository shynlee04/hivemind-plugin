# External Integrations

**Analysis Date:** 2026-03-21

## OpenCode SDK Integration

**Plugin Hooks (12 used):**
- `event` - Lifecycle event handling via `src/hooks/event-handler.ts`
- `chat.message` - Per-session message tracking, governance context injection
- `permission.ask` - State mutation gating with toast notifications for write operations
- `tool.execute.before` - Pre-tool execution tracking for managed tools
- `tool.execute.after` - Post-tool state capture and trajectory tracking
- `shell.env` - Environment variable injection (HIVEMIND_RUNTIME_ATTACHED, HIVEMIND_ATTACHMENT_MODE, etc.)
- `experimental.chat.messages.transform` - Message history injection (context packets, route hints)
- `experimental.session.compacting` - Session compaction context preservation
- `command.execute.before` - Pre-command context injection and tool precedence
- `system.transform` - System prompt transformation (deprecated in Phase 11)
- `chat.params` - Per-agent parameter control (adopted later)

**Client API Surface:**
- `createOpencodeClient()` - SDK client creation in `src/control-plane/sdk-runtime.ts` and `src/tui/sse.ts`
- `client.app.log()` - Structured server-side logging via `src/shared/logging.ts`
- `client.tui.showToast()` - User-facing toast notifications via `src/hooks/soft-governance.ts`
- `client.event.subscribe()` - Server-SSE event subscription in TUI

**Tool Registration (8 tools):**
All tools registered through `src/plugin/opencode-plugin.ts` tool object:
- `hivemind_doc` - Read-only document intelligence tool
- `hivemind_task` - Workflow task management tool
- `hivemind_trajectory` - Trajectory state tool
- `hivemind_handoff` - Delegation handoff tool
- `hivemind_runtime_status` - Runtime status inspection tool
- `hivemind_runtime_command` - Runtime command execution tool
- `hivemind_agent_work_create_contract` - Agent work contract creation
- `hivemind_agent_work_export_contract` - Agent work contract export

## Data Storage

**State Persistence:**
- Filesystem-based state in `.hivemind/` directory
- Trajectories: `.hivemind/trajectory/{id}.json` - State authority in `src/core/trajectory/`
- Workflows: `.hivemind/workflow/{id}.json` - State authority in `src/core/workflow-management/`
- Sessions: `.hivemind/sessions/{id}.json` - Session records
- Schema kernel records: `.hivemind/state/schema-kernel/` - Contract authority in `src/schema-kernel/`

**No Databases:**
- No SQL/NoSQL database
- No external caching layer
- All state is JSON file-based

**File Storage:**
- Local filesystem only
- No cloud storage integration
- Runtime directory is generated by `hm-init` command

## Data Flow Patterns

**CQRS Architecture:**
- Tools (`src/tools/`) are write-side
- Hooks (`src/hooks/`) are read-side/intercept
- Plugin (`src/plugin/`) assembles read and write planes

**Control Plane (`@opencode-ai/sdk`):**
- CLI operations in `src/cli/` and `src/control-plane/`
- Creates OpenCode servers via `createOpencode()`
- Attaches to runtime via `createOpencodeClient()`
- Never uses plugin hooks

**Execution Plane (`@opencode-ai/plugin`):**
- Plugin entry in `src/plugin/` registers tools/hooks
- Tools execute inside agent loop
- Uses `ToolContext` (sessionID, agent, directory, worktree, abort)
- Never uses Client SDK (prevents recursion)
