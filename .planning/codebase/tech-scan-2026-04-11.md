## External SDK Usage (@opencode-ai/plugin APIs)

### Plugin API
- **`Plugin` type** — async function signature `({ client }) => Promise<PluginHooks>` used in `src/plugin.ts`
- The plugin receives an OpenCode client instance and returns a merged object of hooks + tools

### Tool API
- **`tool()` function** from `@opencode-ai/plugin/tool` — factory for defining OpenCode tools
- Used in all 5 tool implementations:
  - `delegate-task` — `src/tools/delegate-task.ts`
  - `background` — `src/tools/background/index.ts`
  - `prompt-skim` — `src/tools/prompt-skim/tools.ts`
  - `prompt-analyze` — `src/tools/prompt-analyze/tools.ts`
  - `session-patch` — `src/tools/session-patch/tools.ts`

### OpenCode Client SDK (@opencode-ai/sdk)
- **Type-only usage:** `ReturnType<typeof createOpencodeClient>` produces the `OpenCodeClient` type
- **Session API methods wrapped:**
  - `session.create({ body, query })` — create child sessions
  - `session.get({ path: { id } })` — fetch session details
  - `session.status()` — get status map for all sessions
  - `session.abort({ path: { id } })` — abort a session
  - `session.messages({ path: { id }, query: { limit } })` — fetch session messages
  - `session.prompt({ path: { id }, body })` — synchronous prompt
  - `session.promptAsync({ path: { id }, body })` — async prompt (returns 204)
- **Additional client methods used directly (not via session-api.ts wrappers):**
  - `client.session.abort()` — in `src/lib/lifecycle-manager.ts:230` (direct call, bypasses wrapper)
  - `client.session.prompt()` — in `src/lib/notification-handler.ts:146` (direct call for notifications)

### Hook API
- **No imports from `@opencode-ai/plugin/hook`** — hooks are plain objects returned by the plugin function. OpenCode matches them by key name convention (e.g., `event`, `system.transform`, `messages.transform`, `tool.execute.before`, `tool.execute.after`, `shell.env`, `session.compacting`).
