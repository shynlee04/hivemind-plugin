## Directory Purposes

**`src/` — Hard Harness (npm package):**
- Purpose: All business logic, tools, hooks, and plugin assembly for `opencode-harness`
- Contains: TypeScript source organized by responsibility (hooks, tools, lib, shared, schema-kernel)
- Key files: `plugin.ts` (composition root), `index.ts` (public API)

**`src/hooks/` — Event Hook Factories:**
- Purpose: Factory functions that produce OpenCode hook handlers
- Contains: Three main factories (core, session, tool-guard) plus shared types and message transformer
- Key files: `create-core-hooks.ts` (event routing), `create-session-hooks.ts` (auto-loop), `create-tool-guard-hooks.ts` (budget enforcement)

**`src/tools/` — Plugin Tools:**
- Purpose: User-facing callable tools registered with OpenCode
- Contains: Tool implementations grouped by feature (delegation, prompt-enhance)
- Key files: `delegate-task.ts` (core delegation), `delegation-status.ts` (status polling), `run-background-command.ts` (PTY commands)

**`src/lib/` — Core Library:**
- Purpose: Business logic — delegation orchestration, state management, persistence, concurrency
- Contains: All library modules, spawner subsystem, PTY subsystem
- Key files: `delegation-manager.ts` (orchestration), `continuity.ts` (persistence), `state.ts` (in-memory state), `types.ts` (shared types)

**`src/lib/spawner/` — Spawner Subsystem:**
- Purpose: Child session creation with permission profiles
- Contains: Session creation, directory resolution, concurrency key building, type contracts
- Key files: `session-creator.ts` (SDK session creation), `spawner-types.ts` (contracts)

**`src/lib/pty/` — PTY Subsystem:**
- Purpose: Interactive command execution via pseudo-terminals
- Contains: PTY manager (bun-pty wrapper), ring buffer, type contracts
- Key files: `pty-manager.ts` (PTY lifecycle)

**`src/shared/` — Cross-Cutting Utilities:**
- Purpose: Shared tool response patterns and helpers
- Contains: Response envelope types, JSON serialization
- Key files: `tool-response.ts` (success/error/pending envelope)

**`src/schema-kernel/` — Zod Schemas:**
- Purpose: Type-safe contracts for prompt-enhance pipeline
- Contains: All Zod schemas and inferred TypeScript types
- Key files: `prompt-enhance.schema.ts` (all schemas)

**`tests/` — Test Suite:**
- Purpose: Unit and integration tests mirroring `src/` structure
- Contains: Vitest test files with globals (no imports needed for `describe`, `it`, `expect`)
- Key files: Mirror `src/` structure — `tests/lib/*.test.ts`, `tests/tools/*.test.ts`

**`.opencode/` — Soft Meta-Concepts:**
- Purpose: OpenCode configuration — agents, skills, commands, rules
- Contains: 57 agent definitions, 22 skills, 13 commands, governance rules
- Not committed as part of npm package — project configuration only

**`dist/` — Compiled Output:**
- Purpose: TypeScript compilation output (ES2022, NodeNext modules)
- Generated: Yes — `npm run build` compiles `src/` → `dist/` with declarations + source maps
- Committed: No — generated artifact

## Key File Locations

**Entry Points:**
- `src/plugin.ts`: OpenCode plugin composition root — instantiates managers, wires hooks, registers tools
- `src/index.ts`: Public API barrel — re-exports `HarnessControlPlane` as default + all lib modules

**Configuration:**
- `package.json`: npm package manifest — name `opencode-harness`, Node >= 20, peer dep `@opencode-ai/plugin` >= 1.1.0
- `tsconfig.json`: TypeScript strict mode, ES2022 target, NodeNext modules, `verbatimModuleSyntax`
- `vitest.config.ts`: Vitest test configuration with globals enabled
- `opencode.json`: OpenCode project config — references `AGENTS.md` as instructions

**Core Logic:**
- `src/lib/delegation-manager.ts`: Delegation orchestration — dispatch, recovery, safety ceiling, session event handling
- `src/lib/continuity.ts`: Durable JSON persistence — load, save, normalize, deep-clone
- `src/lib/state.ts`: `TaskStateManager` — in-memory Maps for stats, budgets, delegation metadata
- `src/lib/concurrency.ts`: `DelegationConcurrencyQueue` — keyed semaphore with priority lanes
- `src/lib/completion-detector.ts`: `CompletionDetector` — two-signal completion (idle + stability)
- `src/lib/sdk-delegation.ts`: `SdkDelegationHandler` — stability polling, result extraction
- `src/lib/command-delegation.ts`: `CommandDelegationHandler` — PTY dispatch, headless fallback
- `src/lib/runtime-policy.ts`: Policy loading, validation, per-session override resolution

**Testing:**
- `tests/lib/`: Unit tests for library modules — mirrors `src/lib/` structure
- `tests/tools/`: Unit tests for tool implementations
- `tests/plugins/`: Plugin-level tests (lifecycle, prompt-enhance compaction)
- `tests/integration/`: End-to-end pipeline tests
- `tests/lib/helpers/in-memory-client.ts`: Mock OpenCode client for test isolation

## Naming Conventions

**Files:**
- kebab-case for all source files: `delegation-manager.ts`, `create-core-hooks.ts`
- `.test.ts` suffix for test files: `delegation-manager.test.ts`
- `index.ts` for barrel/directory entry points: `tools/prompt-skim/index.ts`
- `*-types.ts` suffix for type-only modules: `spawner-types.ts`, `pty-types.ts`

**Directories:**
- kebab-case: `prompt-skim/`, `session-patch/`, `schema-kernel/`
- Singular for subsystems: `spawner/`, `pty/`, `hooks/`, `tools/`, `lib/`

**Exports:**
- PascalCase for classes: `DelegationManager`, `TaskStateManager`, `PtyManager`, `CompletionDetector`
- camelCase for functions: `createCoreHooks`, `getSessionContinuity`, `buildDelegationQueueKey`
- PascalCase for types/interfaces: `Delegation`, `SessionContinuityRecord`, `RuntimePolicy`
- UPPER_SNAKE_CASE for constants: `MAX_DESCENDANTS_PER_ROOT`, `STABILITY_THRESHOLD`, `DEFAULT_SAFETY_CEILING_MS`

**Import Style:**
- `import type` for type-only imports (enforced by `verbatimModuleSyntax: true`)
- `.js` extension in all import paths (ESM convention, resolved by NodeNext)
- Relative imports only — no path aliases

## Where to Add New Code

**New Tool:**
- Implementation: `src/tools/<tool-name>/index.ts` (factory function returning `tool()`)
- Types: `src/tools/<tool-name>/types.ts` (if complex)
- Tests: `tests/tools/<tool-name>.test.ts`
- Registration: Add to `tool` object in `src/plugin.ts`

**New Hook:**
- Factory: `src/hooks/create-<hook-name>-hooks.ts` (export factory function)
- Types: Add to `src/hooks/types.ts` if new dependency shape needed
- Wiring: Spread factory result into plugin return in `src/plugin.ts`
- Tests: `tests/hooks/<hook-name>.test.ts` (if hooks test dir exists) or `tests/lib/`

**New Library Module:**
- Implementation: `src/lib/<module-name>.ts`
- Types: Add to `src/lib/types.ts` if shared, or inline if module-local
- Tests: `tests/lib/<module-name>.test.ts`
- Re-export: Add to `src/index.ts` if public API

**New Spawner Feature:**
- Implementation: `src/lib/spawner/<feature>.ts`
- Types: `src/lib/spawner/spawner-types.ts` (extend existing contracts)
- Tests: `tests/lib/spawner/<feature>.test.ts`

**New PTY Feature:**
- Implementation: `src/lib/pty/<feature>.ts`
- Types: `src/lib/pty/pty-types.ts`
- Tests: `tests/lib/pty/<feature>.test.ts`

**New Zod Schema:**
- Implementation: `src/schema-kernel/prompt-enhance.schema.ts` (add schema + inferred type)
- Re-export: Add to `src/schema-kernel/index.ts` barrel

**New Shared Utility:**
- Implementation: `src/shared/<utility>.ts`
- Tests: Co-located test or `tests/lib/helpers.test.ts`
