│   ├── coordination/             # Delegation, completion, concurrency, spawner

│   ├── hooks/                    # Lifecycle, guards, observers, transforms, CQRS

## Directory Purposes

**src/plugin.ts:**
- Purpose: Plugin composition root — wires tools, hooks, lifecycle, delegation
- Key files: `src/plugin.ts` (267 LOC)

**src/index.ts:**
- Purpose: Public API re-exports — `import hivemind` and `import hivemind/plugin`
- Key files: `src/index.ts` (26 LOC)

**src/cli/:**
- Purpose: CLI substrate for `npx hivemind` commands
- Contains: Router, discovery, commands (version, init, help, doctor, recover)
- Key files: `src/cli/index.ts`, `src/cli/router.ts`, `src/cli/commands/doctor.ts`

**src/config/:**
- Purpose: Config compilation, subscription, workflow state machine
- Contains: subscriber (lazy cache), compiler (workspace → runtime policy), workflow
- Key files: `src/config/subscriber.ts`, `src/config/compiler.ts`

**src/coordination/:**
- Purpose: Delegation dispatch, completion detection, concurrency gating, session spawning
- Contains: delegation/, completion/, concurrency/, sdk-delegation/, command-delegation/, spawner/
- Key files: `src/coordination/delegation/manager.ts` (~500 LOC), `src/coordination/concurrency/queue.ts` (310 LOC), `src/coordination/completion/detector.ts`

**src/features/:**
- Purpose: Standalone runtime features
- Contains: session-tracker/, bootstrap/, background-command/pty/, prompt-packet/, doc-intelligence/, runtime-pressure/, agent-work-contracts/, sdk-supervisor/, steering-engine/
- Key files: `src/features/session-tracker/index.ts` (1035 LOC — outlier), `src/features/background-command/pty/pty-manager.ts`

**src/hooks/:**
- Purpose: Read-side CQRS — observe events, shape responses, guard decisions
- Contains: lifecycle/, guards/, transforms/, observers/, composition/
- Key files: `src/hooks/lifecycle/core-hooks.ts`, `src/hooks/transforms/tool-before-guard.ts`, `src/hooks/composition/cqrs-boundary.ts`

**src/routing/:**

- Contains: session-entry/, behavioral-profile/, command-engine/
- Key files: `src/routing/session-entry/intake-gate.ts`, `src/routing/behavioral-profile/resolve-behavioral-profile.ts`

**src/schema-kernel/:**
- Purpose: Zod v4 validation contracts for all meta-concepts

- Key files: `src/schema-kernel/index.ts` (377 LOC), `src/schema-kernel/hivemind-configs.schema.ts` (446 LOC), `src/schema-kernel/permission.schema.ts`

**src/shared/:**

**src/task-management/:**
- Purpose: Durable state, lifecycle state machine, journal, trajectory, recovery

**src/tools/:**

**Entry Points:**
- `src/plugin.ts`: HarnessControlPlane — OpenCode plugin composition root
- `src/index.ts`: Public API re-exports (hivemind package)
- `bin/hivemind.cjs`: CLI entry script

**Configuration:**
- `package.json`: npm manifest, exports, scripts, dependencies
- `tsconfig.json`: TypeScript strict mode, ES2022 target, NodeNext modules
- `vitest.config.ts`: Test configuration

**Core Logic:**
- `src/coordination/delegation/manager.ts`: DelegationManager — WaiterModel dispatch
- `src/coordination/concurrency/queue.ts`: DelegationConcurrencyQueue — per-key gating
- `src/task-management/lifecycle/index.ts`: HarnessLifecycleManager — state machine
- `src/task-management/continuity/index.ts`: ContinuityStore — dual-layer state

- `src/shared/types.ts`: Canonical type contracts (TaskStatus, HarnessStatus, RuntimePolicy)

**Testing:**
- `tests/lib/`: Library module tests (mirrors src/ subdirectories)
- `tests/tools/`: Tool-focused unit tests

## Naming Conventions

**Files:**
- Source files: `kebab-case.ts` (e.g., `delegate-task.ts`, `tool-before-guard.ts`)
- Schema files: `kebab-case.schema.ts` (e.g., `permission.schema.ts`, `bootstrap.schema.ts`)
- Test files: `kebab-case.test.ts` mirroring source (e.g., `delegate-task.test.ts`)
- Barrel exports: `index.ts` in each subdirectory
- Type files: `types.ts` co-located with module (e.g., `src/coordination/delegation/types.ts`)

**Directories:**
- Feature modules: `src/{layer}/{domain}/` (e.g., `src/coordination/delegation/`)
- Sub-features: `src/features/{feature-name}/` with internal subdirectories (e.g., `src/features/session-tracker/capture/`, `src/features/session-tracker/persistence/`)
- Sector guidance: `AGENTS.md` in each source directory

**Classes:**
- Manager classes: `PascalCaseManager` (e.g., `DelegationManager`, `HarnessLifecycleManager`)
- Factory functions: `create{Name}Tool()`, `create{Name}Hooks()` (e.g., `createDelegateTaskTool()`, `createCoreHooks()`)
- Schema objects: `{Name}Schema` (e.g., `AgentFrontmatterSchema`, `PermissionRulesetSchema`)

## Where to Add New Code

**New Tool:**
- Implementation: `src/tools/{tool-name}/index.ts` (factory function) or `src/tools/{tool-name}.ts`
- Schema: `src/schema-kernel/{tool-name}.schema.ts` (if input validation needed)
- Registration: Add to tool map in `src/plugin.ts:216-239`
- Tests: `tests/tools/{tool-name}.test.ts`

**New Feature:**
- Implementation: `src/features/{feature-name}/` (directory with index.ts, types.ts, submodules)
- If PTY-related: `src/features/background-command/pty/`
- If session-related: `src/features/session-tracker/`
- Tests: `tests/lib/features/{feature-name}/`

**New Coordination Module:**
- Implementation: `src/coordination/{domain}/` (manager.ts, types.ts, handler.ts)
- Tests: `tests/lib/coordination/{domain}/`

**New Hook:**
- Lifecycle: `src/hooks/lifecycle/{name}-hooks.ts`
- Transform: `src/hooks/transforms/{name}.ts`
- Guard: `src/hooks/guards/{name}-hooks.ts`
- Observer: `src/hooks/observers/{name}-consumer.ts`
- Tests: `tests/hooks/{name}.test.ts`

**New Schema:**
- Definition: `src/schema-kernel/{name}.schema.ts`
- Barrel export: Add to `src/schema-kernel/index.ts`
- Tests: `tests/schema-kernel/{name}.test.ts`

**New Shared Utility:**
- Implementation: `src/shared/{name}.ts` (must remain leaf — no imports from deeper layers)
- Types: `src/shared/types.ts` (if cross-cutting type)
- Tests: `tests/lib/shared/{name}.test.ts`

**New Config Workflow:**
- Implementation: `src/config/workflow/{name}.ts`
- Schema: `src/schema-kernel/{name}.schema.ts`
- Tests: `tests/lib/config/`

**New Routing Module:**
- Implementation: `src/routing/{module-name}/` (index.ts, types.ts, specific modules)
- Tests: `tests/routing/{module-name}/`

## Special Directories

**.opencode/:**
- Purpose: Soft Meta-Concepts — agents, commands, skills, rules, permissions
- Generated: No — authored in `.hivefiver-meta-builder/`, reflected via symlink
- Committed: Yes — runtime copies of skills/agents/commands
- **NEVER** stores internal state, business logic, or build artifacts

**.hivemind/:**
- Purpose: Internal deep module state — session journals, delegation records, event tracker, lineage
- Generated: Yes — written at runtime by continuity store, session tracker, journal
- Committed: No — gitignored
- Canonical state root per Q6 decision

**.hivefiver-meta-builder/:**
- Purpose: Meta-authoring lab — source-of-truth for skills, agents, commands before reflection
- Generated: No — hand-authored
- Committed: Yes
- Contains: skills-lab/, agents-lab/, commands-lab/

**.planning/:**
- Purpose: Governance — requirements, roadmaps, architecture maps, phase authorization
- Generated: No — authored by planning workflows
- Committed: Yes
- Contains: codebase/ (this file), architecture/, phases/, research/

**dist/:**
- Purpose: Compiled TypeScript output
- Generated: Yes — `npm run build`
- Committed: No — gitignored
- Entry points: `dist/index.js`, `dist/plugin.js`, `dist/cli/index.js`

**plans/:**
- Purpose: Phase plans, executable plans, archives
- Generated: Yes — by planning workflows
- Committed: Yes
