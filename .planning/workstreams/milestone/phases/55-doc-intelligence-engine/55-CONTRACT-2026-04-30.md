# Phase 55 Contract: Doc Intelligence Engine

## Verdict

Phase 55 is complete as a **planning contract**. It defines what future implementation must build, but it does not add dependencies, code, or runtime behavior.

## Boundary Contract

Doc intelligence is a read-only document layer. It reads source documents, extracts structured views, and returns tool responses. It must not write `.hivemind/` state, mutate `.opencode/` primitives, or replace prompt analysis tools.

## Component Contracts

| Component | Responsibility | Explicit Non-Authority |
|---|---|---|
| Parser | Extract markdown frontmatter, outline nodes, and heading hierarchy. | Does not decide workflow state or mutate documents. |
| Chunker | Produce stable ordered chunks with configurable token budget metadata. | Does not create a search index or persistent cache in this phase. |
| Router | Dispatch file, directory, read, chunk, and search requests to parser/chunker operations. | Does not bypass tool response envelopes or path boundaries. |
| Types | Define outline nodes, document metadata, chunks, and search result shape. | Does not import runtime state managers. |
| Tool | Expose `skim`, `skim_directory`, `read`, `chunk`, and `search`. | Does not replace `prompt-skim` or `prompt-analyze`. |
