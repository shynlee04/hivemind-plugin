# Phase 16.2: AUDIT AMENDMENT — PTY Non-Functional on Node.js

**Audit Date:** 2026-04-30
**Previous Status:** REMEDIATED (CR-01, CR-03 resolved)
**Amended Status:** **NOT REMEDIATED — CRITICAL FAILURE**

---

## Audit Override

The 2026-04-30 comprehensive audit of `src/` (main repo) and `.worktrees/harness-experiment/src/` (worktree) reveals that **PTY is 100% non-functional on Node.js**. The previous Phase 16.2 remediation claims are **false** because:

1. `pty-manager.ts:1` imports `bun-pty` at module level — this is a **Bun-only native module**
2. The "lazy-load" in `createPtyManagerIfSupported()` (`pty-runtime.ts:13-21`) does **not** prevent the crash because the dynamic import resolves the module graph, which hits the failed `bun-pty` import immediately
3. On Node.js, `args.ptyManager` is `null`, so all non-`run` actions in `run-background-command.ts` return error: "PTY not available"
4. PTY OS processes **cannot survive harness restarts** — every PTY delegation fails recovery unconditionally (`command-delegation.ts:129` logs "[Harness] PTY session not found on recovery")

**The "29 consecutive child session not found on recovery" errors are explained:** `command-delegation.ts:129` logs this exact string. This is not a bug in recovery logic — it is the expected behavior of PTY processes that die when the parent process restarts.

---

## Amended Requirements (replacing 16.2-SPEC-2026-04-22.md Section 1)

### R-PTY-01-AMENDED: Remove or replace `bun-pty` dependency

**Previous:** Command delegations execute through PTY as primary path.
**Amended:** PTY execution is **non-functional on Node.js**. Choose one:

- **Option A (Recommended):** Remove PTY entirely. Use headless `child_process.spawn` as the only execution path. Delete `src/lib/pty/`, remove `bun-pty` from dependencies.
- **Option B:** Replace `bun-pty` with `node-pty` (cross-platform, works on Node.js). Move import from module-level to inside dynamic import block.
- **Option C (Not recommended):** Keep `bun-pty` as optional dependency with graceful degradation. Still fails on Node.js at import time unless module-level import is removed.

**Acceptance Criterion:** `npm test` passes on Node.js with no `bun-pty` installed. No module resolution errors. `run-background-command` tool works in headless mode.

**Priority:** P0 CRITICAL

**Affected Files:** `src/lib/pty/pty-manager.ts`, `src/lib/pty/pty-runtime.ts`, `src/lib/command-delegation.ts`, `src/tools/run-background-command.ts`, `package.json`

---

### R-PTY-02-AMENDED: Simplify `run-background-command` to headless-only

**Previous:** PTY process output is captured and returned as delegation result.
**Amended:** If PTY is removed, simplify the tool:

- Remove `output`, `input`, `list`, `terminate` actions (these required PTY)
- Keep only `run` action with `child_process.spawn`
- Capture stdout/stderr directly via Node.js streams
- Return captured output in the tool result (no PTY buffer needed)

**Acceptance Criterion:** `run-background-command` with action `run` executes a command and returns stdout + stderr + exit code. No PTY dependency. No "PTY not available" errors.

**Priority:** P0 CRITICAL

**Affected Files:** `src/tools/run-background-command.ts`

---

### R-PTY-03-AMENDED: Document PTY non-resumability

**Previous:** PTY process lifecycle tied to delegation status transitions.
**Amended:** Add explicit documentation that PTY processes (if kept) cannot survive harness restarts. Recovery code for PTY delegations must return `non-resumable-after-restart` as the terminal reason, not attempt to reconnect.

**Acceptance Criterion:** No "PTY session not found on recovery" errors after restart. Recovery handler returns truthful status: `terminalState: "error"`, `terminalReason: "non-resumable-after-restart"`.

**Priority:** P1 HIGH

**Affected Files:** `src/lib/command-delegation.ts:126-143`

---

### R-PTY-04-AMENDED: SDK delegations remain SDK-only (unchanged)

**Status:** UNCHANGED — This requirement is correct. SDK delegations use `client.session.prompt()`. PTY is not used for SDK delegations.
