```typescript
export function buildDelegationQueueKey(args: {
  model?: string

  category?: string
}): string
```

Current live semantics are precedence-based, not composite:
- if `model` is present -> `model:${model}`
- else if `agent` and `category` are present -> `agent:${agent}:category:${category}`
- else if `agent` is present -> `agent:${agent}`
- else if `category` is present -> `category:${category}`
- else -> `default`

Target for this plan: extend this same canonical builder in `src/lib/concurrency.ts` so it also understands provider-aware model lanes (e.g. `provider:${provider}:model:${model}` when both are present), while preserving the existing fallback precedence for callers that only have agent/category data. Do **not** create a second policy owner in `src/lib/spawner/concurrency-key.ts`.
