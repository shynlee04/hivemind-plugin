## Intent
Port the agent work contract concepts from product-detox. Agent work contracts define bounded work scope, expected evidence, completion criteria, and exportable handoff artifacts for delegated work. The compaction preservation path extracts briefing, summary, and anchors before context loss, then reinjects those artifacts so resumed agents keep scope and evidence intact. This phase creates the work-contract layer that Agent Synthesis AS-8 depends on.

## Scope
- Agent work scope contract — bounded task, allowed surfaces, dependencies, and non-goals
- Evidence contract — required proof, acceptable evidence levels, and explicit blocked-state reporting
- Compaction preservation — briefing extraction, summary extraction, anchor extraction, and reinjection contract
- Tool: `hivemind_agent_work_create` — creates work contract artifacts for delegated or resumed work
- Tool: `hivemind_agent_work_export` — exports contract, evidence, summary, and anchors for handoff
- Integration with Phase 57 pressure/control-plane gates before work contract creation or export
- Integration with Phase 56 trajectory/session v3 so work contracts can attach to session lineage and recovery records

## Non-Goals
- Do not rewrite the existing `delegate-task` tool — work contracts wrap and constrain delegated work, not replace delegation dispatch
- Do not implement Agent Synthesis AS-8 — this phase provides the contract substrate AS-8 consumes
- Do not build a generic memory system — compaction preservation is scoped to briefing, summary, anchors, and reinjection
- Do not change source-code verification gates — evidence contracts describe expected proof and handoff shape

## Assumptions
- Agent work contracts are durable planning/runtime artifacts, not transient prompt-only instructions
- Evidence contracts can use the same evidence hierarchy language already used by quality gates without importing gate implementation
- Compaction preservation must be explicit and exportable so it survives session restarts and handoffs
- Work contract creation should be pressure-aware and may be blocked by Phase 57 control-plane decisions

## Success Criteria
1. A planner can distinguish work contracts (scope/evidence/handoff) from delegation records (runtime dispatch/status)
2. The `hivemind_agent_work_create` and `hivemind_agent_work_export` tools are contract-defined with input/output schemas
3. Compaction preservation defines briefing, summary, anchor extraction, and reinjection responsibilities
4. Evidence requirements and blocked-state reporting are explicit enough for downstream verification
5. Downstream dependency on Agent Synthesis AS-8 is documented
