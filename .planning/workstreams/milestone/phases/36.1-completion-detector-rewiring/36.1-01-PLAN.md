# Phase 36.1 Plan — CompletionDetector Re-Wiring

## Objective

Replace `SdkDelegationHandler`'s adaptive-polling finalization with `CompletionDetector`-driven finalization. Share one detector instance per delegation. Consolidate status enums.

## Tasks

### PH36-04 — Replace adaptive polling

1. **RED:** Add a `tests/lib/sdk-delegation.test.ts` case that asserts: given a delegation with `executionMode: "sdk"`, when the SDK fires `session.idle` AND message count is stable for `stabilityTimeoutMs`, finalization happens via `CompletionDetector` callback (not via stable-poll counter). Should fail under current code.
2. **RED:** Add a case asserting `STABLE_POLLS_REQUIRED` does NOT gate finalization (i.e., a delegation with `stablePollCount = 0` but a fired `session.idle` and stable message count must still finalize).
3. **GREEN:** Wire `SdkDelegationHandler` to receive a `CompletionDetector` (constructor injected). Replace the `hasMinStabilityTime && hasEnoughStablePolls` decision with `completionDetector.feed("session.idle"|"session.error"|"session.deleted", sessionID)` + `completionDetector.feedMessageCount(currentCount)` and a callback that calls `finalizeSdkDelegation(...)`.
4. **REFACTOR:** Delete `calculateAdaptiveInterval()`, `MIN_IDLE_TIME_MS`, `MIN_STABILITY_TIME_MS`, `STABLE_POLLS_REQUIRED`, and any constants/imports they depended on. Keep `DEFAULT_STALE_TIMEOUT_MS` only if it survives as the detector's stability timeout source.

### PH36-05 — Single detector instance

1. **RED:** Add a test that asserts there is exactly one `CompletionDetector` instance per delegation (use a factory mock or reflect on the manager).
2. **GREEN:** `LifecycleManager` either (a) owns a per-delegation `CompletionDetector` map and hands the instance to `SdkDelegationHandler` at dispatch time, or (b) removes its own detector and lets `SdkDelegationHandler` own it. Pick whichever yields cleaner ownership.
3. **REFACTOR:** Remove the orphaned `feed()` / `cancel()` calls in `lifecycle-manager.ts:124,166` if they are no longer the source of truth.

### PH36-06 — Status enum consolidation

1. **RED:** Add a type-level test (`expectTypeOf` or equivalent) asserting that `DelegationStatus` is the only runtime status type referenced by `delegation-manager.ts` and `sdk-delegation.ts`.
2. **GREEN:** Find all references to `HarnessStatus` and `DelegationPacketStatus`. Replace with `DelegationStatus` (or `SessionLifecyclePhase` for continuity records).
3. **REFACTOR:** Delete `HarnessStatus` and `DelegationPacketStatus` from `src/lib/types.ts`.
