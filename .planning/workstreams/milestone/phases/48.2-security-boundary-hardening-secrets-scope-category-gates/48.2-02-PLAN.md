<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller session → delegation-status tool | Caller-controlled tool request may name arbitrary delegation IDs. |
| persisted delegation store → public tool output | Durable records may outlive in-memory ownership maps. |
| caller/generated file path → harness write boundary | Caller or derived paths may attempt traversal or writes outside project/runtime roots. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-48.2-02-01 | Information Disclosure | `delegation-status` specific lookup | mitigate | Require caller session and deny if `canSessionAccessDelegation()` is false. |
| T-48.2-02-02 | Information Disclosure | `delegation-status` list mode | mitigate | Filter active and persisted records through the same visibility predicate. |
| T-48.2-02-03 | Repudiation | deny responses | mitigate | Return explicit `[Harness] Access denied...owner lineage` messages for auditability. |
| T-48.2-02-04 | Tampering | harness write boundaries | mitigate | Use shared path-scope assertions before session patch, primitive generation, event-tracker artifact, and continuity state writes. |
</threat_model>
