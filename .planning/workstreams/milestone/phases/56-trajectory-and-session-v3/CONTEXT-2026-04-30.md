# Phase 56 Context: Trajectory & Session v3
**Date:** 2026-04-30
**Workstream:** milestone

**Status:** research-locked / pre-planning
**Reader:** researcher, planner, verifier, or human reviewer preparing Phase 56 as a product-detox concept migration phase.
**Post-read action:** derive bounded implementation scope and evidence-first planning for migrating trajectory engine and session v3 schema from product-detox into harness-experiment.

## Intent
Port the trajectory engine and session v3 schema from product-detox. The trajectory ledger provides cross-session truth anchoring with checkpoints, evidence-anchored events, and recovery logs. The session v3 schema adds semantic IDs, lineage, purpose class, auto-generated TOC, key findings, and resumable flags. This supports future/reopened Phase 52 journal lineage (E52-03) and recovery proof (E52-05) gap-closure planning without reopening or closing Phase 52 by itself.

## Scope
- `lib/trajectory/ledger.ts` — cross-session truth-anchored ledger with checkpoint and evidence-anchored events
- `lib/trajectory/types.ts` — trajectory event types, checkpoint schema, recovery log schema
- `lib/trajectory/store-operations.ts` — CRUD operations for trajectory events and checkpoint records
- Extend `event-tracker/types.ts` with session v3 schema (semantic IDs, lineage, purpose class, auto-generated TOC, key findings, resumable flags)
- `lib/session-hierarchy/tree-writer.ts` — parent-child session tree writer
- Tools: `hivemind_trajectory` — 6 actions: `inspect`, `traverse`, `attach`, `checkpoint`, `event`, `close`
- Integration with Phase 55 doc intelligence (trajectory events reference doc intelligence for evidence)

## Non-Goals
- Do not replace existing `continuity.ts` or `delegation-persistence.ts` — trajectory is complementary
- Do not rebuild the existing session journal (journal is append-only timeline; trajectory adds truth anchoring)
- Do not modify `session-journal-export` tool (journal export is separate from trajectory inspection)
- Do not implement the full time-machine from Phase 41 (trajectory is a prerequisite for time-machine, not the time-machine itself)

- Session v3 schema can extend existing event-tracker types without breaking existing journal consumers
