# Phase 48.4 Context: Production Evidence & Coverage Recovery

**Inserted:** 2026-04-27
**Depends on:** Phase 48.3 (complete)
**Research completed:** 2026-04-27

## Goal

Recover production-grade verification evidence and coverage for the remediated runtime path, replacing degraded/mock-heavy proof with reproducible live checks. Phase 48.3 completed integration of lifecycle manager, delegation manager, and continuity store. This phase closes the evidence gap: 8 critical modules (1,601 LOC) have zero dedicated tests, 17 test files are mock-heavy existence assertions, and coverage infrastructure (`@vitest/coverage-v8`) is missing entirely.

## Success Criteria

1. **Live runtime evidence** proves tool registration, hook execution, and delegated child completion without false lifecycle states.
2. **Coverage reflects meaningful behavior checks** rather than mock-only existence assertions.
3. **Remaining gaps classified** as blocker, deferred, or external-environment limitations.
4. **All 8 untested critical modules** have dedicated test files with behavioral assertions (runtime.ts, notification-handler.ts, event-tracker/writer.ts, create-core-hooks.ts, create-session-hooks.ts, create-tool-guard-hooks.ts, agent-primitive-policy.ts, workspace-runtime-policy.ts).
5. **Coverage thresholds enforced** at 70% statements, 60% branches, 70% functions, 70% lines via `vitest.config.ts`.
6. **Mock-heavy tests upgraded** with stateful fake client pattern for stability polling, grace period cleanup, and recovery flow verification.
7. **REM-RUNTIME-04 and REM-RUNTIME-05** deferred and documented as external-environment limitations with manual verification procedures.

## Research Findings

Phase 48.4 research (see `48.4-RESEARCH-2026-04-27.md`) audited the entire test evidence quality across 61 test files and 982 passing tests. Key findings:

- **Evidence quality tiers:** 41 test files exercise real code paths (REAL tier), ~5 are mixed, 12 are mock-heavy, and 8 critical modules (1,601 LOC) have zero dedicated tests.
- **Coverage infrastructure gap:** `vitest.config.ts` lacks coverage provider configuration. `npm run test:coverage` fails because `@vitest/coverage-v8` is not installed.
- **Two deferred gaps:** REM-RUNTIME-04 (dynamic tool execution proof) and REM-RUNTIME-05 (successful real child delegation) require a live OpenCode runtime — impossible in CI. Classified as external-environment limitations with documented manual verification procedures.
- **Recommended approach:** 4-wave plan — install coverage infrastructure, add new behavioral tests for untested modules, upgrade mock-heavy tests with stateful fake client patterns, classify gaps.
- **Stateful fake client pattern** recommended over `vi.fn()` for behavioral tests. A `createFakeClient()` factory that maintains internal session state (`Map<string, SessionState>`) produces realistic responses and catches real integration bugs.
- **Coverage target:** 70% initial threshold (statements, functions, lines), 60% branches, with types-only files excluded.

## Known Gaps

The following requirements from Phase 48 cannot be fully automated and are deferred to manual verification:

| Gap | Requirement | Reason | Mitigation |
|-----|------------|--------|------------|
| REM-RUNTIME-04 | Dynamic tool execution proof | Requires live OpenCode runtime to prove a tool executes with correct contracts end-to-end | Manual verification procedure documented in `48.4-GAP-CLASSIFICATION.md` (Wave 3) |
| REM-RUNTIME-05 | Successful real child delegation | Requires live OpenCode runtime to spawn a real child session with non-empty provider completion | Manual verification procedure documented in `48.4-GAP-CLASSIFICATION.md` (Wave 3) |

Both are classified as **external-environment limitations** — the tests cannot run in CI because they require a running OpenCode instance with provider connectivity. The fake client pattern provides partial confidence: tool factories and hook factories are tested with realistic inputs and stateful fakes. Only the final SDK→OpenCode boundary remains untestable without a live runtime.
