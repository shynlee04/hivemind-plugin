## Amended Requirements

### REM-RUNTIME-04-AMENDED: Add integration tests for delegation flow

**New Requirement:** Create comprehensive tests for the worktree's delegation subsystem. Target: 50+ tests covering critical paths.

**Required Test Files:**

1. **`tests/lib/delegation-manager.test.ts`**
   - Dispatch: verify SDK vs command routing, agent validation, category gates
   - Concurrency: verify keyed semaphore, queue limits
   - Recovery: verify `recoverPending()` reads from persistence, handles stale records
   - Terminal notification: verify parent receives structured JSON on completion
   - Status transitions: verify `dispatched` → `running` → `completed`/`error`/`timeout`

2. **`tests/lib/sdk-delegation.test.ts`**
   - Event-driven completion: verify `session.idle` triggers completion (after Phase 36 fix)
   - Message stability: verify `feedMessageCount()` + stability timer
   - Result harvesting: verify last assistant message extracted on completion
   - Stale timeout: verify timeout after idle period
   - (DELETE after Phase 36 fix: adaptive polling tests, `calculateAdaptiveInterval` tests)

3. **`tests/lib/command-delegation.test.ts`**
   - Headless execution: verify `child_process.spawn` works, returns exit code
   - Output capture: verify stdout/stderr captured in `delegation.result`
   - PTY non-resumability: verify recovery returns `non-resumable-after-restart` (or remove PTY)
   - Fallback: verify headless used when PTY unavailable

4. **`tests/tools/delegate-task.test.ts`**
   - Parameter validation: verify Zod schema rejects invalid inputs
   - Background mode: verify `run_in_background: true` triggers async dispatch (after Phase 46 fix)
   - Sync mode: verify `run_in_background: false` triggers sync dispatch
   - Error handling: verify clear errors for invalid agent, missing task

5. **`tests/tools/run-background-command.test.ts`**
   - `run` action: verify command execution, output capture
   - Headless-only: verify no PTY dependency (after Phase 16.2 fix)
   - Output truncation: verify headless output capped with metadata

6. **`tests/lib/delegation-persistence.test.ts`**
   - JSON serialization: verify round-trip persistence
   - Normalization: verify older records upgraded on read
   - Grace period: verify `gracePeriodExpiresAt` persisted (after Phase 16.2 CR-03 fix)

**Acceptance Criterion:**
- Test suite passes with 50+ new tests
- Coverage reports for `src/lib/delegation-manager.ts`, `src/lib/sdk-delegation.ts`, `src/lib/command-delegation.ts` > 80%
- No mocked internal business logic (mock SDK transport only, per D-21)
