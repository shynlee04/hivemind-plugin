## Objective

Produce a durable grounding map that tells future agents which stack/research skill to load for OpenCode harness work, what evidence each skill can provide, and which verification gate must close the workflow.

2. Map each skill to the concrete harness workflow it supports.

4. Update roadmap/state so Phase 52 can consume the grounding map without rediscovering skill routing.
