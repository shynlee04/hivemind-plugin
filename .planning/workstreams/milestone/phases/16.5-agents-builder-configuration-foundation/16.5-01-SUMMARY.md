# Phase 16.5 Plan 01: Cross-Primitive Validator Summary

Pure validation module detecting misconfigurations across all OpenCode primitives before config load.

## What Was Built

- `src/lib/cross-primitive-validator.ts` (274 LOC) — cross-primitive conflict detection engine
- `tests/lib/cross-primitive-validator.test.ts` (205 LOC) — 11 unit tests covering all 6 categories

## Exported API

- `validateCrossPrimitive(primitives: PrimitiveMap): ValidationReport`
- `PrimitiveMap`, `ValidationReport`, `ValidationIssue`, `ValidationSeverity`, `ValidationCategory`

## 6 Conflict Detection Categories

| Category | Severity | Detector |
|----------|----------|----------|
| agent-command-binding | block | Missing agent referenced by command |
| permission-deadlock | warn | Agent denies tool that command needs |
| role-overlap | warn | Primary agents with >50% keyword overlap |
| missing-skill-dependency | warn | Agent body references non-existent skill |
| mcp-server-gap | warn | Tool references unconfigured MCP server |
| rule-file-gap | block | Config instructions contain invalid paths |
