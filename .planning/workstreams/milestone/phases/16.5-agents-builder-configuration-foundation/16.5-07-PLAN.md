<objective>
Close three gaps identified in 16.5-GAP-ANALYSIS.md:

1. **BLOCKER-2: Zero Framework-Aware Stacking Logic** — No framework metadata parsing exists. No boundary detection. The skill is literally named `hivefiver-agent-config` and references `hivefiver-orchestrator`, making it framework-specific rather than framework-compatible. Cannot safely coexist with GSD's `.planning/` directory, BMAD's `bmad.yaml`, Speckit's configs, or any other framework.

2. **CRITICAL-4: Missing Runtime Validation** — Only static structural validation exists. No runtime or topological validation. Users can create configurations that pass validation but fail at runtime due to loading order issues, circular dependencies, or pipeline misalignment.

3. **CRITICAL-7: No Post-Restart Validation** — No restart simulation. No validation that compiled `.md` files are actually discoverable by OpenCode. Configurations may compile successfully but fail to load at runtime.

Purpose: Make the harness framework-agnostic and add runtime validation capabilities.

Output: Framework detector, runtime validator, validate-restart tool, renamed skill.
</objective>

     valid: boolean

   interface InheritanceChainResult {
     valid: boolean

   }
