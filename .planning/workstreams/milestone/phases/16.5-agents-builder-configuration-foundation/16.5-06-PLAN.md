<objective>
Close two BLOCKER gaps identified in 16.5-GAP-ANALYSIS.md:

1. **BLOCKER-3: Schema Rigid `.strict()` Rejects Future Fields** — All 7 schemas use `.strict()` or fixed enums. If OpenCode adds a new frontmatter field, permission key, or config level, the harness will reject valid configs. This is a compatibility time-bomb.

2. **BLOCKER-1: No Builtin/Existing Object Discovery** — The system can only CREATE new primitives; it never reads existing ones. The `configure-primitive` tool writes files but cannot read them. The workflow skill mentions "scanning" in Turn 0/1 but has no actual file-system scanning implementation.

Purpose: Make the harness resilient to OpenCode updates and enable reading existing configurations from disk.

Output: Updated schemas with resilient validation, new primitive-loader.ts module, enhanced configure-primitive tool with read operations.
</objective>
