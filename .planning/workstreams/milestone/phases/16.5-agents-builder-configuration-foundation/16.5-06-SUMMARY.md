## Decisions Made

- Used `.strip()` for Lenient variants to match the existing codebase pattern (pre-existing schemas already used `.strip()`, not `.passthrough()`)
- Soften string-based enums (PermissionKey, ConfigPrecedenceLevel) rather than creating separate strict/lenient value schemas, since the values are OpenCode-defined and can change
- primitive-loader.ts uses only Lenient schemas for reads, ensuring maximum compatibility with real-world configs
- Cross-reference validation treats missing opencode.json sections as empty sets, so all disk files without corresponding config entries are flagged as orphaned
