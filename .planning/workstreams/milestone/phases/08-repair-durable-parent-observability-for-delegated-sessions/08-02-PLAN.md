<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| child lifecycle observation → parent-visible notification | async session outcomes cross from child execution into parent-facing status |
| continuity lifecycle state → pending notification delivery | durable truth is replayed into later parent delivery |
| builtin-process/background manager → lifecycle reconciliation | owned-process results are converted into harness lifecycle status |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05 | Repudiation | parent notification flow | mitigate | Persist fallback pending notifications with status, summary, output link, and timestamps when direct notification delivery fails. |
| T-08-06 | Tampering | lifecycle reconciliation | mitigate | Keep parent-visible status derived from continuity-backed lifecycle transitions validated by `isValidTransition`; do not allow observer events to overwrite terminal truth arbitrarily. |
| T-08-07 | Denial of Service | background observer | mitigate | Cover deleted/retry/timeout paths so parents receive deterministic failed status instead of silent hangs. |
| T-08-08 | Information Disclosure | notification payloads | accept | Parent notifications expose only delegated task description, agent, status, summary, and session link already available to the owning parent session. |
</threat_model>
