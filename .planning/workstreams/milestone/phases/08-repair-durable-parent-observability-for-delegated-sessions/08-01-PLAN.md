## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| parent runtime state → child delegation metadata | trusted parent-derived policy crosses into new child session state |
| continuity file → hydrated in-memory delegation meta | persisted JSON is reloaded into live enforcement state |
| delegation metadata → tool guard enforcement | session policy drives budget and circuit-breaker behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | `src/lib/continuity-normalizers.ts` | mitigate | Preserve only typed `SessionPolicyOverride` fields during normalization; ignore malformed values instead of hydrating them. |
| T-08-02 | Elevation of Privilege | `src/tools/delegate-task.ts` | mitigate | Do not accept `runtimePolicyOverride` from public tool args or prompt text; derive only from trusted parent runtime metadata. |
| T-08-03 | Denial of Service | `src/hooks/create-tool-guard-hooks.ts` | mitigate | Add regression proving lowered per-session budgets still enforce after reload so runaway sessions cannot bypass limits through missing metadata. |
| T-08-04 | Repudiation | continuity + tests | mitigate | Add explicit write/persist/reload assertions so audit evidence shows exactly where override values were produced and preserved. |
</threat_model>
