For each of 24 skills, assess:

### Zod Feasibility
Can the skill's frontmatter + config be expressed as a Zod schema?
- If skill has simple frontmatter (name, description, layer) → READY
- If skill has complex conditional logic in body → PARTIAL
- If skill embeds shell scripts or external tool calls → COMPLEX
- If skill body is the entire value (no separable config) → NOT YET

### SDK Surface
What typed API would this skill expose?
- Tool skills → expose tool config schema
- Reference skills → expose reference loading protocol
- Process skills → expose pipeline stage definitions
- Meta skills → expose skill composition protocol

### Migration Blocker
What prevents migration TODAY?
- "No evals — cannot validate migration correctness"
- "Body contains embedded bash — needs extraction"
- "References not typed — need schema definitions"
- "No stacked eval — cannot verify runtime behavior"
- "Grade < B — must reach target grade first"
- "NONE — ready for migration pipeline"
