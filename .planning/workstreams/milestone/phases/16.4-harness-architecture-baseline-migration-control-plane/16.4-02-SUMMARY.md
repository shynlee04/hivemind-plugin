# Plan 16.4-02 Summary: Architecture Baseline

## What was delivered

`16.4-ARCHITECTURE-BASELINE.md` — a 180-section evidence-backed architecture baseline covering:

1. **Mutation Authority Matrix** — 9 surfaces with owner, allowed callers, persistence target, idempotency, recovery, evidence, decision trace.
2. **State Ownership Model** — Roles for continuity records, delegation records, pending notifications, journals, trajectory, task graph, agent-work-contract, .hivemind categories, vector/graph memory, sidecar read models, current .opencode/state.
3. **Runtime Classification Matrix** — 5 runtime classes (session initiation, mid-flow intervention, post-completion, delegated subtask, background command) with full field set.
4. **Lifecycle Ownership Table** — Primary owner per transition, pass/fail rule for duplicate terminal-state writers.
5. **Current Component Map + Target Code-Tree** — Supersedes Phase 11 with file classification move map.
6. **Hiveminder/Hivefiver Boundary** — Neither is a soft harness; .md files are transition-stage artifacts.
7. **OpenCode Platform Conformance** — Evidence-backed rules for plugins, tools, hooks, permissions, SDK.
8. **Human/Agent Collaboration Model** — Required views for active tasks, delegated tasks, hierarchy, blockers, deferred ideas.
9. **Memory and Retrieval Taxonomy** — 8 categories with purpose, consumer, metadata, access strategy.
10. **Sidecar Compatibility Boundaries** — Read/query/render/request allowed; must not become runtime authority.
