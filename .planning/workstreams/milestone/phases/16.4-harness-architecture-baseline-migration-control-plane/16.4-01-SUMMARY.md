# Plan 16.4-01 Summary: Validation Control Surface

## What was delivered

Two guard artifacts and the decision-validation register:

1. **`guards/README.md`** — Guard manifest declaring `architecture-facts.mjs` as `fact-reporting` classification with input/output/failure/SPEC fields.
2. **`guards/architecture-facts.mjs`** — Executable Node script with 10 check flags (`--check-register`, `--check-mutation-authority`, `--check-state-taxonomy`, `--check-lifecycle-owners`, `--check-move-map`, `--check-platform-evidence`, `--check-migration-gates`, `--check-first-big-win`, `--check-source-coverage`, `--check-guard-manifest`). Emits JSON, exits 0 on pass, exit 1 on failure, never writes files.
3. **`16.4-DECISION-REGISTER.md`** — 12-row register (REG-01 through REG-12) with verdict vocabulary and reuse rules. Classifies 7 stale/rejected claims (CQRS dogma, hook mutation prohibition, Hiveminder/Hivefiver flattening, Phase 11 authority, nonexistent file claims, notification-handler dead-code, product-detox wholesale copy) and 4 valid claims (plugin.ts composition root, src/opencode split, current state root, .hivemind preferred root).

## Verification

- `node .../guards/architecture-facts.mjs --check-guard-manifest --check-register` passes
- `npm run typecheck` passes
- Guard checks are consumed by Plans 02, 03, and 04

## Threat mitigations

T-16.4-01 (tampering register): evidence-backed rows. T-16.4-02 (repudiation guard): JSON + non-zero exit. T-16.4-04 (stale doc poisoning): reuse-safe markers.
