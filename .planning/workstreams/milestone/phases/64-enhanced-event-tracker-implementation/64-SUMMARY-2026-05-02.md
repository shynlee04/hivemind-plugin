## Architecture Decisions

1. **Separate ClassifiedEventType from JourneyEventType** — The existing `JourneyEventType` (5 values: session_start, session_updated, session_idle, session_end, session_event) is used by the hook-event pipeline. The new `ClassifiedEventType` (10 values) is a higher-level semantic taxonomy. Keeping them separate avoids breaking changes to existing event-tracker consumers.

2. **In-memory delegation evidence tracker** — `createDelegationEvidenceTracker()` returns a stateful object with `track()`, `query()`, and `latestState()` methods. No disk I/O — callers use dual persistence for durable storage. This follows the project's CQRS pattern (tools are write-side, state is in-memory Maps).

3. **Priority-ordered classification** — The classifier examines events in deterministic priority order: explicit type → error → delegation_created → delegation_returned → tool_invocation → role-based (user/assistant) → unknown. This ensures consistent classification when multiple patterns could match.

4. **Dependency-injected filesystem** — Both dual-persistence and existing artifact-writer use injected filesystem adapters. Tests use in-memory Map-based filesystems, following the established pattern.
