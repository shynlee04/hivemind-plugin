# Phase 52 Plan 04 — Journal/Lineage Export and Primitive Boundary Proof

## Goal

Prove that `session-journal-export` reflects the same acceptance workflow and that `configure-primitive` / `validate-restart` can run without corrupting `.opencode/` and `.hivemind/` boundaries.

## Scope / Out of Scope

**In scope:** same-run journal/lineage export, read-only primitive operations, restart-validator invocation, before/after boundary notes.

**Out of scope:** actual primitive writes, real app restart claim from `validate-restart` alone, release readiness, or source changes.

## Dependencies / Wave

- **Wave:** 4
- **Depends on:** Plan 03 same-run PTY lifecycle
- **Unlocks:** Plan 05 interruption/recovery

- `52-EVIDENCE-MATRIX-2026-04-29.md`

## Acceptance Criteria

- Export output references the same session/workflow identifiers, or records an exact mismatch.
- `configure-primitive` uses only read/list/inspect/dryRun behavior.
- `validate-restart` result is captured and explicitly labeled as validator evidence, not actual restart proof.
- Boundary notes show `.opencode/` remains primitive space and `.hivemind/` remains runtime-state space.

## Verification Evidence Expected

- L1: raw live tool outputs from `session-journal-export`, `configure-primitive`, and `validate-restart`.
- L2: persisted `.hivemind/state` observations for the same workflow when available.
- L5: structural boundary notes, labeled as context.

## Failure / Blocker Handling

- Unrelated export records or relabeled filters mean FAIL/PARTIAL for E52-03.
- Any unintended primitive write means FAIL for E52-04.
- Passing `validate-restart` without same-run live acceptance remains PARTIAL.

## Rollback / Safety Notes

- Do not write or overwrite primitives.
- Do not mutate `.hivemind/state`.
- Preserve raw export mismatches instead of cleaning them up.
