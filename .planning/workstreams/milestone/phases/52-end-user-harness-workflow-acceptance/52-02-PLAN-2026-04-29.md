# Phase 52 Plan 02 — Live Delegation and Status Acceptance

## Goal

Prove that a real parent OpenCode session can dispatch a specialist through `delegate-task` and observe the same delegation through `delegation-status` until terminal state, or capture the exact runtime blocker.

## Scope / Out of Scope

**In scope:** parent-session prompt, live `delegate-task`, live `delegation-status`, parent/child/delegation ID capture, status progression, terminal result/error classification.

**Out of scope:** PTY command lifecycle, journal export, recovery testing, source-code fixes, or direct TypeScript/mock invocation.

## Dependencies / Wave

- **Wave:** 2
- **Depends on:** Plan 01 readiness and capture templates
- **Unlocks:** Plan 03 PTY lifecycle using the same acceptance workflow

## Files and Commands to Use

Create/update only:

- `52-02-DELEGATION-TRANSCRIPT-2026-04-29.md`
- `52-RUNTIME-TRANSCRIPT-2026-04-29.md`
- `52-EVIDENCE-MATRIX-2026-04-29.md`

Live tool actions:

```json
{
  "tool": "delegate-task",
  "args": {
    "agent": "researcher",
    "title": "Phase 52 acceptance delegation smoke",
    "prompt": "You are a delegated specialist for Phase 52 acceptance. Return a concise terminal result proving this child session received and completed the task. Do not modify files.",
    "safetyCeilingMs": 60000
  }
}
```

Then poll:

```json
{ "tool": "delegation-status", "args": { "delegationId": "<captured-delegation-id>" } }
```

## Acceptance Criteria

- `delegate-task` returns a delegation ID from a real parent session.
- `delegation-status` can read the delegation from the same owner lineage.
- Status progression reaches terminal `completed`, `error`, or `timeout` with raw output captured.
- E52-01 in the evidence matrix is classified PASS only if live dispatch + status progression + terminal result are all present.

## Verification Evidence Expected

- L1: raw parent-session tool outputs for `delegate-task` and `delegation-status`.
- L2: `.hivemind/state/delegations.json` observation for the same delegation ID, if available.
- L5: static source references are allowed only as context, not acceptance proof.

## Failure / Blocker Handling

- Missing `context.sessionID` means BLOCKED by runtime context injection.
- Access denied from `delegation-status` means FAIL for lineage/status visibility.
- Stalled child session means PARTIAL only if IDs and intermediate status were captured; otherwise FAIL/BLOCKED based on evidence.

## Rollback / Safety Notes

- Do not switch to direct function calls.
- Do not edit agents or source to force a pass.
- Do not modify Phase 49 artifacts.
