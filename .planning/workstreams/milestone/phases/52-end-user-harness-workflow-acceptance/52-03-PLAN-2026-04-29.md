# Phase 52 Plan 03 — Live `run-background-command` Lifecycle

## Goal

Prove that a user can run, observe, list, and terminate a background PTY command through the production `run-background-command` surface during the same acceptance workflow.

## Scope / Out of Scope

**In scope:** `run`, `output`, `list`, `terminate`, PTY session ID capture, cancellation metadata, discoverability/accessibility note for terminal evidence capture.

**Out of scope:** custom terminal tools, shell-string expansion, source fixes, or treating a missing PTY as pass.

## Dependencies / Wave

- **Wave:** 3
- **Depends on:** Plan 02 live parent workflow
- **Unlocks:** Plan 04 journal/export correlation

## Files and Commands to Use

Create/update only:

- `52-03-PTY-TRANSCRIPT-2026-04-29.md`
- `52-RUNTIME-TRANSCRIPT-2026-04-29.md`
- `52-EVIDENCE-MATRIX-2026-04-29.md`

Live tool action sequence:

```json
{ "tool": "run-background-command", "args": { "action": "run", "command": "bash", "args": ["-lc", "printf 'phase52-ok\\n'; sleep 30"] } }
{ "tool": "run-background-command", "args": { "action": "output", "sessionId": "<pty-session-id>", "offset": 0 } }
{ "tool": "run-background-command", "args": { "action": "list" } }
{ "tool": "run-background-command", "args": { "action": "terminate", "sessionId": "<pty-session-id>" } }
```

## Acceptance Criteria

- `run` returns a caller-visible PTY/session identifier.
- `output` from offset `0` captures expected output or a truthful PTY blocker.
- `list` shows the caller-visible session while active.
- `terminate` returns cancellation/terminal metadata for the same session.
- E52-02 is classified PASS only if the full lifecycle runs live.

## Verification Evidence Expected

- L1: raw tool outputs for all four live actions.
- L2: delegation/PTY record observations where available.
- UX note: if the operator used a view named `accessible-view-terminal`, record it only as contextual evidence-capture UX. No such repo artifact is assumed.

## Failure / Blocker Handling

- `[Harness] PTY not available...` means BLOCKED/PARTIAL, not pass.
- Access denied for PTY session means FAIL for ownership/lineage.
- Missing first output or unusable action naming means FAIL/PARTIAL with exact evidence.

## Rollback / Safety Notes

- Always terminate any started PTY session.
- Do not use unsupported `start`/`read` action names except as negative UX notes.
- Do not create or depend on an `accessible-view-terminal` feature.

## Output

After execution, create `52-03-SUMMARY.md` in the Phase 52 directory.
