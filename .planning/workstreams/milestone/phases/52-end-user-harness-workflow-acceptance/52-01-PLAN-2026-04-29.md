# Phase 52 Plan 01 — Readiness, Capture Scaffolding, Root-Boundary Preflight

## Goal

Create the Phase 52 acceptance capture surface before any live runtime claim is made. This plan establishes the transcript, evidence matrix, and `.opencode/`/`.hivemind/` boundary snapshot needed to prevent post-hoc or docs-only acceptance claims.

## Scope / Out of Scope

**In scope:** runtime readiness checks, acceptance transcript template, evidence matrix template, root-boundary snapshot, provider/PTY availability notes, and read-only primitive readiness preflight.

**Out of scope:** source-code changes, Phase 49 edits, release-readiness claims, sidecar/product-detox planning, or treating `validate-restart` as actual restart/recovery proof.

## Dependencies / Wave

- **Wave:** 1
- **Depends on:** none
- **Unlocks:** Plans 02-06
- **Reason:** all later live acceptance evidence must write into the capture artifacts created here.

## Files and Commands to Use

Create/update only under the Phase 52 directory:

- `52-RUNTIME-TRANSCRIPT-2026-04-29.md`
- `52-EVIDENCE-MATRIX-2026-04-29.md`
- `52-ROOT-BOUNDARY-SNAPSHOT-2026-04-29.md`

Commands/actions:

```bash
node --version
npm --version
opencode --version
npm run build
```

Live harness/tool actions to capture if available:

- `configure-primitive` in read-only modes: `list`, `read`, `inspect`, or `dryRun`
- `validate-restart` with `projectRoot` set to the workspace root and `verbose: true`

## Acceptance Criteria

- `52-RUNTIME-TRANSCRIPT-2026-04-29.md` exists and has fields for parent session ID, child session ID, delegation ID, PTY session ID, journal/export IDs, and operator notes.
- `52-EVIDENCE-MATRIX-2026-04-29.md` exists and has rows for E52-01 through E52-06 with L1/L2/L3/L4/L5 and PASS/PARTIAL/FAIL/BLOCKED columns.
- `52-ROOT-BOUNDARY-SNAPSHOT-2026-04-29.md` exists and records `.opencode/` as primitives-only and `.hivemind/` as runtime-state-only.
- Runtime readiness is classified as GO or BLOCKED before Plan 02 starts.

## Verification Evidence Expected

- L1: live tool invocation outputs from `configure-primitive` and `validate-restart` if OpenCode tool context is available.
- L4: command outputs from `node --version`, `npm --version`, `opencode --version`, and `npm run build`.
- L5: root-boundary observations, clearly labeled as structural context only.

## Failure / Blocker Handling

- If `opencode --version` or `npm run build` fails, mark all downstream live acceptance plans BLOCKED.
- If provider-backed child sessions are not configured, record BLOCKED for delegation/recovery acceptance instead of falling back to mock evidence.
- If PTY support is unavailable, Plan 03 must classify the PTY lifecycle as BLOCKED/PARTIAL.

## Rollback / Safety Notes

- Do not write to `.opencode/` during this plan.
- Do not mutate or delete `.hivemind/state`.
- Do not claim production-ready or release-ready status from this plan.

## Output

After execution, create `52-01-SUMMARY.md` in the Phase 52 directory.
