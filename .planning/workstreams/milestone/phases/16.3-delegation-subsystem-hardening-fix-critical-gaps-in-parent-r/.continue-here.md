## Top 3 Incidents Requiring Architectural Work

### 1. Notification Firehose (INC-01 + INC-03 + INC-06)
- **Problem:** Delegate-task results arrive as raw system_reminders mid-conversation. Next agent instance has no delegation awareness. Stream ends before tasks complete.
- **Root cause:** No reconciliation layer between delegation lifecycle and conversation lifecycle.
- **User's proposed solution:** Delegation manifest + turn-based briefing (Approach 2 from session line 1863-1926).
- **What needs building:**
  1. PostToolUse hook on `delegate-task` that appends to session-scoped manifest
  2. Pre-response hook or agent behavior that reads manifest at turn start
  3. `retrieved` flag tracking to prevent double-reporting
  4. Optional: pipeline categorization for grouped delegations

### 2. delegation-status List View Bug (INC-02)
- **Problem:** `delegation-status` with `status: "all"` returns `data: []` but `metadata.total: 6`
- **Root cause:** Filter logic doesn't handle `"all"` as wildcard
- **Fix:** Simple — update filter to treat `"all"` or missing status as unfiltered query

### 3. No Save-to-Disk (INC-08)
- **Problem:** No mechanism to export delegation results to human-readable files
- **User asked specifically:** "what if I want to retrieve back these result to human readable in my local disk?"
- **Fix scope:** Design a convenience wrapper or tool option
