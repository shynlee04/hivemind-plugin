## Design Decisions

1. **Read-only assessor** — `assessRecoveryState` never mutates state; corrupt
   checkpoint files are silently skipped (the assessor surfaces the issue
   through its return value rather than throwing).
2. **Atomic writes** — both `createRecoveryCheckpoint` and `repairRecoveryState`
   write to a `.tmp` file with a randomized suffix and `renameSync` into place.
3. **Quarantine-and-reset** — mirrors the convention already used in
   `src/lib/trajectory/ledger.ts` and `src/lib/agent-work-contracts/store.ts`.
4. **Path scope enforcement** — all candidate paths flow through
   `assertPathWithinRoot(.hivemind/state, …)` so a malicious `checkpointPath`
   cannot escape the canonical state directory.
5. **No new runtime deps** — uses only `node:fs`, `node:path`, `node:crypto`.

## Out of Scope (Deferred)

- Wire-up into `delegation-manager` — recovery is invoked manually by tools or
  CLI for now; integration is tracked under future phases.
- CLI command surface — adding a `hivemind-tools recover` subcommand is a
  separate phase under the CLI substrate (Phase 40).
