# Phase 50 Plan 01: Primitive Restart Readiness

## Objective

Turn `validate-restart` from a noisy false-blocking check into a restart-readiness gate that accurately loads the current project primitives and blocks only real missing/circular/runtime errors.

## Tasks

1. Add regression tests for primitive-loader and runtime-validator behavior exposed by restart validation.
2. Fix runtime dependency detection to require explicit `/command`, `command("name")`, or `@agent` references instead of plain prose mentions.
3. Treat agent-level permission denies as valid narrowing of globally allowed tools, while preserving global-deny widening detection.
4. Accept rich OpenCode skill metadata and avoid orphan warnings when `opencode.json` omits optional primitive registries.
5. Repair malformed active primitive frontmatter through symlink targets.
6. Verify with full tests, typecheck, build, and rebuilt `validate-restart` execution.

## Success Criteria

- `npm test` passes.
- `npm run typecheck` passes.
- `npm run build` passes.
- Rebuilt `createValidateRestartTool()` returns `Restart validation passed` for the project root.
