<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Existing JSON artifact → document-store | Previously persisted JSON must be schema-checked and normalized before reuse. |
| Parsed session export Markdown → document-store | Manual export Markdown can contain untrusted text and must remain redacted/bounded when merged. |
| Document model → Markdown renderer | Rendered Markdown must not expose raw secrets or uncontrolled control characters. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-48.5-04 | Tampering | `document-store.ts` JSON parsing | mitigate | Preserve schema guard for `_schema: harness/event-tracker/v1`; throw `[Harness] Failed to parse event-tracker JSON` on invalid content. |
| T-48.5-05 | Information Disclosure | `markdown-renderer.ts` and export metadata merge | mitigate | Preserve `redactTextSecrets()` for last message output and scalar sanitization/truncation for Markdown. |
| T-48.5-06 | Denial of Service | event retention | mitigate | Preserve `MAX_EVENTS_PER_DOCUMENT` bounded retention behavior. |
</threat_model>
