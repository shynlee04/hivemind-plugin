## Architecture Replan

- Replace broad `src/lib/session-artifact-parser.ts` and `src/lib/session-journey-events.ts` placement with a coherent deep module boundary: `src/lib/event-tracker/`.
- Keep the module independent from `continuity.ts` and `delegation-manager.ts`; it may import only leaf helpers or Node filesystem/path APIs.
- Treat the event tracker as an automatic projection writer/parser flow wired by `src/plugin.ts` event observers, not as a user-invoked tool.
- Target path is `.hivemind/event-tracker/{safeSessionStem}.json` and `.hivemind/event-tracker/{safeSessionStem}.md`; no `.hivemind/sessions/journey-events/` writes remain in the corrected implementation.
- Safe session stem is `ses_` plus the sanitized four-character suffix from IDs such as `ses_2b7a`; traversal/injection characters are removed or normalized before file naming.
- Add filesystem adapter seams so negative tests can prove directory/write failures without brittle OS permissions.

## Acceptance Criteria

- OpenCode session events can automatically create/update `.hivemind/event-tracker/ses_xxxx.json` and `.hivemind/event-tracker/ses_xxxx.md` without a tool call.
- Tests fail if the event-tracker directory cannot be created.
- Tests fail if JSON cannot be written.
- Tests fail if Markdown cannot be written.
- Parser tests prove required selective meta/context can be parsed back from JSON and Markdown artifacts.
- Writer tests prove bounded JSON/Markdown output and no raw hook payload firehose.
- Session ID traversal/injection attempts cannot escape `.hivemind/event-tracker/` and resolve to a sanitized `ses_xxxx` filename stem.
- Plugin wiring test proves automatic observer registration.
- No new `.opencode/state` writer is introduced.

## Threat Model

| Threat | Mitigation |
|---|---|
| Raw payload information disclosure | Journey Markdown renders only bounded event metadata and summary fields. |
| Runtime event handling disruption | Plugin observer catches writer errors and never blocks canonical event processing. |
| State root regression | Writer path is hard-coded under project-root `.hivemind/event-tracker`. |
| Path traversal via session ID | Writer derives a `ses_` + 4-character sanitized suffix and joins only under the computed event-tracker directory. |
