## Info

### IN-01: Redundant mkdirSync in delegation-persistence persistDelegations

**File:** `src/lib/delegation-persistence.ts:58,65`
**Issue:** `mkdirSync(dirname(filePath), { recursive: true })` is called twice — once before the temp file write (line 58) and again before the rename (line 65). The second call is redundant because `{ recursive: true }` on an existing directory is a no-op.
**Fix:** Remove line 65 (`mkdirSync(dirname(filePath), { recursive: true })`). The directory is already guaranteed to exist after the first call.

### IN-02: Dead code in noteObservedActivity lifecycle spread

**File:** `src/lib/lifecycle-manager.ts:106-109`
**Issue:** The explicit `phase: currentLifecycle?.phase ?? "running"` on line 107 is immediately overwritten by `...currentLifecycle` on line 108. The spread always wins, making the explicit phase assignment dead code.
**Fix:** Remove the explicit `phase` line and rely on the spread, or reorder to spread first then set observation:
```ts
lifecycle: {
    ...currentLifecycle,
    observation: { source, observedAt: now, detail: `activity noted by ${source}` },
},
```

### IN-03: Non-null assertion on guaranteed value in getGovernancePersistenceState

**File:** `src/lib/continuity.ts:437`
**Issue:** `emptyStore().governance!` uses a non-null assertion on a value that is always defined (`emptyStore()` always returns a `governance` field). The `!` is technically safe but is a code smell that suppresses the type checker unnecessarily.
**Fix:** Replace with explicit fallback: `(ensureStoreLoaded().governance ?? emptyStore().governance)`
