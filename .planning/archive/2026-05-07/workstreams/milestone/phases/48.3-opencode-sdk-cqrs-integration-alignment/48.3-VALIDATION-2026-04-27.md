## Validation Scope

Phase 48.3 passes only when implementation evidence proves all three success criteria from `48.3-CONTEXT-2026-04-27.md`:

1. SDK calls match installed package contracts and have regression evidence for request-shape drift.
2. CQRS boundaries are explicit: tools mutate, hooks observe, shared modules stay leaf-safe.
3. Plugin composition has no hidden business logic or unsupported permission/session payload assumptions.

These checks are evidence aids for plan execution summaries. They should be reported as facts, not governance scripts:

- Search for unmanaged SDK calls outside approved wrappers:
  - Allowed direct SDK request-shape files: `src/lib/session-api.ts`, `src/lib/app-api.ts`.
  - Any remaining direct SDK calls elsewhere require documented impossible-case rationale and focused tests.

  - Hook modules must not directly call continuity/delegation persistence writers.
  - OpenCode response shaping (`messages.transform`, `shell.env`, metadata/output shaping) must be documented as response shaping.
- Confirm runtime state root discipline:
  - No new runtime state writes under `.opencode/`.
  - Canonical deep module state remains under `.hivemind/` via existing write-side services.

## Pass / Fail Criteria

### PASS

Phase 48.3 can be marked validation-ready when:

- All five focused command groups pass.
- Phase-final verification passes: `npm run typecheck && npm test && npm run build`.
- Each plan summary cites the evidence IDs it satisfies.
- No unmanaged SDK calls, hook-side durable writes, or hidden plugin business logic remain without documented proof and tests.
- Phase 48.2 security gates remain preserved: redaction, scoped PTY/delegation visibility, and category narrowing do not regress.

### FAIL

Phase 48.3 fails validation when any of the following is true:

- Any focused or phase-final command fails.
- SDK request shapes are asserted only indirectly or through mocks that do not check path/body/query structure.
- Hook modules perform durable persistence writes or hidden state mutation.
- Plugin extraction preserves business logic in `src/plugin.ts` or moves persistence into hook modules.
- Public no-PTY behavior remains unreachable or undocumented.
