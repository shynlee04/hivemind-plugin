<objective>
Build the config compiler library and tool surface that enables programmatic compilation of OpenCode primitives from JSON/YAML into .md files, and reverse decompilation from .md files back to structured data.

Purpose: Enable the configure-primitive tool to accept JSON/YAML specs, validate them against schema-kernel schemas, compile them to valid OpenCode .md files, and support round-trip fidelity. This is the compilation engine the workflow skill (Plan 04) and orchestrator (Plan 05) will call.

Output: `src/lib/config-compiler.ts` (core lib) + `src/tools/configure-primitive.ts` (tool wrapper).
</objective>

  <name>Task 1: Create config-compiler.ts — Core compile and decompile library</name>
  <files>src/lib/config-compiler.ts</files>
  <action>
    Install dependencies first: `npm install yaml gray-matter`

    Create `src/lib/config-compiler.ts` (target: ~250-350 LOC, max 500).

    **Step 1 — Types (top of file):**

    ```typescript
    import { stringify as yamlStringify } from "yaml"
    import matter from "gray-matter"
    import {
      AgentFrontmatterSchema,
      CommandFrontmatterSchema,
      SkillFrontmatterSchema,
    } from "../schema-kernel/index.js"
    import type {
      AgentFrontmatter,
      CommandFrontmatter,
      SkillFrontmatter,
    } from "../schema-kernel/index.js"
    ```

    Define these exported types:
    - `CompileResult` = `{ success: boolean; content: string; filePath: string; errors?: string[] }`
    - `DecompileResult<T>` = `{ success: boolean; spec: T | null; body: string; warnings: string[] }`
    - `AgentSpec` = `{ name: string; frontmatter: AgentFrontmatter; body: string }`
    - `CommandSpec` = `{ name: string; frontmatter: CommandFrontmatter; body: string }`
    - `SkillSpec` = `{ name: string; frontmatter: SkillFrontmatter; body: string }`
    - `CompileOptions` = `{ scope?: "project" | "global"; basePath?: string }`
      - `scope: "project"` (default): uses project-relative `.opencode/` directory
      - `scope: "global"`: uses `~/.config/opencode/` (or `process.env.OPENCODE_CONFIG_DIR` if set)
      - `basePath`: optional override for the base directory (takes precedence over scope resolution)

    **Step 2 — Path resolution helper:**

    `resolveBasePath(options?: CompileOptions): string`:
     1. If `options.basePath` is provided, validate it (see path traversal rules below) and return it
     2. If `options.scope === "global"`:
        - Use `process.env.OPENCODE_CONFIG_DIR` if set
        - Otherwise use `path.join(os.homedir(), ".config", "opencode")`
     3. Default (`scope: "project"` or omitted): return `".opencode"` (project-relative)

    `validateOutputPath(filePath: string, basePath: string): string | null`:
     - Reject paths containing `..` (path traversal attempt)
     - Reject absolute paths that resolve outside `basePath`
     - Return `null` if valid, otherwise return error message string
     - Use `path.resolve(basePath, filePath)` and verify it starts with `path.resolve(basePath)`

    **Step 3 — Compile functions:**

    `compileAgent(spec: AgentSpec, options?: CompileOptions): CompileResult`:
     1. Validate frontmatter with `validateWithFallback(AgentFrontmatterSchema, AgentFrontmatterSchemaLenient, spec.frontmatter, "agent frontmatter")` — this tries strict validation first, strips unknown fields with a warning if needed, and never crashes. If validation fails, return `{ success: false, errors: [...] }`
     2. Stringify frontmatter to YAML using `yamlStringify(spec.frontmatter, { lineWidth: -1 })` (no line wrap)
     3. Build content: `"---\n" + yamlString + "---\n\n" + spec.body + "\n"`
     4. Resolve base path via `resolveBasePath(options)`
     5. Compute relative filePath: `"agents/" + spec.name + ".md"`
     6. Validate with `validateOutputPath(relativePath, basePath)` — if invalid, return `{ success: false, errors: [validationMessage] }`
     7. Compute final filePath: `path.join(basePath, relativePath)`
     8. Return `{ success: true, content, filePath }`

    `compileCommand(spec: CommandSpec, options?: CompileOptions): CompileResult`:
    Same pattern but uses `CommandFrontmatterSchema`, relative path: `"commands/" + spec.name + ".md"`

    `compileSkill(spec: SkillSpec, options?: CompileOptions): CompileResult`:
    Same pattern but uses `SkillFrontmatterSchema`, relative path: `"skills/" + spec.name + "/SKILL.md"`

    **Step 4 — Decompile functions:**

    `decompileAgent(md: string): DecompileResult<AgentSpec>`:
    1. Use `matter(md)` to extract frontmatter object + body content
    2. Validate with `validateWithFallback(AgentFrontmatterSchema, AgentFrontmatterSchemaLenient, frontmatter, "decompile agent")`
    3. If validation fails, return `{ success: false, spec: null, body, warnings: zodErrors }`
    4. Return `{ success: true, spec: { name: "unknown", frontmatter: result.data, body }, warnings: result.warnings }`
    NOTE: name is set to "unknown" because agent .md filename is the name, not embedded in frontmatter.

    `decompileCommand(md: string): DecompileResult<CommandSpec>` — same pattern with `CommandFrontmatterSchema`

    `decompileSkill(md: string): DecompileResult<SkillSpec>`:
    Same pattern with `SkillFrontmatterSchema`, but also extract `name` from `frontmatter.name` field (skills DO embed name in frontmatter).

    **Step 5 — Helper function:**

    `formatCompileError(zodError: import("zod").ZodError): string[]`:
    Map ZodError issues to human-readable strings: `issue.path.join(".") + ": " + issue.message`.

    **Step 6 — Batch compile function (R-03 atomic batch operations):**

    Define exported type `BatchCompileResult = { results: CompileResult[]; allSucceeded: boolean; failureReport?: CompileResult }`.

    `batchCompile(specs: Array<{ type: "agent" | "command" | "skill"; spec: AgentSpec | CommandSpec | SkillSpec }>, options?: { atomic?: boolean; compileOptions?: CompileOptions }): BatchCompileResult`:
     1. Iterate over specs array, route each to the correct compile function based on `type`, passing `options.compileOptions`
     2. Collect all `CompileResult` objects into `results` array
     3. If `options.atomic === true` AND any result has `success: false`:
        - Return `{ results, allSucceeded: false, failureReport: <first failed result> }` — caller handles rollback
     4. If all succeed: return `{ results, allSucceeded: true }`
     5. Default `atomic` to `false` if options not provided
     6. **Logical atomicity:** All specs are validated before any content is generated. If atomic=true and any fail, NO files are written. This is logical atomicity (all-or-nothing validation), NOT filesystem atomicity (no atomic write guarantees).

    **Constraints:**
    - NO direct `fs` imports — compiler produces content strings, caller handles I/O
    - NO `any` types
    - Max 500 LOC
    - Export: compileAgent, compileCommand, compileSkill, decompileAgent, decompileCommand, decompileSkill, batchCompile, resolveBasePath, validateOutputPath, plus all type exports
    - Import `path` from `node:path` and `os` from `node:os` for path resolution

    ```typescript
    export function createConfigurePrimitiveTool(): ReturnType<typeof tool> {
      const s = tool.schema
      return tool({
        description: "Configure an OpenCode primitive (agent, command, or skill) from JSON/YAML input. Validates against schemas, compiles to .md format. Use dryRun to preview without writing.",
        args: {
          primitive: s.string().describe("Type: agent, command, or skill"),
          spec: s.string().describe("JSON or YAML configuration string"),
          dryRun: s.boolean().describe("Preview mode — compile without writing"),
          validate: s.boolean().describe("Run schema validation before compilation"),
          scope: s.string().describe("project or global scope"),
          overwrite: s.boolean().describe("Overwrite existing files"),
        },
        async execute(rawArgs, _context): Promise<string> {
          // 1. Parse input with ConfigurePrimitiveInputSchema
          // 2. Parse spec string: try JSON.parse first, fall back to YAML.parse
          // 3. Route to correct compile function based on primitive type, pass { scope: rawArgs.scope }
          //    - "agent" → compileAgent({ name, frontmatter: parsedSpec, body: parsedSpec.body ?? "" }, { scope: rawArgs.scope })
          //    - "command" → compileCommand(...)
          //    - "skill" → compileSkill(...)
          // 4. If compilation fails, return error via renderToolResult
          // 5. If dryRun: return success("Dry run complete", { content, filePath })
          // 6. If !dryRun:
          //    a. Check if file exists AND overwrite is false → return error("File exists, use overwrite: true to replace")
          //    b. mkdirSync(path.dirname(filePath), { recursive: true })
          //    c. await fs.writeFile(filePath, content, "utf-8") — use async writeFile, NOT writeFileSync
          //    d. Return success via renderToolResult
        },
      })
    }
    ```

    **Step 3 — Spec parsing helper:**

    Private function `parseSpec(specStr: string): { frontmatter: Record<string, unknown>; body: string }`:
    1. Try `JSON.parse(specStr)` — if succeeds, extract `body` field if present, rest is frontmatter
    2. If JSON fails, try `yaml.parse(specStr)` from `yaml` package — same extraction
    3. If both fail, return error response with parse failure details

    **Constraints:**

    - The tool IS allowed to do filesystem I/O (unlike the lib) — this is the write layer
    - Use `fs.promises.writeFile` (async) instead of `writeFileSync` (sync)
    - Use `mkdirSync(path.dirname(filePath), { recursive: true })` before write to ensure directory exists
    - Respect `overwrite: false` (default) — reject if target file already exists
