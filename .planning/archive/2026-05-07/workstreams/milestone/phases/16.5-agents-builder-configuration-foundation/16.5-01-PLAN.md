    Create `src/lib/cross-primitive-validator.ts` (target: ~300-400 LOC, max 500).

    **Step 1 — Types and interfaces (top of file):**

    ```typescript
    import type {
      AgentFile, CommandFile, SkillFile, ToolFile,
      MCPServerConfig, OpenCodeConfig
    } from "../schema-kernel/index.js"
    ```

    Define these exported types:
    - `ValidationSeverity` = `"block" | "warn" | "info"`
    - `ValidationCategory` = one of these 8 string literals: `"agent-command-binding"`, `"permission-deadlock"`, `"role-overlap"`, `"missing-skill-dependency"`, `"mcp-server-gap"`, `"rule-file-gap"`, `"missing-model"`, `"command-skill-ref"`
    - `ValidationIssue` = `{ severity: ValidationSeverity; category: ValidationCategory; message: string; source: { type: string; name: string }; target?: { type: string; name: string } }`
    - `ValidationReport` = `{ valid: boolean; errors: ValidationIssue[]; warnings: ValidationIssue[]; info: ValidationIssue[] }`
    - `PrimitiveMap` = `{ agents: Map<string, AgentFile>; commands: Map<string, CommandFile>; skills: Map<string, SkillFile>; tools: Map<string, ToolFile>; mcpServers: Map<string, MCPServerConfig>; config: OpenCodeConfig }`

    **Step 2 — Main function `validateCrossPrimitive(primitives: PrimitiveMap): ValidationReport`:**

    Collect all issues from 6 detector functions, partition by severity, set `valid = errors.length === 0`.

    **Step 3 — 6 detector functions (private, not exported):**

    1. `detectMissingAgentBindings(commands, agents)` — For each command with `frontmatter.agent` set, check that agent name exists in the agents map. Severity: `"block"`. Category: `"agent-command-binding"`.

    2. `detectPermissionDeadlocks(agents, commands)` — For each agent with `permission` field containing a `"ask"` action, check if any of its bound commands (commands referencing this agent) need that denied tool. Severity: `"warn"`. Category: `"permission-deadlock"`.

    3. `detectRoleOverlaps(agents)` — Find all agents with `mode === "primary"`. Compare descriptions using simple keyword overlap (split into words, count intersection). If overlap > 50% of shorter description's unique words, emit warning. Severity: `"warn"`. Category: `"role-overlap"`.

    4. `detectMissingSkillDependencies(agents, skills)` — Scan each agent's `body` string for `skill()` tool calls extracting skill names. Check each referenced skill exists in the skills map. Severity: `"warn"`. Category: `"missing-skill-dependency"`.

    5. `detectMCPServerGaps(tools, mcpServers)` — Scan tool definitions for MCP-related patterns (names containing `"mcp"` or args referencing MCP server names). Check referenced MCP servers exist. Severity: `"warn"`. Category: `"mcp-server-gap"`.

    6. `detectRuleFileGaps(config)` — For each path in `config.instructions`, check it looks like a valid path (non-empty string). This is a structural check, not filesystem — just validate the config value format. Severity: `"block"`. Category: `"rule-file-gap"`.

    **Step 4 — Helper `partitionBySeverity(issues: ValidationIssue[])`:**

    Returns `{ errors: ValidationIssue[], warnings: ValidationIssue[], info: ValidationIssue[] }` by filtering on severity.

    **Constraints:**
    - NO direct `fs` imports — accept PrimitiveMap as input
    - NO `any` types
    - Max 500 LOC
    - Export only: `validateCrossPrimitive`, `PrimitiveMap`, `ValidationReport`, `ValidationIssue`, `ValidationSeverity`, `ValidationCategory`
