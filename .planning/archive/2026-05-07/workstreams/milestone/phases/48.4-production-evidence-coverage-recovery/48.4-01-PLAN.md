**Step 1:** Install `@vitest/coverage-v8` as a devDependency:
```bash
npm install -D @vitest/coverage-v8
```
This must succeed without errors. The package is compatible with vitest ^1.0.0.

**Step 2:** Update `vitest.config.ts` to add `provider: 'v8'` and `thresholds` to the existing `coverage` block. Preserve all existing settings (`globals`, `include`, `reporter`, `exclude`). Add:

```typescript
coverage: {
  provider: 'v8',                          // NEW
  include: ['src/**/*.ts'],                // EXISTING
  exclude: ['src/index.ts', 'src/**/index.ts'], // EXISTING
  reporter: ['text', 'lcov'],              // EXISTING
  thresholds: {                            // NEW
    statements: 70,
    branches: 60,
    functions: 70,
    lines: 70,
  },
}
```

Do NOT remove any existing configuration. The `thresholds` block is placed INSIDE the existing `coverage` object, as a peer to `include`, `exclude`, and `reporter`.

**Step 3:** Verify the install succeeded by checking `npx vitest run --coverage --reporter=text` produces a coverage table (thresholds will likely fail at this point — that's expected and will be resolved in Waves 1-3).

**Why these thresholds:**
- 70% statements/functions/lines: achievable given 41 existing behavioral test files
- 60% branches: branches are harder to cover in plugin code with many guard clauses; raise after Wave 1
- Types-only `index.ts` re-export barrels excluded (they contain no logic)
- These are initial thresholds — tighten after all waves complete (see Plan 05)
