## Resolution

root_cause: |
  OpenCode server's `/agent` endpoint returns agent objects with missing required
  string fields (name, description, mode, color, variant, prompt, model.modelID,
  model.providerID). The SDK's Zod schema validates the response and rejects it
  with 8× "expected string, received undefined" errors. `unwrapData()` extracts
  these errors and throws `[Harness] ...`. The delegate-task tool catches this
  and returns it as a tool error. delegation-status works because it never calls
  `client.app.agents()` — it only reads from in-memory Maps.

fix: |
  Wrapped `client.app.agents()` + `unwrapData()` in `validateAgent()` with a
  try/catch that detects the specific Zod response-validation error
  (`"expected string, received undefined"`). When this occurs, we log a warning
  and gracefully degrade to unvalidated agent acceptance (`return { name: agent }`),
  allowing delegation to proceed instead of hard-failing.

  Also removed temporary diagnostic logging from `unwrapData()` in helpers.ts.
