| # | Surface | Owner | Authority Path |
|---|---------|-------|----------------|
| 1 | Tools | `tools-owner` | `src/tools/` |
| 2 | Hooks | `hooks-owner` | `src/hooks/` |
| 3 | Plugin | `plugin-owner` | `src/plugin/` |
| 4 | Core | `core-owner` | `src/core/` |
| 5 | Shared | `shared-owner` | `src/shared/` |
| 6 | Schema Kernel | `schema-owner` | `src/schema-kernel/` |
| 7 | SDK Supervisor | `supervisor-owner` | `src/sdk-supervisor/` |
| 8 | Docs | `docs-owner` | `docs/` |
| 9 | Tests | `test-owner` | `tests/` |
| 10 | Config | `config-owner` | Root config files |
| 11 | Agents | `agent-owner` | `agents/`, `.opencode/agents/` |
| 12 | Runtime | `runtime-owner` | `.hivemind/`, `dist/` |

## Ownership Verification

Before committing changes to a surface, verify:

1. **The committing agent has authority** over the surface class
2. **No cross-surface writes** without explicit permission
3. **Shared surfaces** (`src/shared/`) require coordination — any agent may read, write requires declaration

Verification is path-based:

```bash
# Check which surface a file belongs to
echo "src/tools/runtime/status.ts" | awk -F/ '{print $1"/"$2"/"}'
# → src/tools/ → surface: tools, owner: tools-owner
```

## Conflict Resolution

When a commit touches multiple surfaces:

1. Each surface class is a separate batch (no cross-surface mixing)
2. If surfaces have conflicting owners, the orchestrator arbitrates
3. If a file is claimed by two surfaces (e.g., shared test utility), use the dominant surface:
   - Test files always belong to `test-owner` regardless of location
   - Shared utilities belong to `shared-owner`
   - Config in non-config paths belongs to the path's surface owner

## Cross-Surface Commit Policy

Commits may span multiple surfaces ONLY when:
- All surfaces share the same owner (e.g., `tools-owner` owns both `src/tools/` and `tests/tools/`)
- The orchestrator explicitly approves multi-surface batching
- Each surface in the batch has its rollback method independently verified

## Ownership Override

Override ownership via `.hivemind-ownership.json`:

```json
{
  "overrides": {
    "src/shared/paths.ts": "schema-owner",
    "tests/integration/": "core-owner"
  },
  "shared_writers": ["tools-owner", "core-owner"]
}
```

Shared writers are agents allowed to write to `src/shared/` without escalation.

## Governance Surfaces

| Surface | Rule |
|---------|------|
| `AGENTS.md` | Root governance — owned by orchestrator, not any single agent |
| `opencode.json` | Config — owned by config-owner, requires user approval to change |
| `*.skill.md` | Skills — owned by skill-author, but governed by pack-level AGENTS.md |
| `.opencode/` | User project config — DIRECT_WRITE_BAN for all agents |

## Runtime Surfaces

| Surface | Rule |
|---------|------|
| `.hivemind/` | Runtime output — generated by tools, never hand-written |
| `dist/` | Build output — generated by build pipeline, never hand-written |
| `node_modules/` | Dependencies — never modified directly |
