# Emergence Path

## Verified Gap
- The root router package exists, but other active root skills do not currently point back to it.
- The emergence path therefore has to be made explicit by staged documentation first, then by later inbound route cleanup.

## Path
1. `Router entry`
   - user invokes `use-hivemind-detox-refactor`
2. `Concern classification`
   - choose `context`, `delegation`, `git-memory`, `codemap`, `system-debug`, or a cross-family path
3. `Baseline evidence`
   - verify authority, drift, and route validity before edits
4. `Continuity decision`
   - if history/memory quality matters, enter `git-memory`
5. `Partitioning decision`
   - choose sequential vs parallel subagents after scope is isolated
 6. `Codemap or debug branch`
   - route to `codemap` first when structure is unclear
   - route to `system-debug` first when root cause is unresolved
   - route to refactor strategy only after evidence is stable
7. `Restoration`
   - restore slice-by-slice with family-owned mechanics and verification gates
8. `Governance reintegration`
   - update `AGENTS.md`, synthesis outputs, and stabilization follow-up

## Meaning
- The router is not just a concern guard.
- It is the staged orchestration surface that chooses which background knowledge becomes active and when.
- The center stays thin by routing deep work outward to branch families instead of re-implementing them.
- The 8 steps here are the compressed navigation view; the 11-stage workflow expands them into execution-ready contracts.
