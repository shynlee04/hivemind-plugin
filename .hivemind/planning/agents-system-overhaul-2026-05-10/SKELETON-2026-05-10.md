## A. NAMING TAXONOMY

### Prefix Rules

| Prefix | Lineage | Scope | Shipped | Description |
|--------|---------|-------|---------|-------------|
| `hm-` | Hivemind | Product development runtime | YES | Agents/skills that DO work in user's project |
| `hf-` | Hivefiver | Meta-builder configuration | YES | Agents/skills that CONFIGURE OpenCode primitives |
| `gate-` | Internal | Quality gate triad | **NO** — project-internal only | Lifecycle→Spec→Evidence gate pipeline (NOT shipped to users) |
| `stack-` | Reference | Tech stack reference | **CONDITIONAL** — project-specific | Cached SDK/library API references (current 6 are THIS project's deps) |
| `gsd-` | GSD | Developer tooling | NO | Build tools — NOT shipped, NOT in scope |

### Level Rules

| Level | Role | Mode | Can Delegate To | Can Be Called By |
|-------|------|------|-----------------|------------------|
| L0 | Front-facing orchestrator | primary | ALL L1, L2, L3 (both lineages) | User (Tab key) |
| L1 | Mid-handoff coordinator | subagent | L2, L3 (own lineage + cross per rules) | L0 only |
| L2 | Domain specialist | subagent | L2 peers (limited), L3 skills | L0, L1, L2 peers |
| L3 | No agents — reference/reference skills only | N/A | N/A | Loaded by L2 skills |
