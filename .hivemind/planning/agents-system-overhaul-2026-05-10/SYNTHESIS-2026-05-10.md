## 1. Executive Summary

Two investigations — a deep lifecycle research report and a cross-lineage audit — revealed a fundamental classification error in the planning documents: **9 skills (3 gate-\* + 6 stack-\*) were incorrectly classified as "shipped primitives"** when they are project-internal governance and reference artifacts. 3. **Shipped primitive count is 49** (35 hm-\* + 13 hf-\* + 1 opencode-config-workflow), NOT 58.
4. **hf-l0-orchestrator agent file EXISTS** — previously reported MISSING; confirmed at 19,410 bytes (2026-05-09 19:48). Original RESEARCH glob error. See STATE-2026-05-10.md §1.
5. **Zero delegation boundary violations** — the hm↔hf delegation rules are clean; conflicts are in metadata, not architecture.
6. **Delegation loop coverage is 87.5%** — 49 of 56 agents reachable from L0; 21 delegation gaps; 4 blocking agents. Source: MATRIX.
7. **45 gaps across all primitives** — 14 P0, 12 P1, 10 P2, 9 P3. 15 agents have zero skills (27% of fleet). Source: GAPS.
8. **5 broken command→agent→skill chains** — commands that reference agents which reference skills that don't exist. 6 referenced commands are missing entirely. Source: LIFECYCLE.

## 5. Naming Convention — Confirmed Rules

### Per-Tier Naming Rules

| Tier | Prefix Format | Level Designation | Example |
|------|--------------|-------------------|---------|
| Core Shipped (hm) | `hm-l{level}-{domain-role}` | l0, l1, l2, l3 | `hm-l2-debugger`, `hm-l3-deep-research` |
| Core Shipped (hf) | `hf-l{level}-{domain-role}` | l0, l1, l2 | `hf-l2-skill-builder` |
| Project-Internal (gate) | `gate-l3-{gate-name}` | l3 (loaded by L2 agents) | `gate-l3-evidence-truth` |
| Project-Internal (stack) | `stack-l3-{tech-name}` | l3 (reference material) | `stack-l3-vitest` |
| Developer Tooling (gsd) | `gsd-{role}-{name}` | varies | `gsd-code-reviewer` |

### Naming Rules Enforced

1. **Agent names and skill names MUST be disjoint** — `hf-l2-meta-builder` violates this (both agent AND skill).
2. **Level designations in names MUST match `layer` metadata** — 19 skills violate this.
3. **L2 agents MUST use `mode: subagent`** — 3 agents violate this.
4. **All shipped agents MUST have `domain` field** — 15 agents violate this.
5. **gate-\* prefix = project-internal governance** — NOT counted as shipped.
6. **stack-\* prefix = project-specific reference** — NOT counted as shipped.
7. **gsd-\* prefix = developer tooling** — NEVER shipped, NEVER in scope.

### Decision: Keep gate-l3 and stack-l3 Naming

The `l3` designation is architecturally correct because:
- L3 = "loaded by L2 agents" — both gate-\* and stack-\* are loaded by L2 agents.
- The prefix (gate- vs stack-) communicates PURPOSE (governance vs reference).
- No renaming needed — this is a classification correction, not a naming correction.
