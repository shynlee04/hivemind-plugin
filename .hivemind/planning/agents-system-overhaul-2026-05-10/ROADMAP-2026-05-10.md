## Phase 1: Permissions Fix — All `deny` → `ask` for Shipped Agents

| Field | Value |
|-------|-------|
| **ID** | PH-01 |
| **REQ Mapping** | REQ-02 |
| **Priority** | P0 — CRITICAL |
| **Depends On** | None (entry point) |
| **Estimated Scope** | 56 agent YAML files — mechanical find-and-replace |

### Description

Change all `deny` permission values to `ask` across all 45 hm-* and 11 hf-* shipped agents. This is the highest-impact, lowest-risk change — it unblocks every agent's ability to request tool access at runtime.

### Execution Steps

1. Glob all `.opencode/agents/hm-*.md` and `.opencode/agents/hf-*.md` files
2. For each file, locate the `permission:` block in YAML frontmatter
3. Replace every `deny` value with `ask`
4. Preserve `allow` values (these are intentional grants)
5. Special case: hf-l2-prompter retains `edit: allow` and `write: allow`
6. Verify no agent still uses the deprecated `tools` field
7. Spot-check 5 random agents post-change

### Verification Criteria

- [ ] `grep -r "deny" .opencode/agents/hm-*.md .opencode/agents/hf-*.md` returns zero matches in permission blocks
- [ ] All 56 agents have `ask` or `allow` for every permission entry
- [ ] hf-l2-prompter still has `edit: allow` and `write: allow`
- [ ] No agent uses deprecated `tools` field

### Risk Notes

- **Low risk.** Mechanical change. No behavior change at runtime (agents still need user approval for previously-denied tools).
- **Rollback:** git revert restores all deny values instantly.
