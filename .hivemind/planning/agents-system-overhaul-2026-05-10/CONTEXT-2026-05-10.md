## 2. OpenCode SDK Compliance Gaps

### Valid YAML Frontmatter Fields (from official docs)

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| description | string | YES | One-line role + triggers |
| mode | string | No | "primary" \| "subagent" \| "all" |
| hidden | boolean | No | Hides from @ autocomplete |
| model | string | No | provider/model-id format |
| temperature | number | No | 0.0–1.0 |
| top_p | number | No | 0.0–1.0 |
| steps | number | No | Max agentic iterations |
| disable | boolean | No | Disables agent |
| prompt | string | No | System prompt or file ref |
| color | string | No | Hex or theme color |
| permission | object | No | **Preferred** over deprecated `tools` |

### Fields NOT in OpenCode SDK (non-standard / passthrough)

| Field | Used By | Status |
|-------|---------|--------|
| domain | 40 agents | Passthrough — NOT consumed by OpenCode runtime |
| skill/skills | 1 agent | Malformed — `skill: allow` instead of `skills:` array |
| instructions: (plural) | 7 agents | **BROKEN** — correct field is `instruction:` (singular) |
| reasoningEffort | 1 agent | Provider passthrough (valid for OpenAI) |

### Permission System (valid values)

- `allow` — run without approval
- `ask` — prompt user for approval (once/always/reject)
- `deny` — block entirely
- `tools` field is **DEPRECATED** — `permission` is preferred
