### File Naming Conventions

#### Pattern 1: `ses_YYYY-MM-DDTHHMMSS_<purpose>_<agent>.json`
- **Location**: Project root (orphaned) and `.hivemind/sessions/`
- **Example**: `ses_2026-03-25T204658_implementation_unknown.json`
- **Interpretation**: 
  - `ses_` = prefix
  - `2026-03-25T204658` = ISO timestamp (UTC)
  - `implementation` = purpose class
  - `unknown` = agent name
- **Status**: LEGACY format, missing SDK session ID suffix

#### Pattern 2: `ses_YYYY-MM-DDTHHMMSS_<purpose>_<agent>_ses_<sdkId>.json`
- **Location**: `.hivemind/sessions/`
- **Example**: `ses_2026-03-28T143724_implementation_unknown_ses_2cc9fc48bffeKhlJPYEYXo4wGc.json`
- **Interpretation**: Same as above PLUS `_ses_<sdkId>` suffix
- **Status**: CURRENT v2 consolidated format

#### Pattern 3: `session-ses_<sdkId>.md`
- **Location**: Project root
- **Example**: `session-ses_2dad.md`, `session-ses_2e0b.md`, `session-ses_2e54.md`
- **Interpretation**: Manual session exports (markdown format)
- **Content**: Full session history with user messages, assistant outputs, tool calls

}

### Anomalies

1. **Naming Collision**: `getSessionPath` exists in BOTH `src/shared/paths.ts` and `src/features/event-tracker/consolidated-writer.ts` with DIFFERENT return types:
